# Счета и платежи (Invoices & Payments)

## Общая концепция

Счета (Invoices) и платежи (Payments) — основа доходной стороны финансового модуля. Счёт — это конкретный документ на конкретную сумму, выставленный клиенту. Платёж — подтверждение получения денег по счёту.

```
Источник (Order/Subscription/Domain) → Invoice → Payment → Revenue
```

---

## Invoice (Счёт)

### Определение

Invoice — конкретный счёт на конкретную сумму, привязанный к источнику (заказ, подписка, домен/сервис).

### Типы счетов

| Тип | Источник | Создание | Периодичность |
|-----|----------|----------|---------------|
| **Development** | Order (заказ на разработку) | Вручную продажником/финансистом | Разовый (предоплата, остаток) |
| **Extension** | Extension Order (доработка) | Вручную | Разовый |
| **Subscription** | Subscription (подписка) | Автоматически по billing_day | Ежемесячный |
| **Domain/Service** | Domain, License, Service | Вручную или по напоминанию | Ежегодный (или по графику) |

### Поля счёта

| Поле | Описание |
|------|----------|
| `invoice_number` | Уникальный номер счёта |
| `type` | Тип (Development / Extension / Subscription / Domain-Service) |
| `source_entity` | Ссылка на источник (Order ID / Subscription ID / Domain ID) |
| `project` | Проект, к которому относится |
| `company` | Компания-плательщик |
| `contact` | Контактное лицо |
| `amount` | Сумма счёта |
| `currency` | Валюта |
| `tax_status` | Tax / Tax-Free (наследуется от источника) |
| `billing_date` | Дата выставления |
| `due_date` | Срок оплаты |
| `description` | Описание (что оплачивается) |
| `stage` | Текущая стадия |
| `created_by` | Кто создал |
| `paid_date` | Дата фактической оплаты (заполняется при оплате) |

### Стадии счёта (Kanban Board)

```
New → Create in Gov System → Send Message → Overdue → On Hold → Paid / Unpaid
```

| Стадия | Описание | Действие |
|--------|----------|----------|
| **New** | Счёт только создан | Ожидание автоматического перехода (~1 час) |
| **Create in Gov System** | Нужно создать в государственной системе | Бухгалтер получает уведомление. Пропускается если Tax-Free |
| **Send Message** | Отправить уведомление клиенту | WhatsApp-сообщение: «Ваш счёт готов, оплатите до [дата]» |
| **Overdue** | Срок оплаты прошёл | Первое напоминание. Через 2 дня — второе |
| **On Hold** | Оплата отложена по договорённости | Ручной перевод финансовым директором |
| **Paid** | Оплачен | Финальная стадия. Создаётся Payment |
| **Unpaid** | Не оплачен (закрыт без оплаты) | Финальная стадия. Фиксация неоплаты |

### Автоматизация жизненного цикла счёта

```
1. Счёт создан (вручную или авто из подписки)
   │
   ▼ (~1 час)
2. Авто-переход в "Create in Gov System"
   ├─ Tax: уведомление бухгалтеру (компания, сумма, описание, тип)
   │       Бухгалтер создаёт счёт в гос. системе
   └─ Tax-Free: стадия пропускается
   │
   ▼
3. Авто-переход в "Send Message"
   └─ WhatsApp клиенту: "Ваш счёт №X на сумму Y готов. Оплатите до [due_date]"
   │
   ▼ (due_date прошёл)
4. Авто-переход в "Overdue"
   └─ Первое напоминание клиенту
   │
   ▼ (+2 дня)
5. Второе напоминание
   │
   ▼ (возможны дальнейшие эскалации)
6. Финансовый директор:
   ├─ Проверяет банк ежедневно
   ├─ Отмечает оплаченные → Paid
   └─ Нажимает "Run Automation" → напоминания только по неоплаченным
```

### Кнопка «Run Automation»

Финансовый директор имеет специальную кнопку, которая запускает цикл напоминаний:
- Система проходит по всем счетам в статусе Overdue
- Отправляет напоминания только по неоплаченным счетам
- Не затрагивает счета в статусе On Hold
- Позволяет контролировать момент отправки (не автоматически в полночь, а по решению финдиректора)

### Billing Dates

Разные клиенты имеют разные даты выставления счетов:

| Клиент | Billing Day | Описание |
|--------|-------------|----------|
| Клиент A | 5 | Счёт выставляется 5-го числа каждого месяца |
| Клиент B | 15 | Счёт выставляется 15-го числа |
| Клиент C | 20 | Счёт выставляется 20-го числа |

Billing day хранится в подписке и/или в настройках компании клиента.

### Специфика Tax-Free счетов

- Стадия «Create in Gov System» пропускается автоматически
- Счёт переходит сразу к отправке сообщения клиенту
- Типичные случаи: зарубежные клиенты, pass-through платежи
- Tax-Free статус наследуется от источника (Order, Subscription, Domain) и не меняется

## Payment (Платёж)

### Определение

Payment — запись о фактическом получении денег. Создаётся финансовым директором при подтверждении оплаты.

### Процесс создания

```
Финансовый директор проверяет банк
  → Находит поступление
  → Открывает соответствующий Invoice
  → Отмечает как "Paid"
  → Система автоматически создаёт Payment
```

### Поля платежа

| Поле | Описание |
|------|----------|
| `payment_id` | Уникальный идентификатор |
| `invoice_id` | Ссылка на счёт (1 платёж = 1 счёт) |
| `amount` | Сумма платежа |
| `payment_date` | Дата поступления |
| `payment_method` | Способ оплаты (банковский перевод, карта и т.д.) |
| `confirmed_by` | Кто подтвердил (финансовый директор) |
| `project` | Проект (наследуется от Invoice) |
| `order` | Заказ (наследуется от Invoice) |

### Каскадные события при создании Payment

Создание платежа запускает цепочку событий в зависимости от типа счёта:

#### При оплате Development Invoice (заказ на разработку)

```
Payment создан
  ├─ Обновление статуса Order:
  │   ├─ Первая оплата → Order переходит из "Новый" в "Предоплата получена"
  │   └─ Последняя оплата → Order переходит в "Полностью оплачен"
  │
  ├─ Создание бонуса продажнику (если применимо):
  │   └─ % от суммы заказа → карточка в Bonus Ledger (статус: Incoming)
  │
  ├─ Обновление права на бонус доставки:
  │   └─ После полной оплаты → бонус доставки становится доступен
  │
  ├─ Создание партнёрской выплаты (если партнёрский заказ):
  │   ├─ Классический: после полной оплаты + период ожидания → Expense "Оплатить партнёру"
  │   └─ По подписке: ежемесячно после завершения фазы разработки
  │
  └─ Переход Deal → Order (при первой оплате в CRM)
```

#### При оплате Subscription Invoice (подписка)

```
Payment создан
  ├─ Обновление статуса ячейки в Subscription Grid (зелёный = оплачен)
  ├─ Создание партнёрской выплаты (если партнёрская подписка)
  └─ Обновление MRR-метрик
```

#### При оплате Domain/Service Invoice (pass-through)

```
Payment создан
  ├─ Авто-задача: "Купить домен / продлить сервис"
  │   ├─ Если аккаунт у провайдера уже есть → задача "Купить домен"
  │   └─ Если аккаунта нет → задача "Создать аккаунт + Купить домен"
  ├─ Создание/обновление записи Domain с датой истечения
  └─ Сохранение credentials в модуле Credentials
```

## Связь Invoice → Payment

Каждый платёж привязан ровно к одному счёту. Один счёт — один платёж. Частичные оплаты не предусмотрены на текущем этапе: счёт считается оплаченным только при получении полной суммы.

```
Order ──┬── Invoice (предоплата 50%) ── Payment
        └── Invoice (остаток 50%)    ── Payment

Subscription ── Invoice (месяц 1) ── Payment
             ── Invoice (месяц 2) ── Payment
             ── Invoice (месяц 3) ── ...
```

## Уведомления

| Событие | Получатель | Канал |
|---------|-----------|-------|
| Счёт создан → переход в "Create in Gov System" | Бухгалтер | Внутреннее + Email |
| Счёт готов в гос. системе | Клиент | WhatsApp |
| Счёт просрочен (1-е напоминание) | Клиент | WhatsApp |
| Счёт просрочен (2-е напоминание, +2 дня) | Клиент | WhatsApp |
| Счёт оплачен | Продажник, менеджер проекта | Внутреннее |
| Все счета заказа оплачены | Продажник, менеджер, финдиректор | Внутреннее |

## Доска счетов (Invoice Board)

Kanban-доска с колонками по стадиям. Финансовый директор работает с ней ежедневно:

- **Фильтры**: по типу счёта, по проекту, по клиенту, по дате, по статусу Tax/Tax-Free
- **Сортировка**: по дате, по сумме, по просрочке
- **Массовые действия**: «Run Automation» для отправки напоминаний
- **Быстрый поиск**: по номеру счёта, имени клиента, сумме
- **Цветовая индикация**: красный = просрочен, жёлтый = скоро срок, зелёный = оплачен

## Метрики и аналитика

| Метрика | Описание |
|---------|----------|
| Total Invoiced | Сумма всех выставленных счетов за период |
| Total Collected | Сумма всех полученных платежей |
| Outstanding | Сумма неоплаченных счетов |
| Overdue Amount | Сумма просроченных счетов |
| DSO (Days Sales Outstanding) | Среднее время оплаты клиентами |
| Collection Rate | % оплаченных счетов от выставленных |
