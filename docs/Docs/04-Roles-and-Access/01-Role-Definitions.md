# Определение ролей

> NBOS Platform — ролевая модель компании Neetrino

## Обзор

В NBOS каждый сотрудник имеет одну или несколько ролей, определяющих его зону ответственности и уровень доступа к модулям платформы. Роли спроектированы по принципу **минимально необходимого доступа**: каждый видит только то, что нужно для выполнения его работы.

---

## 1. CEO / Founder (Генеральный директор)

**Назначение:** Стратегическое управление компанией, финальные решения, полный контроль.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Определение стратегии, утверждение крупных сделок, разрешение спорных ситуаций, управление партнёрской сетью, контроль P&L |
| **KPI** | Общая выручка, чистая прибыль, количество активных проектов, retention клиентов, рост команды |
| **Видит** | Всё — все модули, все данные, все пароли, все финансы, все проекты |
| **Не видит** | Ограничений нет |

**Особенности:**
- Единственная роль с полным доступом к секретным паролям и учётным данным компании
- Может делать бэкап всех паролей (скачивание зашифрованного архива)
- Принимает финальное решение по отказу клиенту в обслуживании (Maintenance)
- Назначает и меняет роли сотрудников

---

## 2. Seller (Продавец / Менеджер по продажам)

**Назначение:** Ведение клиента от первого контакта до подписания договора и получения первой оплаты.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Обработка лидов, ведение сделок по pipeline, подготовка коммерческих предложений, создание Invoice на предоплату, оформление Extension-сделок |
| **KPI** | Количество закрытых сделок, конверсия Lead→Deal→Won, средний чек, срок закрытия сделки, объём продаж за период |
| **Видит** | CRM (лиды, сделки), Заказы (суммы invoice и факт оплаты), Контакты, Компании, Календарь (свои встречи), Мессенджер (клиентские каналы) |
| **Не видит** | Расходы компании, зарплаты, бонусы других, пароли/credentials, Scrum-доску и задачи разработки, внутренние проектные чаты |

**Бонусная система:**
- 5–10% от суммы закрытой сделки (определяется CEO для каждой сделки)
- Бонус начисляется после полной оплаты заказа (Classic) или после сдачи проекта (Subscription)

## 3. PM (Project Manager / Проект-менеджер)

**Назначение:** Управление доставкой проекта — от получения заказа до сдачи клиенту и перехода на Maintenance.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Приёмка проекта от Seller (Handoff), планирование спринтов, управление задачами, контроль дедлайнов, коммуникация с клиентом, контроль чек-листов качества, тriage обращений поддержки |
| **KPI** | Количество проектов в срок, среднее отклонение от дедлайна, количество багов после сдачи, удовлетворённость клиента, загрузка команды |
| **Видит** | Projects Hub (все данные проекта), задачи, Scrum/Kanban, пароли проекта (назначенные команде), Drive проекта, проектные чаты, Support Tickets, календарь, финансовую сводку проекта (суммы заказа, статус оплат) |
| **Не видит** | Финансы уровня компании (P&L, расходы, зарплаты других), секретные пароли компании, CRM-сделки (кроме привязанных к его проектам) |

**Бонусная система:**
- Процент от проекта в зависимости от типа: WL — 7%, Mix — 10%, Custom Code — 15%
- Начисляется после сдачи проекта клиенту

**Временная функция:** Пока нет отдельного специалиста поддержки, PM выполняет triage Support Tickets.

---

## 4. Developer (Разработчик)

**Назначение:** Выполнение задач по разработке продуктов в рамках проектов.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Выполнение задач из Scrum-доски, написание кода, code review, тестирование, деплой, заполнение технических данных (env, credentials) |
| **KPI** | Количество закрытых задач, story points за спринт, количество багов в production, соблюдение оценок по времени |
| **Видит** | Назначенные задачи, Scrum/Kanban-доску проекта, пароли проекта (назначенные ему), Drive проекта, проектные чаты, свою доску бонусов |
| **Не видит** | Финансовые данные (суммы сделок, оплаты, расходы, зарплаты), CRM (лиды, сделки), заказы, пароли других проектов, секретные пароли |

**Бонусная система:**
- Процент от проекта: WL — 7%, Mix — 10%, Custom Code — 15%
- Начисляется после сдачи проекта (Classic — после полной оплаты, Subscription — после 4-го месяца оплаты)

---

## 5. Junior Developer (Младший разработчик)

**Назначение:** Выполнение назначенных задач под контролем старшего разработчика или PM.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Выполнение конкретных назначенных задач, работа по инструкциям и чек-листам |
| **KPI** | Количество закрытых задач, качество выполнения, скорость обучения |
| **Видит** | Только назначенные ему задачи, только явно предоставленные пароли, ограниченный Drive проекта |
| **Не видит** | Полную Scrum-доску (только свои задачи), финансы, CRM, заказы, пароли проекта (кроме явно выданных), секретные пароли |

**Отличие от Developer:** Минимальный доступ — видит только то, что ему явно назначили. Не видит общую картину проекта.

---

## 6. Designer (Дизайнер)

**Назначение:** Выполнение задач по дизайну — UI/UX, графика, брендинг в рамках проекта.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Разработка дизайна по ТЗ, подготовка макетов, работа с файлами проекта, участие в ревью |
| **KPI** | Количество закрытых задач, соблюдение дедлайнов, количество итераций правок |
| **Видит** | Назначенные задачи, Scrum/Kanban-доску, Drive проекта (файлы, ассеты), проектные чаты, свою доску бонусов |
| **Не видит** | Финансовые данные, CRM, заказы, пароли (кроме явно назначенных, например Figma), секретные пароли |

---

## 7. QA (Тестировщик)

**Назначение:** Контроль качества — проверка задач, написание тест-кейсов, обработка багов.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Тестирование выполненных задач, ведение чек-листов качества, регрессионное тестирование, обработка Support Tickets (баги) |
| **KPI** | Количество найденных/пропущенных багов, покрытие тестами, время на тестирование |
| **Видит** | Задачи (назначенные + тестируемые), Support Tickets, чек-листы, Drive проекта, проектные чаты |
| **Не видит** | Финансовые данные, CRM, заказы, пароли (кроме тестовых аккаунтов) |

---

## 8. Tech Specialist / Ops (Технический специалист)

**Назначение:** Управление технической инфраструктурой — домены, хостинг, серверы, сервисы, аккаунты.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Покупка и продление доменов, настройка хостинга/серверов, создание аккаунтов в сервисах, управление credentials, деплой, мониторинг |
| **KPI** | Время реакции на инфраструктурные задачи, uptime, количество инцидентов |
| **Видит** | Все проектные пароли/credentials, доски Credentials (Project, Department), задачи (инфраструктурные), информацию по доменам и хостингу, расходы на домены/хостинг |
| **Не видит** | Финансы компании (кроме инфраструктурных расходов), CRM, сделки, заказы, зарплаты, бонусы, секретные пароли (только по явному назначению CEO) |

---

## 9. Finance Director (Финансовый директор)

**Назначение:** Полный контроль финансовой стороны бизнеса — от выставления счетов до P&L.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Управление счетами (Invoice), контроль оплат, ведение расходов (плановых и внеплановых), подписочная сетка, начисление бонусов и зарплат, автоматизация финансовых процессов, работа с госсервисами для выставления счетов |
| **KPI** | Процент вовремя оплаченных счетов, просроченная дебиторская задолженность, точность финансового плана, время закрытия месяца |
| **Видит** | Заказы, Invoices, Payments, Subscriptions Grid, Expenses, Salary Board, Bonus Ledger, P&L, финансовые дашборды, контакты и компании (для выставления счетов) |
| **Не видит** | Scrum/Kanban-доски, задачи разработки, пароли/credentials, технические детали проектов |

**Ежедневный процесс:**
1. Проверка банковского счёта → отметка оплаченных Invoice
2. Запуск автоматизации → отправка напоминаний неоплатившим клиентам
3. Контроль просроченных платежей → эскалация

---

## 10. Marketing (Маркетолог)

**Назначение:** Привлечение лидов, аналитика маркетинговых каналов, оценка ROI рекламы.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Управление рекламными кампаниями, отслеживание источников лидов, маркетинговая аналитика, работа с верхней частью воронки CRM |
| **KPI** | Количество лидов по каналам, стоимость лида (CPL), конверсия Lead→SQL, ROI рекламных каналов |
| **Видит** | CRM Leads (чтение + аналитика), источники лидов, маркетинговые дашборды, ограниченный доступ к сделкам (только статусы и суммы для расчёта ROI) |
| **Не видит** | Полные детали сделок, заказы, расходы компании, задачи, пароли, проекты |

---

## 11. Head of Sales (Руководитель отдела продаж)

**Назначение:** Управление командой продаж, контроль pipeline, стратегия продаж.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Контроль работы продавцов, анализ воронки, прогнозирование продаж, обучение команды, эскалация сложных сделок |
| **KPI** | Общий объём продаж отдела, конверсия по стадиям, средний цикл сделки, выполнение плана продаж |
| **Видит** | Всё, что видит Seller + аналитика по всей команде, performance отдельных продавцов, дашборды отдела продаж |
| **Не видит** | Расходы компании, зарплаты, задачи разработки, пароли, внутренние проектные процессы |

---

## 12. Head of Delivery (Руководитель отдела доставки)

**Назначение:** Контроль доставки всех проектов, управление ресурсами и качеством.

| Параметр | Описание |
|---|---|
| **Ключевые обязанности** | Контроль всех проектов, распределение нагрузки между PM и разработчиками, мониторинг дедлайнов, анализ ёмкости команды, стандарты качества |
| **KPI** | Процент проектов в срок, загрузка команды, средний time-to-delivery, клиентская удовлетворённость, количество эскалаций |
| **Видит** | Всё, что видит PM + все проекты компании, capacity planning, delivery дашборды, производительность PM и разработчиков |
| **Не видит** | Детальные финансы компании (P&L, расходы), секретные пароли, детали CRM-сделок (кроме привязанных к проектам) |
