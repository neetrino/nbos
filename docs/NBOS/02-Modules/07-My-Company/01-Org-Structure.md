# Организационная структура

> NBOS Platform — модуль My Company: структура компании на базе EOS Accountability Chart

## Назначение

Организационная структура в NBOS описывает **кто за что отвечает** в компании Neetrino. Модель построена на базе **EOS Accountability Chart** — не просто органиграмма «кто кому подчиняется», а карта ответственности: каждая функция имеет владельца (Owner), который лично отвечает за результат.

**Ключевой принцип:** Accountability Chart описывает **функции**, а не людей. Один человек может занимать несколько функций (особенно на раннем этапе), но у каждой функции есть ровно один владелец.

---

## My Company как модуль

`My Company` - это центр управления внутренней структурой компании.

Внутри модуля:

| Раздел            | Смысл                                                               |
| ----------------- | ------------------------------------------------------------------- |
| `Org Structure`   | Визуальная структура компании: отделы, seats, владельцы, подчинение |
| `Team`            | Сотрудники и их карточки                                            |
| `Departments`     | Отделы компании и их владельцы                                      |
| `Roles & Seats`   | Бизнес-функции: Seller, PM, Developer, Head of Sales и т.д.         |
| `Compensation`    | Compensation profiles, bonus policies, KPI policies                 |
| `KPI / Scorecard` | KPI company / department / employee                                 |
| `SOP & Templates` | Процедуры, onboarding, offboarding, регулярные процессы             |

`My Company` не должен смешиваться с `Settings`. Settings - это системная админка платформы, а My Company - бизнес-структура компании.

## Core model / Базовая модель структуры

Правильная модель:

```text
Department -> Seat -> Seat Assignment -> Employee
```

### Department / Отдел

`Department / Отдел` - бизнес-направление компании: Sales, Marketing, Delivery, Support, Finance и т.д.

Отдел хранит:

| Поле                | Описание                              |
| ------------------- | ------------------------------------- |
| `name`              | Название отдела                       |
| `parent_id`         | Родительский отдел, если это подотдел |
| `head_seat_id`      | Seat, который отвечает за отдел       |
| `owner_employee_id` | Текущий владелец / руководитель       |
| `status`            | Active / Archived                     |
| `kpi_policy_id`     | KPI policy отдела, если есть          |
| `sop_links`         | Связанные процедуры                   |

### Seat / Место / Функция

`Seat / Место` - это функция в компании, а не человек.

Примеры:

- `Head of Sales`;
- `Seller`;
- `Project Manager`;
- `Developer`;
- `Finance Director`;
- `Support Engineer`.

Seat хранит:

| Поле                      | Описание                                 |
| ------------------------- | ---------------------------------------- |
| `department_id`           | В каком отделе находится seat            |
| `title`                   | Название функции                         |
| `accountability`          | За что seat отвечает                     |
| `required_skills`         | Навыки / требования                      |
| `level_range`             | Junior / Middle / Senior / Lead / Head   |
| `default_kpi_policy`      | KPI policy по умолчанию                  |
| `default_bonus_policy`    | Bonus policy по умолчанию                |
| `default_permission_role` | Техническая permission role по умолчанию |
| `status`                  | Active / Vacant / Archived               |

### Seat Assignment / Назначение на место

`Seat Assignment / Назначение на место` связывает сотрудника с seat.

Один сотрудник может занимать несколько seats. Один seat может быть vacant или временно совмещаться.

Seat Assignment хранит:

| Поле             | Описание                                        |
| ---------------- | ----------------------------------------------- |
| `seat_id`        | Какую функцию занимает сотрудник                |
| `employee_id`    | Кто занимает                                    |
| `allocation_pct` | Сколько времени / ответственности занимает seat |
| `is_primary`     | Основная функция сотрудника или дополнительная  |
| `start_date`     | С какой даты действует                          |
| `end_date`       | Когда закончилась                               |
| `status`         | Active / Temporary / Ended                      |

Пример:

```text
Employee: CEO
Assignments:
- CEO / Visionary - 60%
- Finance Director - 25%
- Operations Manager - 15%
```

Так система честно показывает текущую реальность компании, но структура всё равно проектируется под целевое состояние.

---

## Текущая реальность vs Целевая структура

### Текущее состояние (15–20 человек)

На данном этапе многие роли совмещены:

| Сотрудник | Совмещаемые функции                                |
| --------- | -------------------------------------------------- |
| CEO       | Executive + Finance Director + Operations          |
| PM #1     | Project Manager + частично Developer (code review) |
| PM #2     | Project Manager + QA                               |
| Seller #1 | Sales + частично Marketing                         |
| Developer | Development + DevOps                               |

Это нормально для компании из 15–20 человек. Но **структура проектируется на целевое состояние** (30–50 человек), а текущие совмещения фиксируются как временные.

### Принцип роста

```
Текущее: 1 человек = 2–3 функции (совмещение)
Целевое: 1 функция = 1 человек (полная ответственность)
```

При найме нового сотрудника CEO решает, какую функцию «отделить» от текущего совместителя и передать новому человеку.

## L1 Департаменты (уровень 1)

Компания Neetrino разделена на **8 департаментов первого уровня**. Каждый департамент имеет **Head** (руководителя), который подчиняется CEO.

```
                          ┌──────────┐
                          │   CEO    │
                          │ Visionary│
                          └────┬─────┘
                               │
            ┌──────────────────┼──────────────────┐
            │                  │                  │
     ┌──────┴──────┐    ┌─────┴──────┐    ┌──────┴──────┐
     │  Marketing  │    │   Sales    │    │  Delivery   │
     │  (Маркетинг)│    │ (Продажи) │    │ (Доставка)  │
     └─────────────┘    └────────────┘    └─────────────┘
            │                  │                  │
     ┌──────┴──────┐    ┌─────┴──────┐    ┌──────┴──────┐
     │   Support   │    │ HR/Training│    │Finance/Legal│
     │  (Поддержка)│    │ (Кадры)    │    │ (Финансы)   │
     └─────────────┘    └────────────┘    └─────────────┘
            │                  │
     ┌──────┴──────┐    ┌─────┴──────┐
     │ Operations  │    │ Executive  │
     │ (Операции)  │    │(Руководство│
     └─────────────┘    └────────────┘
```

## Описание каждого департамента

### 1. Executive (Руководство)

| Параметр            | Значение                                                 |
| ------------------- | -------------------------------------------------------- |
| **Head**            | CEO (Visionary / Integrator)                             |
| **Ответственность** | Стратегия, видение, ключевые решения, L10 meeting, Rocks |
| **KPI**             | Revenue growth, Gross margin, Rocks completion           |

**Seats внутри:**

| Seat             | Описание                                              | Текущий статус  |
| ---------------- | ----------------------------------------------------- | --------------- |
| CEO / Visionary  | Стратегическое направление, партнёрства, найм C-level | Активна         |
| Integrator (COO) | Операционное управление, координация департаментов    | Совмещена с CEO |

### 2. Marketing (Маркетинг)

| Параметр            | Значение                                      |
| ------------------- | --------------------------------------------- |
| **Head**            | Head of Marketing                             |
| **Ответственность** | Генерация лидов, бренд, контент, реклама      |
| **KPI**             | Leads/week, MQL/week, CPL, MQL→SQL conversion |

**Seats внутри:**

| Seat              | Описание                                                   |
| ----------------- | ---------------------------------------------------------- |
| Head of Marketing | Стратегия маркетинга, бюджет, аналитика каналов            |
| Content Manager   | Контент для соцсетей, блог, кейсы                          |
| Ads Specialist    | Настройка рекламных кампаний (Instagram, Facebook, Google) |
| SMM Manager       | Ведение социальных сетей                                   |

### 3. Sales (Продажи)

| Параметр            | Значение                                             |
| ------------------- | ---------------------------------------------------- |
| **Head**            | Head of Sales                                        |
| **Ответственность** | Конвертация SQL в оплаченные заказы                  |
| **KPI**             | Close rate, Revenue/week, Avg deal size, Sales cycle |

**Seats внутри:**

| Seat                   | Описание                                                |
| ---------------------- | ------------------------------------------------------- |
| Head of Sales          | Управление воронкой, обучение продавцов, прогноз продаж |
| Seller (Sales Manager) | Ведение сделок от SQL до Deal Won                       |
| Presale Specialist     | Подготовка коммерческих предложений, оценка проектов    |

### 4. Delivery (Доставка проектов)

| Параметр            | Значение                                                        |
| ------------------- | --------------------------------------------------------------- |
| **Head**            | Head of Delivery                                                |
| **Ответственность** | Разработка и сдача проектов в срок и бюджет                     |
| **KPI**             | On-time delivery %, Plan vs Actual variance, Bugs to production |

**Seats внутри:**

| Seat                                  | Описание                                                           |
| ------------------------------------- | ------------------------------------------------------------------ |
| Head of Delivery                      | Ресурсное планирование, качество доставки                          |
| Project Manager                       | Управление конкретными проектами, спринты, коммуникация с клиентом |
| Developer (Junior/Middle/Senior/Lead) | Разработка продуктов                                               |
| Designer (UI/UX)                      | Дизайн интерфейсов                                                 |
| QA Engineer                           | Тестирование, контроль качества                                    |

### 5. Support (Техническая поддержка / Maintenance)

| Параметр            | Значение                                                   |
| ------------------- | ---------------------------------------------------------- |
| **Head**            | Head of Support                                            |
| **Ответственность** | Техобслуживание сданных проектов, решение проблем клиентов |
| **KPI**             | SLA compliance %, First response time, Reopen rate         |

**Seats внутри:**

| Seat             | Описание                                    |
| ---------------- | ------------------------------------------- |
| Head of Support  | Приоритизация обращений, SLA, эскалация     |
| Support Engineer | Решение технических проблем клиентов        |
| DevOps / Infra   | Мониторинг серверов, деплой, инфраструктура |

> **Критическое разделение:** Delivery (проекты) ≠ Support (техобслуживание). У них разные KPI, разные процессы и разные приоритеты. Один проект может одновременно быть в Delivery (доработка) и Support (maintenance), но за каждое направление отвечают разные люди.

### 6. HR / Training (Кадры и обучение)

| Параметр            | Значение                                              |
| ------------------- | ----------------------------------------------------- |
| **Head**            | HR Director                                           |
| **Ответственность** | Найм, адаптация, обучение, корпоративная культура     |
| **KPI**             | Time-to-hire, Employee retention, Training completion |

**Seats внутри:**

| Seat                 | Описание                                          |
| -------------------- | ------------------------------------------------- |
| HR Director          | Стратегия найма, политики, корпоративная культура |
| HR Manager           | Рекрутинг, онбординг, оффбординг                  |
| Training Coordinator | Программы обучения, менторство                    |

### 7. Finance / Legal (Финансы и юридические вопросы)

| Параметр            | Значение                                               |
| ------------------- | ------------------------------------------------------ |
| **Head**            | Finance Director                                       |
| **Ответственность** | Контроль денежных потоков, бюджет, юридические вопросы |
| **KPI**             | Cash balance, Overdue invoices, Gross margin %, DSO    |

**Seats внутри:**

| Seat                   | Описание                                                  |
| ---------------------- | --------------------------------------------------------- |
| Finance Director       | Управление финансами, P&L, бюджетирование                 |
| Accountant (Бухгалтер) | Счета в гос. системе, налоги, отчётность                  |
| Legal Advisor          | Договоры, юридическая экспертиза (может быть на аутсорсе) |

### 8. Operations / Admin (Операции и администрирование)

| Параметр            | Значение                                        |
| ------------------- | ----------------------------------------------- |
| **Head**            | Operations Manager                              |
| **Ответственность** | Внутренние процессы, инструменты, SOP, офис     |
| **KPI**             | SOP coverage %, Tool uptime, Process efficiency |

**Seats внутри:**

| Seat                 | Описание                                           |
| -------------------- | -------------------------------------------------- |
| Operations Manager   | Координация внутренних процессов, SOP              |
| System Administrator | Управление инструментами, доступами, лицензиями    |
| Office Manager       | Организация офисного пространства (если применимо) |

## Принцип Accountability

Каждый департамент и роль следуют правилу EOS:

| Принцип                         | Описание                                                                          |
| ------------------------------- | --------------------------------------------------------------------------------- |
| **1 Owner**                     | У каждой функции ровно один владелец. Нельзя «два человека отвечают за маркетинг» |
| **Accountability ≠ Management** | Owner отвечает за результат, но не обязательно лично выполняет всю работу         |
| **Seat = Function**             | Место в структуре = функция. Человек «сидит на 1–3 местах»                        |
| **Right Person, Right Seat**    | Человек должен подходить по ценностям (Right Person) и компетенциям (Right Seat)  |
| **Vacant seat is visible**      | Если функция нужна, но человека нет, seat остаётся видимым как вакансия           |
| **Assignment has history**      | Назначения на seats не перезаписываются, а хранят историю                         |

## Реализация в NBOS Platform

### Визуализация оргструктуры

Основной экран `Org Structure` должен быть визуальным canvas, похожим по удобству на Bitrix24:

- большое рабочее полотно;
- карточки отделов;
- линии подчинения;
- zoom in / zoom out;
- search по отделу, seat или сотруднику;
- collapse / expand подотделов;
- mini-map или быстрый переход к выбранному отделу;
- кнопка `Add` для добавления department / seat;
- правый drawer для детальной информации.

### Department Card / Карточка отдела на оргкарте

Снаружи карточка отдела показывает:

| Элемент         | Смысл                                |
| --------------- | ------------------------------------ |
| Название отдела | `Sales`, `Delivery`, `Finance`       |
| Head / Owner    | Кто сейчас отвечает за отдел         |
| Employees count | Количество сотрудников               |
| Seats count     | Сколько seats внутри отдела          |
| Vacant seats    | Сколько функций без владельца        |
| KPI status      | Green / Yellow / Red по ключевым KPI |
| Children count  | Сколько подотделов                   |

Цветовая логика:

| Цвет / индикатор | Смысл                                    |
| ---------------- | ---------------------------------------- |
| Green            | Все ключевые seats закрыты               |
| Yellow           | Есть совмещения или временные назначения |
| Red              | Есть критичные vacant seats              |
| Blue outline     | Выбранный отдел                          |

### Department Drawer / Детальная панель отдела

Клик по карточке отдела открывает drawer.

Внутри:

- owner / head;
- parent department;
- subdepartments;
- seats;
- employees;
- vacant seats;
- KPI отдела;
- payroll budget summary;
- linked SOP;
- audit history.

### Seat Detail / Детали места

Клик по seat показывает:

- accountability;
- current assignee;
- backup / deputy, если есть;
- allocation percent;
- required skills;
- level range;
- KPI policy;
- bonus policy;
- permission role mapping;
- assignment history.

### Views / Виды

У `Org Structure` должно быть несколько видов:

| Вид                     | Для чего нужен                                     |
| ----------------------- | -------------------------------------------------- |
| `Org Chart`             | Визуальная структура как canvas                    |
| `Departments List`      | Таблица отделов                                    |
| `Seats List`            | Все seats, фильтр по vacant / assigned / temporary |
| `Assignments`           | Кто какие seats занимает                           |
| `Vacancies`             | Открытые или незакрытые функции                    |
| `Accountability Matrix` | Кто за что отвечает                                |

### Управление структурой

- Создание / редактирование департаментов (только CEO / HR Director)
- Создание seats внутри департаментов
- Назначение сотрудников на seats через Seat Assignment
- Поддержка temporary assignment и совмещений
- Отображение vacant seats
- История изменений структуры (Audit Log)

### Связи с другими модулями

```
Org Structure → RBAC (права доступа зависят от роли и департамента)
Org Structure → Team/Employees (сотрудники привязаны к ролям)
Org Structure → KPI Scorecard (KPI привязаны к департаментам)
Org Structure → Projects Hub (ресурсное планирование по ролям)
Org Structure → Finance (зарплатные бюджеты по департаментам)
```

## Дорожная карта роста

### Этап 1: 15–20 человек (текущий)

- CEO совмещает Executive + Finance + Operations
- 1–2 Seller совмещают Sales + частично Marketing
- PM совмещают управление + частично QA/Dev
- Нет выделенного Support — PM занимается maintenance

### Этап 2: 20–30 человек

- Выделяется Finance Director (отделяется от CEO)
- Выделяется Head of Support (отделяется от PM)
- Появляется выделенный QA Engineer
- HR-функция отделяется от CEO

### Этап 3: 30–50 человек

- Полная структура: каждый департамент имеет Head
- Появляются Lead-позиции внутри Delivery
- Выделенный Operations Manager
- Marketing-команда из 2–3 человек
