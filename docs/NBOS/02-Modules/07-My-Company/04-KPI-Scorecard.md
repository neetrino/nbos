# KPI и Scorecard

> NBOS Platform — модуль My Company: система ключевых показателей и еженедельное отслеживание по методологии EOS

## Назначение

Scorecard — это инструмент из методологии **EOS (Entrepreneurial Operating System)**, который позволяет отслеживать **5–15 ключевых чисел** компании на еженедельной основе. Вместо того чтобы ждать конца месяца/квартала для анализа — команда видит тренды и проблемы **каждую неделю**.

**Ключевой принцип:** Scorecard показывает не «что произошло», а «куда мы движемся». Красный показатель на 3-й неделе — сигнал действовать сейчас, а не «удивляться» в конце квартала.

## Company Scorecard (Scorecard компании)

Обзорная таблица ключевых метрик, которая рассматривается на еженедельном **L10 Meeting** (Level 10 Meeting по EOS).

### Формат

Каждая строка Scorecard содержит:

| Поле | Описание |
|------|----------|
| **Metric** | Название метрики |
| **Owner** | Ответственный (владелец показателя) |
| **Target** | Целевое значение на неделю/месяц |
| **W1–W13** | Фактическое значение за каждую неделю квартала |
| **Status** | 🟢 Green (≥ Target) / 🔴 Red (< Target) |
| **Notes** | Комментарий владельца (особенно для красных метрик) |

### Правило L10 Meeting

На еженедельном L10 Meeting:
1. **Scorecard ревью** — 5 минут: каждый Owner называет своё число
2. **Красные метрики** — автоматически попадают в IDS-список (Identify, Discuss, Solve)
3. **IDS** — команда обсуждает корневую причину и определяет действие
4. **To-do** — из IDS рождаются задачи со сроком «до следующего L10»

## KPI и бонусная система

KPI напрямую влияет на бонусы. Для продажников действует **KPI Gate** — пороговый показатель, определяющий процент выплаты бонуса.

### KPI Gate для Seller

```
KPI Score ≥ 70%  →  100% бонуса выплачивается
KPI Score 50–69% →   50% бонуса выплачивается
KPI Score < 50%  →    0% бонуса (бонус сгорает)
```

### Расчёт KPI Score

KPI Score — это взвешенная оценка выполнения индивидуальных KPI:

```
KPI Score = Σ (вес метрики × % выполнения метрики)
```

Пример для Seller:

| Метрика | Вес | Target | Actual | % выполнения | Взвешенный |
|---------|-----|--------|--------|--------------|------------|
| Deals closed | 30% | 4 | 3 | 75% | 22.5% |
| Revenue generated | 40% | 1,200,000 | 1,000,000 | 83% | 33.2% |
| Conversion rate | 20% | 25% | 20% | 80% | 16.0% |
| Follow-up compliance | 10% | 100% | 90% | 90% | 9.0% |
| **ИТОГО** | **100%** | | | | **80.7%** |

KPI Score = 80.7% → ≥ 70% → **100% бонуса выплачивается**.

### Применение KPI Gate

```
Seller Петров:
  ├─ Заработанные бонусы за месяц: 120,000 ₽
  ├─ KPI Score: 80.7%
  ├─ KPI Gate: ≥ 70% → 100%
  └─ К выплате: 120,000 × 100% = 120,000 ₽

Seller Сидоров:
  ├─ Заработанные бонусы за месяц: 95,000 ₽
  ├─ KPI Score: 55%
  ├─ KPI Gate: 50–69% → 50%
  └─ К выплате: 95,000 × 50% = 47,500 ₽

Seller Козлов:
  ├─ Заработанные бонусы за месяц: 60,000 ₽
  ├─ KPI Score: 42%
  ├─ KPI Gate: < 50% → 0%
  └─ К выплате: 0 ₽ (бонус сгорает)
```

## Настройка метрик

### Создание новой метрики

| Поле | Описание |
|------|----------|
| `metric_name` | Название метрики |
| `owner` | Ответственный сотрудник |
| `department` | Департамент |
| `scope` | Company / Department / Personal |
| `frequency` | Weekly / Monthly / Quarterly |
| `target_value` | Целевое значение |
| `target_direction` | ≥ (чем больше, тем лучше) / ≤ (чем меньше, тем лучше) |
| `data_source` | Ручной ввод / Автоматический (из модуля) |
| `weight` | Вес в KPI Score (для персональных KPI) |

### Автоматический сбор данных

Часть метрик собирается автоматически из модулей платформы:

| Метрика | Источник данных |
|---------|-----------------|
| Leads/week | CRM → Lead Pipeline (новые лиды за период) |
| SQL/week | CRM → Lead Pipeline (конвертированные в SQL) |
| Revenue/week | Finance → Payments (оплаченные Invoice) |
| Deals closed | CRM → Deal Pipeline (статус Deal Won) |
| Tasks completed | Tasks → Статус Done за период |
| On-time % | Tasks → Сравнение дедлайна и даты выполнения |
| Tickets closed | Support → Закрытые обращения |
| SLA compliance | Support → Время обработки vs SLA |
| Cash balance | Finance → Текущий баланс |
| MRR | Finance → Subscriptions (активные подписки) |
| Overdue invoices | Finance → Invoices (просроченные) |
| DSO | Finance → Invoices (средний срок оплаты) |

Метрики, которые невозможно собрать автоматически (например, Client satisfaction), вводятся вручную Owner-ом.

## Rocks и Scorecard

В методологии EOS **Rocks** — это квартальные приоритеты (3–7 ключевых целей на квартал). Scorecard и Rocks связаны:

- Rocks определяют стратегическое направление → Scorecard измеряет операционное исполнение
- Если Rock = «Увеличить конверсию до 30%» → метрика Close Rate в Scorecard отслеживает прогресс
- На L10 Meeting обсуждаются и Rocks (On Track / Off Track), и Scorecard (Green / Red)

## Связи с другими модулями

```
KPI Scorecard → CRM (данные по лидам, сделкам, конверсии)
KPI Scorecard → Finance (revenue, cash, margins, DSO)
KPI Scorecard → Tasks (on-time %, tasks completed)
KPI Scorecard → Support (SLA, tickets, response time)
KPI Scorecard → Projects Hub (on-time delivery, variance)
KPI Scorecard → Bonus & Payroll (KPI Gate для бонусов)
KPI Scorecard → Team/Employees (персональные KPI в профиле)
KPI Scorecard → Dashboards (виджеты для L10 Meeting)
```
