# Команда и сотрудники

> NBOS Platform — модуль My Company: управление персоналом, профили, графики, онбординг

## Назначение

Раздел «Команда и сотрудники» — это **внутренний справочник компании** и инструмент управления персоналом. Здесь хранятся профили сотрудников, их роли, уровни, рабочие графики и вся информация, необходимая для ресурсного планирования, расчёта зарплат и контроля доступов.

**Ключевой принцип:** Профиль сотрудника — это единая точка правды. Все модули платформы (Projects, Tasks, Finance, RBAC) ссылаются на один и тот же профиль.

## Профиль сотрудника (Employee Profile)

### Основные поля

| Поле            | Тип      | Описание                                        | Обязательное |
| --------------- | -------- | ----------------------------------------------- | ------------ |
| `employee_id`   | Auto     | Уникальный идентификатор                        | Да           |
| `full_name`     | Text     | ФИО сотрудника                                  | Да           |
| `avatar`        | Image    | Фотография                                      | Нет          |
| `department`    | Relation | Департамент (из Org Structure)                  | Да           |
| `role`          | Select   | Роль в компании (Developer, PM, Seller...)      | Да           |
| `level`         | Select   | Уровень: Junior / Middle / Senior / Lead / Head | Да           |
| `base_salary`   | Currency | Базовая зарплата (видна только Finance + CEO)   | Да           |
| `hire_date`     | Date     | Дата приёма на работу                           | Да           |
| `status`        | Select   | Active / On Leave / Probation / Terminated      | Да           |
| `work_schedule` | Schedule | Рабочий график (см. ниже)                       | Да           |
| `telegram`      | Text     | Telegram username                               | Нет          |
| `email`         | Email    | Рабочая электронная почта                       | Да           |
| `phone`         | Phone    | Телефонный номер                                | Нет          |
| `birthday`      | Date     | Дата рождения                                   | Нет          |
| `notes`         | Text     | Заметки (видны HR + CEO)                        | Нет          |

### Статусы сотрудника

| Статус         | Описание                                                  |
| -------------- | --------------------------------------------------------- |
| **Active**     | Работает, выполняет задачи                                |
| **Probation**  | На испытательном сроке (обычно 1–3 месяца)                |
| **On Leave**   | В отпуске или на больничном                               |
| **Terminated** | Уволен. Профиль сохраняется для истории, доступы отозваны |

## Уровни сотрудников (Employee Levels)

Уровень влияет на несколько аспектов работы в платформе:

| Уровень    | Доступы                                       | Бонусные ставки    | Ответственность                     |
| ---------- | --------------------------------------------- | ------------------ | ----------------------------------- |
| **Junior** | Минимальные: только свои задачи, свой проект  | Базовая ставка     | Выполнение задач под контролем      |
| **Middle** | Расширенные: все задачи проекта, чаты         | Стандартная ставка | Самостоятельное выполнение задач    |
| **Senior** | Полные в рамках роли: code review, менторство | Повышенная ставка  | Ответственность за качество         |
| **Lead**   | Управление командой, планирование             | Lead-ставка        | Ответственность за команду          |
| **Head**   | Полный доступ к департаменту                  | Head-ставка        | Ответственность за весь департамент |

### Влияние уровня на систему

```
Level → RBAC (модульный и проектный доступ)
Level → Bonus rates (процент бонуса может зависеть от уровня)
Level → Task assignment (Junior не назначается на критичные задачи)
Level → Project allocation (ресурсное планирование с учётом уровня)
```

## Карточка сотрудника (Employee Card)

При открытии профиля сотрудника отображаются следующие секции:

### Секция 1: Личная информация

- ФИО, фото, контакты (Telegram, email, phone)
- Департамент, роль, уровень
- Дата найма, стаж в компании
- Текущий статус

### Секция 2: Проекты

- Список проектов, в которых участвует сотрудник
- Роль в каждом проекте (Developer, PM, Seller, Designer, QA)
- Статус проекта (Development, Maintenance, Done)
- Загрузка: сколько часов/процентов выделено на каждый проект

### Секция 3: Задачи

- Активные задачи сотрудника (из модуля Tasks)
- Фильтры: по проекту, по статусу, по приоритету
- Статистика: задач в работе / выполнено за неделю / просрочено

### Секция 4: Бонусы

- Incoming бонусы (прогноз)
- Active бонусы (к выплате)
- Paid бонусы (история выплат)
- Итоговые суммы за месяц / квартал / год
- Действие **«Мои финансы» / «Открыть кошелёк»** — переход в модуль Finance: **[Employee Wallet](../04-Finance/08-Employee-Wallet.md)** (read-only витрина с разрезом по проектам и полным маппингом состояний бонуса для сотрудника)

### Секция 5: KPI

- Индивидуальные KPI сотрудника (из модуля KPI/Scorecard)
- Текущее выполнение за период
- История KPI по месяцам

### Секция 6: Рабочий график

- Текущий недельный график
- Календарь отпусков и отсутствий

## Рабочий график (Work Schedule)

Каждый сотрудник заполняет свой **недельный рабочий график**. Это критически важно для ресурсного планирования.

### Формат графика

| День        | Начало | Конец | Статус   |
| ----------- | ------ | ----- | -------- |
| Понедельник | 09:00  | 18:00 | Рабочий  |
| Вторник     | 09:00  | 18:00 | Рабочий  |
| Среда       | 10:00  | 19:00 | Рабочий  |
| Четверг     | 09:00  | 18:00 | Рабочий  |
| Пятница     | 09:00  | 17:00 | Рабочий  |
| Суббота     | —      | —     | Выходной |
| Воскресенье | —      | —     | Выходной |

### Использование графика

- **Calendar:** график сотрудника отображается в общем календаре команды
- **Resource Planning:** при назначении на проект система показывает доступные часы
- **Task Planning:** при назначении задачи с дедлайном — учёт рабочих дней
- **Absence Tracking:** отпуск/больничный — сотрудник помечает дни как «Отсутствие»

## Онбординг нового сотрудника

При создании нового сотрудника система автоматически генерирует **чеклист онбординга**:

### Чеклист онбординга

| #   | Задача                                            | Ответственный         | Дедлайн  |
| --- | ------------------------------------------------- | --------------------- | -------- |
| 1   | Создать профиль сотрудника в NBOS                 | HR                    | День 1   |
| 2   | Назначить департамент и роль                      | HR + Head of Dept     | День 1   |
| 3   | Настроить рабочий email                           | Operations / SysAdmin | День 1   |
| 4   | Создать аккаунт Telegram (если нет)               | Сотрудник             | День 1   |
| 5   | Добавить в рабочие чаты (Telegram-группы)         | Head of Dept          | День 1   |
| 6   | Выдать доступ к NBOS Platform (по роли)           | Operations / SysAdmin | День 1   |
| 7   | Выдать доступ к инструментам (Git, Figma, IDE...) | Operations / SysAdmin | День 1–2 |
| 8   | Ознакомление с SOP (стандартные процедуры)        | Head of Dept          | День 1–3 |
| 9   | Назначить ментора (для Junior/Middle)             | Head of Dept          | День 1   |
| 10  | Первое знакомство с командой                      | HR                    | День 1   |
| 11  | Назначить на первый проект                        | Head of Dept + PM     | День 2–5 |
| 12  | Заполнить рабочий график                          | Сотрудник             | День 1   |
| 13  | Пройти вводное обучение (платформа, процессы)     | Ментор / HR           | Неделя 1 |
| 14  | Первый 1-on-1 с руководителем                     | Head of Dept          | Неделя 1 |
| 15  | Проверка по итогам первой недели                  | HR + Head of Dept     | День 7   |

### Автоматизация онбординга

- Чеклист создаётся автоматически при смене статуса на **Probation** или **Active**
- Задачи распределяются ответственным лицам
- Прогресс виден в карточке сотрудника
- HR получает уведомления при просрочке пунктов

## Оффбординг сотрудника

При увольнении или завершении сотрудничества запускается **чеклист оффбординга**:

### Чеклист оффбординга

| #   | Задача                                           | Ответственный         | Дедлайн          |
| --- | ------------------------------------------------ | --------------------- | ---------------- |
| 1   | Уведомить команду и клиентов (если нужно)        | Head of Dept          | День 1           |
| 2   | Передать активные проекты другому сотруднику     | PM / Head of Dept     | Неделя 1         |
| 3   | Передать активные задачи                         | PM                    | Неделя 1         |
| 4   | Передать клиентские контакты (для Seller)        | Head of Sales         | Неделя 1         |
| 5   | Отозвать доступ к NBOS Platform                  | Operations / SysAdmin | Последний день   |
| 6   | Отозвать доступ к Git, Figma, IDE                | Operations / SysAdmin | Последний день   |
| 7   | Отозвать доступ к рабочей почте                  | Operations / SysAdmin | Последний день   |
| 8   | Удалить из рабочих Telegram-чатов                | Head of Dept          | Последний день   |
| 9   | Отозвать доступ к Credentials (пароли)           | Operations / SysAdmin | Последний день   |
| 10  | Сменить пароли, к которым был доступ (критичные) | Operations / SysAdmin | Последний день   |
| 11  | Расчёт финальной зарплаты + начисленных бонусов  | Finance Director      | Неделя 1–2       |
| 12  | Выплата финального расчёта                       | Finance Director      | По закону        |
| 13  | Архивировать профиль (статус → Terminated)       | HR                    | Последний день   |
| 14  | Exit-интервью (по желанию)                       | HR / CEO              | Последняя неделя |

### Автоматизация оффбординга

- Чеклист создаётся при смене статуса на **Terminated**
- Система автоматически формирует список всех проектов, задач и доступов сотрудника
- Finance Director получает уведомление о необходимости финального расчёта
- Audit Log фиксирует все отозванные доступы

## Права доступа к разделу Team

| Роль                 | Просмотр списка                    | Просмотр профиля                     | Редактирование      | Зарплаты            |
| -------------------- | ---------------------------------- | ------------------------------------ | ------------------- | ------------------- |
| **CEO**              | Все                                | Полный                               | Полный              | Да                  |
| **HR Director**      | Все                                | Полный                               | Полный              | Только структура ЗП |
| **Finance Director** | Все                                | Ограниченный                         | Нет                 | Да (для payroll)    |
| **Head of Dept**     | Свой департамент                   | Профили своего отдела                | Частичное           | Нет                 |
| **Сотрудник**        | Общий список (имя, роль, контакты) | Свой профиль полный, чужие — базовый | Только свой профиль | Только свою ЗП      |

## Связи с другими модулями

```
Team/Employees → RBAC (уровень и роль определяют доступы)
Team/Employees → Projects Hub (назначение на проекты)
Team/Employees → Tasks (назначение задач)
Team/Employees → Finance / Payroll (зарплата и бонусы)
Team/Employees → Calendar (рабочий график, отпуска)
Team/Employees → Credentials (доступ к паролям по роли)
Team/Employees → Messenger (участие в чатах проектов)
Team/Employees → KPI Scorecard (индивидуальные метрики)
```
