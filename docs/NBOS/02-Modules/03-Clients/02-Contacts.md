# Контакты (Contacts)

## Обзор

**Contact** (Контакт) — это физическое лицо: человек, с которым Neetrino ведёт взаимодействие. Это может быть клиент, представитель партнёра или подрядчик. Contact — это «уровень человека» в иерархии данных, в отличие от Company (уровень юрлица) и Project (уровень бизнеса).

Ключевое правило: **один Contact может иметь МНОГО проектов**. Крупный предприниматель с десятью бизнесами — это один Contact и десять Projects.

Общие границы модуля описаны в `00-Clients-Overview.md`. Важно: `Client` не является отдельной ручной сущностью. Клиентский профиль появляется автоматически, если Contact связан с Lead, Deal, Project, Company, Invoice, Subscription, Support Ticket или Client Service Record.

---

## 1. Иерархия: Contact → Company → Project

```
Contact (человек)
  │
  ├── Company A (ООО "Первый бизнес")
  │     └── Project "Brand A" (веб-сайт, приложение)
  │
  ├── Company B (ИП Иванов)
  │     ├── Project "Brand B" (интернет-магазин)
  │     └── Project "Brand C" (landing page)
  │
  └── Company C (иностранное юрлицо)
        └── Project "Brand D" (CRM-система)
```

**Contact** — это человек. **Company** — это юридическое лицо для биллинга. **Project** — это бизнес/бренд. Один человек может иметь несколько юрлиц и несколько бизнесов.

## 3. Тип контакта

`contact_type` — это общий тип контакта для фильтрации и понимания, кем человек является для Neetrino. Это не процессная роль и не источник бизнес-логики стадий.

| Тип            | Описание                        | Особенности                                                                    |
| -------------- | ------------------------------- | ------------------------------------------------------------------------------ |
| **Client**     | Заказчик услуг Neetrino         | Основной тип. Привязан к проектам как заказчик. Получает Invoice через Company |
| **Partner**    | Представитель компании-партнёра | Может быть связан с Partner-сущностью. Передаёт или получает заказы            |
| **Contractor** | Подрядчик / фрилансер           | Внешний исполнитель, привлекаемый для конкретных задач                         |
| **Other**      | Прочие                          | Консультанты, знакомые, потенциальные контакты                                 |

Правила:

- если Contact создан из CRM / Deal / Project / Company, default = `Client`;
- если Contact создан из Partners, default = `Partner`;
- если Contact создан как внешний исполнитель, default = `Contractor`;
- значение можно поменять вручную в карточке Contact;
- `contact_type` не должен блокировать или разрешать переходы Deal / Project / Invoice.

Процессный контекст определяется связями: `deal.contact`, `company.primary_contact`, `billing_contact`, `project.contacts`, `support_ticket.contact`.

## 4. Создание контакта

### Когда создаётся:

| Источник | Описание                                                        |
| -------- | --------------------------------------------------------------- |
| Из Lead  | При квалификации лида как SQL — автоматически создаётся Contact |
| Вручную  | Seller или PM создаёт Contact из раздела Clients → Contacts     |
| Из Deal  | Если при создании сделки указан новый контакт                   |

### Процесс из Lead:

```
Lead получает статус SQL (Quality Lead)
         ↓
Система проверяет: существует ли Contact с таким телефоном/email?
         ↓
┌────────┴────────┐
↓                 ↓
Найден            Не найден
↓                 ↓
Привязать         Создать новый Contact
к существующему   из данных Lead
```

### Автоматические действия при создании:

- Поля заполняются из Lead (если создан из Lead)
- Контакт автоматически привязывается к Deal
- Уведомление Seller о новом контакте

### Кто может создавать:

- **Seller** — основной создатель контактов (из лидов и сделок)
- **PM** — при необходимости (новое контактное лицо по проекту)
- **CEO** — в любом случае

## 5. Карточка Contact — что отображается

При открытии карточки контакта пользователь видит:

### 5.1. Основная информация

Все поля из таблицы в разделе 2: ФИО, телефон, email, мессенджеры, тип контакта, источник, заметки.

### 5.2. Список проектов

Все проекты, где данный контакт является основным контактом (заказчиком):

| Поле             | Описание                                             |
| ---------------- | ---------------------------------------------------- |
| Название проекта | Ссылка на Project                                    |
| Компания         | Юрлицо, привязанное к проекту                        |
| Статус           | Computed-статус (Development / Maintenance / Closed) |
| Тип              | WL / Mix / Custom Code                               |
| Продукты         | Список продуктов с мини-статусами                    |
| Дата начала      | Когда проект был создан                              |

### 5.3. Список компаний

Все Company, привязанные к этому контакту:

| Поле                | Описание                                    |
| ------------------- | ------------------------------------------- |
| Название            | Название компании / ФИО                     |
| Тип                 | Legal Entity / Sole Proprietor / Individual |
| Tax Status          | Tax / Tax-Free                              |
| Количество проектов | Сколько проектов привязано к этой Company   |

### 5.4. История сделок

Все Deal (текущие и прошлые), связанные с этим контактом:

| Поле            | Описание                                    |
| --------------- | ------------------------------------------- |
| Название сделки | Ссылка на Deal                              |
| Статус          | Текущий этап pipeline или итог (Won/Failed) |
| Сумма           | Ожидаемая / фактическая сумма               |
| Дата            | Дата создания / закрытия                    |
| Seller          | Ответственный продавец                      |

### 5.5. История коммуникаций

Хронологический лог взаимодействий с контактом:

| Тип записи | Описание                                   |
| ---------- | ------------------------------------------ |
| Звонок     | Дата, время, длительность, результат       |
| Встреча    | Дата, время, участники, итог               |
| Сообщение  | Отправленные уведомления (WhatsApp, Email) |
| Заметка    | Ручная заметка Seller/PM                   |

### 5.6. Финансовая сводка (для CEO, Finance, Seller)

| Показатель           | Описание                                               |
| -------------------- | ------------------------------------------------------ |
| Total Lifetime Value | Общая сумма всех оплат по всем проектам этого контакта |
| Active Subscriptions | Количество и сумма активных подписок                   |
| Outstanding Balance  | Непогашенная задолженность по всем проектам            |
| Last Payment         | Дата и сумма последнего платежа                        |

## 6. Связь Contact с другими сущностями

### Contact → Projects (1:N)

Один контакт может быть заказчиком множества проектов:

```
Арам Петросян (Contact)
  ├── Project "Обувной бренд" (Website + App)
  ├── Project "Цветочный магазин" (Website)
  └── Project "Транспортная компания" (CRM)
```

### Contact → Companies (1:N)

Один контакт может представлять несколько юрлиц:

```
Арам Петросян (Contact)
  ├── ООО "АрамОбувь" (Legal Entity, Tax)
  ├── ИП Петросян (Sole Proprietor, Tax)
  └── Aram Transport LLC (Legal Entity, Tax-Free)
```

### Contact → Deals (1:N)

Один контакт может иметь множество сделок (текущих и прошлых):

```
Арам Петросян (Contact)
  ├── Deal "Website для обувного бренда" → Won
  ├── Deal "App для обувного бренда" → Won
  ├── Deal "Website для цветов" → Won
  └── Deal "Extension: личный кабинет" → In Progress
```

### Contact → Leads (1:1 или 1:N)

Contact создаётся из Lead. Если один человек обращался несколько раз через разные каналы — множественные Lead привязываются к одному Contact.

## 7. Дедупликация контактов

### Проблема:

Один человек может обратиться через Instagram, потом через сайт, потом позвонить. Каждое обращение создаёт Lead. Важно не создать дублирующие Contact.

### Решение:

При создании Contact система проверяет:

1. Совпадение по номеру телефона
2. Совпадение по email
3. Совпадение по имени + фамилии (с предупреждением, т.к. может быть тёзка)

Если найдено совпадение:

- Система предлагает привязать Lead к существующему Contact
- Seller подтверждает или создаёт новый (если это действительно другой человек)

### Merge контактов:

Если дубликат всё-таки был создан, CEO/PM может объединить два Contact:

- Выбирается «основной» контакт (чьи данные сохраняются)
- Все проекты, компании, сделки, история «дубликата» переносятся на основной
- Дубликат удаляется
- Audit Log фиксирует merge

## 8. Доступ к Contacts

| Роль                 | Уровень доступа                                                                                |
| -------------------- | ---------------------------------------------------------------------------------------------- |
| **CEO**              | Полный доступ: все контакты, все данные, финансовая сводка                                     |
| **Finance Director** | Чтение: основная информация, финансовая сводка                                                 |
| **Seller**           | Полный доступ: свои контакты + создание новых. Чтение: все контакты                            |
| **PM**               | Чтение: контакты по своим проектам. Основная информация + мессенджеры                          |
| **Developer**        | Нет прямого доступа (видит контакт только в контексте проекта — имя и нужные проектные данные) |
| **Designer**         | Нет прямого доступа                                                                            |
| **QA**               | Нет прямого доступа                                                                            |
| **Marketing**        | Чтение: контакты и источники (для аналитики)                                                   |

## 10. Автоматизация

### Уведомления:

- При создании Contact из Lead — уведомление Seller
- При просрочке Invoice по проекту контакта — контакт получает напоминание через preferred_channel (WhatsApp по умолчанию)
- При приближении дедлайна проекта — PM и контакт получают уведомления

### Связь с Messenger:

- Входящие сообщения от контакта (Instagram, Facebook, WhatsApp) автоматически привязываются к его карточке
- История переписки доступна из карточки контакта
- При ответе через CRM-мессенджер сообщение отправляется в канал контакта

## 11. Особые случаи

### 11.1. Контакт без проектов

Contact может существовать без проектов — это потенциальный клиент, который ещё не заказал ничего. Такие контакты создаются из квалифицированных лидов, даже если сделка пока не закрыта.

### 11.2. Контакт-партнёр

Contact с типом `Partner` связан с Partner-сущностью. Через карточку контакта можно перейти к деталям партнёрского соглашения.

### 11.3. Контакт с множеством бизнесов

Крупный предприниматель может иметь 10+ проектов. Карточка контакта показывает вход в Client Portfolio, но сам Client Portfolio остаётся отдельным вычисляемым view (см. документ `03-Client-Portfolio.md`).

### 11.4. Несколько контактных лиц на один проект

Хотя у проекта есть один основной контакт, могут быть и другие контактные лица: технический директор клиента, бухгалтер, маркетолог клиента. Они создаются как отдельные Contact и привязываются к Company / Project / Invoice через конкретные связи.

Сотрудник не обязан выбирать сложную роль. Если роль не влияет на процесс, достаточно связи. Например:

- контакт выбран в Deal — это контакт сделки;
- контакт указан в Company — это контакт компании;
- контакт указан в invoice settings — это контакт для оплаты;
- контакт добавлен в Project — это контакт проекта.

## 12. Валидация данных

| Правило               | Описание                                                                                                          |
| --------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Обязательные поля     | first_name, last_name, phone, contact_type                                                                        |
| Уникальность телефона | Предупреждение при дублировании (не жёсткая блокировка, т.к. один номер может быть у нескольких людей — компания) |
| Формат телефона       | Международный формат (+374..., +7..., +1...)                                                                      |
| Формат email          | Валидация email-адреса                                                                                            |
| Не удаляется          | Contact с привязанными проектами, сделками или компаниями не удаляется, только архивируется                       |
