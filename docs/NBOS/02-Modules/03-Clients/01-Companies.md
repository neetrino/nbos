# Компании (Companies)

## Обзор

**Company** (Компания) — это юридическое лицо или физическое лицо, на которое выставляются счета. Компания представляет собой «биллинговую сущность»: все Invoice, которые Neetrino выставляет клиенту, привязаны к конкретной Company с её реквизитами.

Компания — это **НЕ** проект. Одна компания может иметь несколько проектов (хоть и редко), а один проект всегда привязан к одной компании для целей биллинга.

---

## 1. Типы компаний

| Тип | Описание | Пример |
|-----|----------|--------|
| **Legal Entity** (ООО) | Юридическое лицо с полным комплектом реквизитов | ООО «Арменфуд», Neetrino LLC |
| **Sole Proprietor** (ИП) | Индивидуальный предприниматель | ИП Петросян А.Г. |
| **Individual** (Физлицо) | Физическое лицо без статуса ИП | Паспортные данные для выставления счёта |

---

## 2. Поля карточки Company

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| name | String | ✅ | Название компании / ФИО физлица |
| type | Enum | ✅ | Legal Entity, Sole Proprietor, Individual |
| tax_id | String | Для Legal/SP | ИНН / VOEN / Tax ID (государственный идентификатор) |
| legal_address | String | Для Legal/SP | Юридический адрес |
| bank_details | JSON | Для Legal/SP | Банковские реквизиты: банк, номер счёта, БИК/SWIFT |
| tax_status | Enum | ✅ | **Tax** (налогооблагаемый) или **Tax-Free** |
| primary_contact | FK → Contact | ✅ | Основной контакт (человек) |
| phone | String | — | Телефон компании (если отличается от контакта) |
| email | String | — | Email компании |
| country | String | — | Страна (влияет на налоговый статус) |
| notes | Text | — | Заметки, особые условия |
| created_at | DateTime | auto | Дата создания записи |

---

## 3. Налоговый статус (Tax Status) — критически важное поле

### Что это

Поле **tax_status** определяет, выставляются ли счета через государственную налоговую систему Армении или нет.

| Статус | Описание | Когда используется |
|--------|----------|--------------------|
| **Tax** | Счёт обязательно проходит через государственную систему | Армянские клиенты (резиденты) |
| **Tax-Free** | Счёт НЕ проходит через госсистему | Иностранные клиенты, некоторые специальные случаи |

### Правила:

1. **Tax-status устанавливается один раз** при создании Company и не меняется на протяжении всего сотрудничества.

2. **Наследование:** все Orders, Subscriptions, Invoices, созданные для этой компании, наследуют её tax_status:
   ```
   Company (tax_status: Tax)
     └── Order → Invoice (tax_status: Tax)
     └── Subscription → monthly Invoice (tax_status: Tax)
   ```

3. **Влияние на процесс Invoice:**
   - **Tax:** Invoice создан → Уведомление бухгалтеру → Бухгалтер создаёт счёт в государственной программе → Отправка клиенту → Оплата
   - **Tax-Free:** Invoice создан → Отправка клиенту напрямую → Оплата (без этапа государственной системы)

4. **Иностранные клиенты** — практически всегда Tax-Free, так как нет необходимости в армянском налоговом инвойсе.

## 4. Связь Company с другими сущностями

### Company ≠ Project

```
Company (юрлицо)
  ├── Project "Brand A"    ← биллинг через эту Company
  ├── Project "Brand B"    ← (редко) тот же юрлицо, другой бренд
  └── Project "Brand C"    ← (ещё реже) третий бренд
```

В 95% случаев связь 1:1 — одна компания, один проект. Но архитектура поддерживает 1:N, потому что один предприниматель может использовать одно юрлицо для нескольких бизнесов.

### Company ≠ Contact

```
Contact (человек: Арам Петросян)
  ├── Company "ООО АрамОбувь"     ← юрлицо для обувного бизнеса
  ├── Company "ИП Петросян А.Г."  ← ИП для цветочного магазина
  └── Company "Aram Shoes LLC"    ← зарубежное юрлицо
```

Один человек может представлять несколько юридических лиц. Каждое юрлицо — отдельная Company.

### Полная иерархия:

```
Contact (человек)
  └── Company (юрлицо)
        └── Project (бизнес/бренд)
              ├── Order (заказ)
              │     └── Invoice (счёт) ← выставляется на Company
              └── Subscription (подписка)
                    └── Invoice (ежемесячный счёт) ← выставляется на Company
```

## 5. Карточка Company — что отображается

При открытии карточки компании пользователь видит:

### 5.1. Основная информация

Все поля из таблицы в разделе 2: название, тип, ИНН, адрес, реквизиты, налоговый статус, основной контакт.

### 5.2. Список проектов

Все проекты, привязанные к этой компании:

| Поле | Описание |
|------|----------|
| Название проекта | Ссылка на Project в Projects Hub |
| Статус | Computed-статус проекта (Development / Maintenance / Closed) |
| Дата создания | Когда проект был создан |

### 5.3. Список контактов

Все контакты, связанные с этой компанией:

| Поле | Описание |
|------|----------|
| Имя | ФИО контакта |
| Роль | Client, Partner, Contractor |
| Телефон | Основной телефон |
| Email | Email |

### 5.4. Финансовая сводка

| Показатель | Описание |
|------------|----------|
| Общий доход (Lifetime Revenue) | Сумма всех оплаченных Invoice по всем проектам |
| Непогашенная задолженность (Outstanding Balance) | Сумма неоплаченных Invoice |
| Активные подписки | Количество и общая сумма активных Subscription |
| Последний платёж | Дата и сумма последнего полученного Payment |

### 5.5. Список Invoice

Все счета, выставленные на эту компанию, с фильтрами по статусу (Paid, Overdue, Pending) и по типу (Development, Subscription, Domain).

## 6. Создание Company

### Когда создаётся:
- При конвертации Lead → Deal, если клиент новый
- Вручную из раздела Clients → Companies
- При создании проекта, если нужно новое юрлицо

### Кто создаёт:
- **Seller** — при работе со сделкой
- **Finance Director** — при необходимости корректировки реквизитов
- **CEO** — в любом случае

### Обязательные поля при создании:
- Название
- Тип (Legal / Sole Proprietor / Individual)
- Налоговый статус (Tax / Tax-Free) — **определяется один раз и не меняется**
- Основной контакт

---

## 7. Доступ к Company

| Роль | Уровень доступа |
|------|----------------|
| **CEO** | Полный доступ: все поля, все компании |
| **Finance Director** | Полный доступ: реквизиты, финансовая сводка, Invoice |
| **Seller** | Чтение: основная информация, список проектов. Создание новых Company |
| **PM** | Чтение: название, контакт, налоговый статус (для понимания контекста) |
| **Developer** | Нет доступа |
| **Designer** | Нет доступа |
| **QA** | Нет доступа |

## 8. Особые случаи

### 8.1. Смена юрлица клиента

Клиент может попросить перевести биллинг на другое юрлицо (например, реорганизация бизнеса). В этом случае:
1. Создаётся новая Company с новыми реквизитами
2. Project перепривязывается к новой Company
3. Старые Invoice остаются привязанными к старой Company (для истории)
4. Новые Invoice создаются на новую Company
5. Audit Log фиксирует изменение

### 8.2. Одна Company — несколько проектов

Редкий, но поддерживаемый сценарий. Один предприниматель с одним юрлицом заказывает несколько проектов для разных бизнесов. В финансовой сводке Company видны все проекты суммарно.

### 8.3. Иностранные клиенты

Иностранные клиенты практически всегда имеют статус Tax-Free:
- Нет армянского налогового ID
- Счёт выставляется напрямую (без государственной системы)
- Адрес и реквизиты в свободной форме
- Валюта может отличаться (USD, EUR)

### 8.4. Pass-Through расходы (домены, сервисы)

Когда Neetrino покупает домен или сервис от имени клиента, Invoice для возмещения этих расходов выставляется на Company клиента. Налоговый статус этого Invoice наследуется от Company:
- Если Company = Tax → Invoice за домен тоже Tax → проходит через госсистему
- Если Company = Tax-Free → Invoice за домен тоже Tax-Free → напрямую клиенту

---

## 9. Валидация данных

| Правило | Описание |
|---------|----------|
| Уникальность tax_id | Два Company не могут иметь одинаковый Tax ID |
| Tax Status неизменяем | После создания Company нельзя менять tax_status (только CEO через специальную процедуру) |
| Обязательный Contact | Company не может существовать без привязанного Contact |
| Bank Details для Tax | Если tax_status = Tax, банковские реквизиты обязательны |
| Не удаляется | Company с привязанными проектами или Invoice не может быть удалена, только архивирована |
