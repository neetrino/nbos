# Lead Pipeline (Воронка лидов)

> NBOS Platform — обработка входящего потока лидов

## Назначение

Lead Pipeline — это первый этап воронки продаж. Сюда попадают все входящие обращения — из рекламы, сайта, партнёров, холодных звонков. Задача воронки — быстро отфильтровать качественных лидов (SQL) от спама и «замороженных», не тратя время команды продаж на мусорный трафик.

**Объём:** 100–200+ лидов в месяц  
**Обработка:** минимальная — зафиксировать контакт, классифицировать, передать качественных в Deals  
**Ответственные:** Seller (обработка), Marketing (аналитика источников)

## Стадии воронки лидов

### Активные стадии

#### 1. New (Новый)

| Параметр | Описание |
|---|---|
| **Что происходит** | Лид только что поступил в систему. Ещё не обработан ни одним продавцом |
| **Ответственный** | Назначается автоматически (round-robin по продавцам) или вручную Head of Sales |
| **Доступные действия** | Позвонить, написать, переместить в следующую стадию, отметить как SPAM |
| **Обязательные поля** | Имя (или название компании), источник лида, хотя бы один контакт (телефон или email) |
| **Автоматизация** | При создании лида — уведомление назначенному Seller. Если лид не обработан 24 часа — повторное уведомление + уведомление Head of Sales |

**Правила:**
- Новый лид должен быть обработан в течение 24 часов (рабочих)
- Если источник — Instagram/Facebook DM, лид создаётся автоматически из CRM-мессенджера
- Если источник — форма на сайте, лид создаётся автоматически с заполненными полями

---

#### 2. Didn't Get Through (Не дозвонились)

| Параметр | Описание |
|---|---|
| **Что происходит** | Seller попытался связаться, но не смог достичь контакта — не берёт трубку, не отвечает на сообщение |
| **Ответственный** | Назначенный Seller |
| **Доступные действия** | Повторная попытка связи, добавление заметки, перемещение дальше или в Frozen |
| **Обязательные поля** | Заметка о попытке связи (дата, канал, результат) |
| **Автоматизация** | Напоминание Seller о повторной попытке через 1 день, через 3 дня, через 7 дней. После 3-х неудачных попыток — предложение перевести в Frozen |

**Правила:**
- Минимум 3 попытки связи через разные каналы (звонок + сообщение)
- Между попытками — минимум 1 рабочий день
- После 3-х неудачных попыток Seller решает: Frozen или ещё одна попытка

---

#### 3. Contact Established (Контакт установлен)

| Параметр | Описание |
|---|---|
| **Что происходит** | Первичный контакт состоялся. Seller поговорил с потенциальным клиентом, выяснил базовую информацию |
| **Ответственный** | Назначенный Seller |
| **Доступные действия** | Заполнить карточку лида подробнее, назначить следующий шаг, перевести на Qualification |
| **Обязательные поля** | Заметка о результате разговора, тип интереса (какая услуга нужна) |
| **Автоматизация** | Нет автоматизации — Seller сам решает следующий шаг |

**Задача на этой стадии:**
- Узнать, что нужно клиенту (веб-сайт, приложение, доработка и т.д.)
- Оценить предварительный бюджет и сроки
- Понять, насколько серьёзно намерение (просто интересуется vs. готов обсуждать)

---

#### 4. Qualification / MQL (Маркетинг-квалификация)

| Параметр | Описание |
|---|---|
| **Что происходит** | Seller оценивает лида по критериям качества. Определяется, стоит ли переводить в полноценную сделку |
| **Ответственный** | Seller + Head of Sales (при сомнениях) |
| **Доступные действия** | Квалифицировать как SQL (→ Deal), перевести в Frozen, отметить как SPAM |
| **Обязательные поля** | Тип услуги, примерный бюджет, примерные сроки, источник |
| **Автоматизация** | Нет — решение принимает Seller на основе разговора |

**Критерии квалификации (MQL → SQL):**
- Клиент имеет конкретную потребность (не «просто узнать»)
- Есть бюджет или готовность обсуждать бюджет
- Есть сроки (хотя бы примерные)
- Клиент принимает решение (или имеет доступ к ЛПР)

---

### Терминальные стадии

#### SPAM (Спам)

| Параметр | Описание |
|---|---|
| **Что происходит** | Лид определён как нецелевой: рекламные рассылки, ботоферма, ошибочные обращения, нерелевантные запросы |
| **Текущий объём** | ~1,512 записей (из Bitrix) |
| **Действия** | Перемещение в архив. Данные сохраняются для аналитики (какой % от источника даёт спам) |
| **Обратный переход** | Не предусмотрен. Если ошибка — создаётся новый лид |

---

#### Frozen (Заморожен)

| Параметр | Описание |
|---|---|
| **Что происходит** | Клиент не готов сейчас, но может вернуться в будущем. «Нет бюджета», «не сейчас», «подумаю», «через полгода» |
| **Текущий объём** | ~101 запись (из Bitrix) |
| **Действия** | Архивация с пометкой причины заморозки и предполагаемой датой возврата |
| **Автоматизация** | Напоминание Seller через установленный период (по умолчанию 3 месяца): «Проверить статус замороженного лида» |
| **Обратный переход** | Да — можно реактивировать в стадию «Contact Established» и продолжить работу |

---

#### Quality Lead / SQL (Качественный лид → переход в Deal)

| Параметр | Описание |
|---|---|
| **Что происходит** | Лид квалифицирован как готовый к работе. Создаётся полноценная сделка (Deal) в Deal Pipeline |
| **Текущий объём** | ~920 конвертированных (из Bitrix) |
| **Действия** | Конвертация в Deal — автоматическое создание карточки сделки |
| **Обязательные поля для конвертации** | Имя контакта, телефон/email, источник, тип услуги, ответственный Seller |
| **Автоматизация** | При конвертации: создание Deal на стадии «Start a Conversation», перенос данных контакта, привязка к источнику лида |

## Визуальная схема воронки

```
          ┌─────────────────────────────────────────────────────┐
          │                  LEAD PIPELINE                       │
          │                                                      │
  INPUT → │  New → Didn't Get → Contact → Qualification (MQL)  │
          │   │     Through      Established        │            │
          │   │        │              │              │            │
          │   ▼        ▼              │              ▼            │
          │  SPAM    Frozen ◄─────────┘         Quality Lead     │
          │ (archive) (archive)                    (SQL)         │
          │                                         │            │
          └─────────────────────────────────────────┼────────────┘
                                                    │
                                                    ▼
                                              DEAL PIPELINE
                                          (Start a Conversation)
```

## Карточка лида (Lead Card)

### Основные поля

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| Имя контакта | Текст | Да | ФИО или название компании |
| Телефон | Телефон | Да* | Основной номер |
| Email | Email | Да* | Электронная почта |
| Источник (Source) | Выбор из списка | Да | Откуда пришёл лид |
| Статус / Стадия | Автоматически | Да | Текущая стадия в pipeline |
| Ответственный Seller | Выбор сотрудника | Да | Кто обрабатывает лид |
| Тип интереса | Выбор из списка | Нет | Какая услуга интересует |
| Примерный бюджет | Число | Нет | Ориентировочный бюджет клиента |
| Заметки | Текст | Нет | Свободные заметки о контакте |
| Дата создания | Дата/время | Авто | Когда лид попал в систему |
| Дата последнего контакта | Дата/время | Авто | Когда был последний контакт |

> *Обязательно хотя бы одно: телефон или email

## Источники лидов (Lead Sources)

Отслеживание источников критично для маркетинговой аналитики и расчёта ROI.

| Источник | Описание | Автоматизация |
|---|---|---|
| **Instagram** | DM из Instagram-аккаунта компании | Авто-создание лида из CRM-мессенджера |
| **Facebook** | Сообщения из Facebook-страницы | Авто-создание лида из CRM-мессенджера |
| **Website** | Форма заявки на сайте | Авто-создание лида с заполненными полями |
| **Cold Call** | Исходящий холодный звонок | Ручное создание Seller |
| **Partner** | Передано партнёром | Ручное создание + привязка к партнёру |
| **Referral** | Рекомендация от существующего клиента | Ручное создание + пометка «реферал» |
| **Other** | Другие каналы | Ручное создание |

### Аналитика по источникам

Для каждого источника отслеживается:
- Количество лидов за период
- % SPAM от источника
- Конверсия в SQL
- Конверсия в Deal Won
- Сумма закрытых сделок
- Стоимость привлечения лида (CPL) — если известны рекламные расходы
- ROI канала

## Автоматизация воронки лидов

| Триггер | Действие | Задержка |
|---|---|---|
| Новый лид создан | Уведомление назначенному Seller | Мгновенно |
| Лид в «New» > 24 часов | Повторное уведомление Seller + Head of Sales | 24 ч |
| Лид в «Didn't Get Through» | Напоминание о повторном звонке | 1 день, 3 дня, 7 дней |
| 3 неудачных попытки связи | Предложение перевести в Frozen | После 3-й попытки |
| Лид переведён в Frozen | Напоминание проверить статус | Через 3 месяца |
| Лид квалифицирован как SQL | Создание Deal, перенос данных | Мгновенно |
| Лид из Instagram/Facebook | Авто-создание карточки лида | Мгновенно |
| Лид с формы сайта | Авто-создание с заполнением полей | Мгновенно |

---

## Метрики воронки лидов

| Метрика | Формула | Целевое значение |
|---|---|---|
| Скорость обработки | Среднее время New → следующая стадия | < 24 часов |
| Конверсия в SQL | SQL / (Все лиды − SPAM) | Отслеживается |
| % SPAM | SPAM / Все лиды | Отслеживается по каналам |
| Frozen rate | Frozen / (Все лиды − SPAM) | Отслеживается |
| Время в Frozen | Среднее время до реактивации или архивации | Отслеживается |
| Источник с лучшей конверсией | Конверсия SQL по каждому источнику | Аналитика для Marketing |
