# Расходы (Expenses)

## Общая концепция

`Expenses / Расходы` - это контур исходящих денег компании.

Главная задача модуля:

- заранее видеть будущие расходы;
- контролировать текущие платежи;
- не терять долги и отложенные расходы;
- фиксировать частичные и полные оплаты;
- связывать расходы с проектами, заказами, сервисами клиента, зарплатой, бонусами и P&L.

Расходы нельзя хранить как одну простую карточку со стадиями. Нужно разделить четыре слоя:

| Слой              | Русское название | Смысл                                    |
| ----------------- | ---------------- | ---------------------------------------- |
| `Expense Plan`    | План расхода     | Правило или ожидание будущего расхода    |
| `Expense Card`    | Карточка расхода | Конкретная сумма, которую нужно оплатить |
| `Expense Payment` | Оплата расхода   | Факт частичной или полной оплаты         |
| `Expense Board`   | Доска расходов   | Рабочий экран контроля текущих расходов  |

---

## Expense Plan / План расхода

`Expense Plan / План расхода` описывает расход, который известен заранее.

Примеры:

- аренда офиса каждый месяц;
- бухгалтерские услуги каждый месяц;
- hosting каждый месяц;
- домен раз в год;
- Apple Developer Account раз в год;
- Figma / GitHub / Google Workspace;
- регулярный SaaS или внешний сервис.

План расхода сам по себе не означает, что деньги уже ушли. Он только говорит системе, когда нужно создать конкретную `Expense Card / Карточку расхода`.

### Поля плана расхода

| Поле                    | Описание                                                        |
| ----------------------- | --------------------------------------------------------------- |
| `name`                  | Название расхода                                                |
| `category`              | Domain / Hosting / Service / Office / Tools / Marketing / Other |
| `amount`                | Ожидаемая сумма                                                 |
| `currency`              | Валюта                                                          |
| `frequency`             | Monthly / Quarterly / Yearly / One-time / Custom                |
| `next_due_date`         | Следующая дата оплаты                                           |
| `provider`              | Поставщик                                                       |
| `project`               | Проект, если расход проектный                                   |
| `product`               | Продукт, если применимо                                         |
| `client_service_record` | Связанный сервис клиента, если расход идёт от него              |
| `auto_generate`         | Создавать ли карточки автоматически                             |
| `notes`                 | Комментарии                                                     |

### Где смотреть планы расходов

Планы расходов должны жить на отдельном экране:

`Expense Plans / Плановые расходы`

Основной вид:

`Calendar Grid / Календарная сетка`

- слева строки: статьи расходов;
- сверху месяцы;
- в ячейке: ожидаемая сумма и статус;
- по клику: карточка плана или список карточек за месяц.

Дополнительные виды:

- `List / Список`;
- `Board / Доска`;
- `Calendar/Grid / Календарная сетка`.

Последний выбранный пользователем вид должен запоминаться.

## Expense Card / Карточка расхода

`Expense Card / Карточка расхода` - это конкретная сумма, которую нужно оплатить.

Примеры:

- `Beget Hosting / April / 6,000 AMD`;
- `Domain example.am / Renewal / 12,000 AMD`;
- `Office Rent / May / 300,000 AMD`;
- `Salary Anna / January / 300,000 AMD`.

Карточка может создаваться:

- автоматически из `Expense Plan`;
- автоматически из `Client Service Record` после оплаты клиента;
- автоматически из `Payroll Run`;
- автоматически из `Bonus / Partner Payout`;
- вручную как `Unplanned Expense / Внеплановый расход`.

### Поля карточки расхода

| Поле                    | Описание                                                                  |
| ----------------------- | ------------------------------------------------------------------------- |
| `name`                  | Название                                                                  |
| `source_type`           | Manual / Expense Plan / Client Service / Payroll / Bonus / Partner Payout |
| `category`              | Категория расхода                                                         |
| `original_amount`       | Исходная сумма                                                            |
| `paid_amount`           | Уже оплачено                                                              |
| `remaining_amount`      | Осталось оплатить                                                         |
| `currency`              | Валюта                                                                    |
| `due_date`              | Дата оплаты                                                               |
| `workflow_status`       | Текущий статус на доске                                                   |
| `payment_status`        | Unpaid / Partially Paid / Paid                                            |
| `project`               | Проект, если расход проектный                                             |
| `product`               | Продукт, если применимо                                                   |
| `order`                 | Заказ, если применимо                                                     |
| `client_service_record` | Связанный сервис клиента                                                  |
| `invoice_card`          | Связанная карточка оплаты клиента, если pass-through                      |
| `owner`                 | Ответственный                                                             |
| `notes`                 | Комментарии                                                               |

## Expense Payment / Оплата расхода

Расход должен поддерживать частичные оплаты.

Нельзя просто уменьшать сумму карточки с `300,000` до `200,000`, потому что потеряется история. Вместо этого создаются отдельные записи `Expense Payment / Оплата расхода`.

Пример:

| Дата       | Сумма   | Комментарий           |
| ---------- | ------- | --------------------- |
| 2026-05-05 | 100,000 | Первая часть зарплаты |
| 2026-05-12 | 100,000 | Вторая часть          |
| 2026-05-20 | 100,000 | Финальная часть       |

На карточке видно:

- исходная сумма: `300,000`;
- оплачено: `100,000`;
- осталось: `200,000`;
- статус: `Partially Paid / Частично оплачено`.

Когда `remaining_amount = 0`, карточка получает `payment_status = Paid`.

### Поля оплаты расхода

| Поле             | Описание                              |
| ---------------- | ------------------------------------- |
| `expense_card`   | Связанная карточка расхода            |
| `amount`         | Сумма оплаты                          |
| `currency`       | Валюта                                |
| `paid_at`        | Дата оплаты                           |
| `paid_by`        | Кто отметил оплату                    |
| `payment_method` | Метод оплаты, если используется       |
| `attachment`     | Файл / скрин подтверждения, если есть |
| `comment`        | Комментарий                           |

## Expense Board / Доска расходов

`Expense Board / Доска расходов` показывает текущие карточки расходов, с которыми Finance работает сейчас.

Колонки:

| Колонка                     | Описание                                |
| --------------------------- | --------------------------------------- |
| `Planned / Запланировано`   | Карточка создана, но срок ещё не близко |
| `Due Soon / Скоро оплатить` | Оплата приближается                     |
| `Due Now / Нужно оплатить`  | Пора оплачивать сейчас                  |
| `Overdue / Просрочено`      | Срок оплаты прошёл                      |
| `On Hold / На паузе`        | Временная пауза внутри текущего цикла   |

`Paid / Оплачено` и `Cancelled / Отменено` не должны мешать рабочей доске. Они уходят в `Closed / Закрытые`.

Оплата делается действием:

`Add Payment / Добавить оплату`

Это действие может быть частичной или полной оплатой.

## Expense Backlog / Долги и отложенные расходы

Старый смысл стадии `Old` нужно заменить на отдельный контур:

`Expense Backlog / Долги и отложенные расходы`

Это место для расходов, которые вышли из текущего цикла оплаты и не должны мешать видеть реальные расходы текущего месяца.

Примеры:

- долг, который будем закрывать позже;
- расход, по которому ждём решение;
- временно непонятно, нужно ли платить;
- старый долг, который оплачивается частями по мере возможности.

Важно:

- backlog не считается активным текущим расходом месяца;
- backlog не должен искажать сумму денег, нужную на текущий платёжный цикл;
- backlog должен иметь отдельную сумму накопленного долга;
- карточку можно вернуть из backlog обратно на `Expense Board`, когда решили оплачивать.

### Причины backlog

| Причина                                    | Описание                            |
| ------------------------------------------ | ----------------------------------- |
| `Debt to Pay Later / Долг, оплатить позже` | Обязательство есть, но платим позже |
| `Waiting for Decision / Ждём решение`      | Нужна управленческая ясность        |
| `Waiting for Client / Ждём клиента`        | Клиентский фактор                   |
| `Waiting for Provider / Ждём поставщика`   | Поставщик не готов / не ответил     |
| `Other / Другое`                           | Другая причина                      |

---

## On Hold / Пауза

`On Hold / На паузе` остаётся как временный статус внутри текущего цикла.

Он отличается от backlog:

| Состояние         | Когда использовать                                                            |
| ----------------- | ----------------------------------------------------------------------------- |
| `On Hold`         | Расход всё ещё относится к текущему рабочему циклу, но временно приостановлен |
| `Expense Backlog` | Расход выведен из текущего цикла и ведётся отдельно                           |

Для `On Hold` желательно хранить:

- причину;
- дату постановки;
- дату пересмотра, если известна.

---

## Unplanned Expense / Внеплановый расход

`Unplanned Expense / Внеплановый расход` создаётся вручную, без плана.

Примеры:

- срочная покупка сервиса;
- незапланированная реклама;
- ремонт техники;
- подарок клиенту;
- экстренная лицензия.

После создания такая карточка попадает на обычную `Expense Board`.

Обязательные поля:

- название;
- сумма;
- валюта;
- категория;
- дата оплаты;
- описание или причина.

## Pass-through / Клиентский расход через нас

Pass-through - это расход, который Neetrino делает от имени клиента.

Правильный процесс:

```text
Client Service Record
    ->
Invoice Card
    ->
Payment
    ->
Expense Card
    ->
Task
    ->
Paid
```

Смысл:

1. в `Client Service Record` есть сервис, который клиент должен оплатить;
2. создаётся `Invoice Card`;
3. клиент платит;
4. после оплаты создаётся `Expense Card`;
5. к расходу прикрепляется задача на покупку / продление / оплату;
6. когда мы реально оплатили поставщику, расход отмечается как `Paid`.

На `Client Service Record` нужно показывать обе стороны:

- связанная `Invoice Card`: получили ли деньги от клиента;
- связанная `Expense Card`: оплатили ли мы поставщику.

Это позволяет видеть полный цикл по каждому сервису клиента.

## Salary, Bonus и Payroll

Зарплата и бонусы не должны вручную жить как обычные расходы.

Правильный процесс:

```text
Compensation Profile
    ->
Bonus Ledger / KPI
    ->
Payroll Run
    ->
Expense Card
    ->
Expense Payment
```

То есть:

- `Payroll Run / Зарплатный расчёт` является источником зарплатных расходов;
- утверждённый payroll создаёт `Expense Cards`;
- выплата зарплаты может быть полной или частичной;
- каждая частичная выплата фиксируется через `Expense Payment`.

## Привязка к P&L

Каждый расход должен иметь уровень привязки:

| Уровень        | Где отражается                            |
| -------------- | ----------------------------------------- |
| Company        | Только Company P&L                        |
| Project        | Project P&L и Company P&L                 |
| Product        | Product/Project P&L и Company P&L         |
| Order          | Order P&L, Project P&L и Company P&L      |
| Client Service | Сервис клиента, Project P&L и Company P&L |

Это нужно, чтобы видеть маржинальность на каждом уровне.

## Виды отображения

Для расходов должны быть доступны:

- `List / Список`;
- `Board / Доска`;
- `Calendar/Grid / Календарная сетка`.

Правило:

- система может иметь default view;
- но последний выбранный пользователем вид должен запоминаться;
- при следующем открытии раздела пользователь видит тот же вид.

### Expense Plans

Основной default view:

`Calendar Grid / Календарная сетка`

Потому что планы удобнее смотреть по месяцам.

### Expense Cards

Основной default view:

`Board / Доска`

Потому что карточки расходов являются текущей операционной работой Finance.

### Expense Backlog

Основной default view:

`List / Список`

Потому что это отдельный список долгов и отложенных обязательств.

## Уведомления и автоматизация

| Событие                                | Действие                                 |
| -------------------------------------- | ---------------------------------------- |
| Наступает дата из Expense Plan         | Создать Expense Card                     |
| Расход скоро нужно оплатить            | Перевести в Due Soon и уведомить Finance |
| Срок оплаты наступил                   | Перевести в Due Now                      |
| Срок прошёл                            | Перевести в Overdue                      |
| Клиент оплатил Invoice Card по сервису | Создать Expense Card                     |
| Expense Card создана из Client Service | Создать Task на покупку / продление      |
| Expense Card частично оплачена         | Обновить paid_amount и remaining_amount  |
| Expense Card полностью оплачена        | Перевести в Paid / Closed                |
| Backlog item долго не пересматривался  | Уведомить Finance / CEO                  |

## Связи с другими модулями

```text
Expense Plan -> Expense Card
Expense Card -> Expense Payment
Client Service Record -> Invoice Card -> Payment -> Expense Card -> Task -> Paid
Payroll Run -> Expense Card -> Expense Payment
Bonus Ledger -> Payroll Run -> Expense Card
Partner Payout -> Expense Card
Expense Cards -> P&L
```

Связанные модули:

- `Finance / Invoices` - если расход создаётся после оплаты клиента;
- `Finance / Client Services` - домены, hosting, accounts, services;
- `Tasks / Work Space` - задачи на покупку / продление / оплату;
- `My Company / Payroll` - зарплаты и бонусы;
- `Projects Hub` - привязка к проектам и продуктам;
- `P&L Reports` - отчётность по затратам.
