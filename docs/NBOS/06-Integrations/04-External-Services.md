# Интеграция с внешними сервисами

## Общее описание

Помимо ключевых интеграций (WhatsApp, Telegram, банк), NBOS планирует подключение к ряду внешних сервисов для расширения функциональности. Большинство этих интеграций не являются приоритетом для MVP и запланированы на более поздние фазы.

---

## Google Drive

### Назначение

Синхронизация файлов проектов между NBOS Drive и Google Drive. Neetrino активно использует Google Drive для хранения проектной документации, дизайнов и файлов клиентов.

### Текущий процесс

- Файлы проектов хранятся в Google Drive в папках по проектам
- Сотрудники вручную загружают/скачивают файлы
- Нет единой точки доступа — нужно переключаться между NBOS и Google Drive

### Планируемая интеграция

| Функция | Описание | Направление |
|---------|----------|-------------|
| Привязка папки | Каждый проект в NBOS привязан к папке в Google Drive | Настройка |
| Загрузка → синхронизация | Файл, загруженный в NBOS Drive, автоматически копируется в Google Drive | NBOS → Google Drive |
| Обратная синхронизация | Файлы из Google Drive отображаются в NBOS Drive | Google Drive → NBOS |
| Ссылки | В карточке проекта — прямая ссылка на папку Google Drive | Навигация |

### Техническая реализация

```
NBOS Drive ←→ Google Drive API (v3)
  - OAuth 2.0 авторизация (сервисный аккаунт Neetrino)
  - Webhook / Polling для отслеживания изменений
  - Двусторонняя синхронизация файлов
```

| Аспект | Детали |
|--------|--------|
| **API** | Google Drive API v3 |
| **Авторизация** | OAuth 2.0 / Service Account |
| **Синхронизация** | По событию (upload/change) или периодическая (каждые N минут) |
| **Ограничения** | Квоты Google API, размер файлов, форматы |
| **Конфликты** | Файл изменён в обоих местах — последняя версия побеждает + уведомление |

### Фаза реализации

| Этап | Описание |
|------|----------|
| **MVP** | Ссылка на Google Drive папку в карточке проекта (без синхронизации) |
| **Фаза 2** | Одностороняя синхронизация: NBOS → Google Drive |
| **Фаза 3** | Двусторонняя синхронизация |

## GitHub

### Назначение

Привязка репозиториев кода к проектам в NBOS. Разработчики и PM получают контекст о разработке прямо в проектной карточке.

### Текущий процесс

- Репозитории хранятся в GitHub организации Neetrino
- Нет связи между проектами в CRM/PM и репозиториями
- PM не видит активность разработки без захода в GitHub

### Планируемая интеграция

| Функция | Описание |
|---------|----------|
| **Привязка репозитория** | Проект → один или несколько репозиториев GitHub |
| **Последние коммиты** | В карточке проекта отображаются 5–10 последних коммитов |
| **Статус CI/CD** | Статус последнего билда / деплоя (зелёный / красный) |
| **Pull Requests** | Открытые PR видны в карточке проекта |
| **Ссылки** | Прямые ссылки на репозиторий, PR, issues |

### Техническая реализация

```
NBOS ←→ GitHub API (REST / GraphQL)
  - OAuth App или GitHub App
  - Webhooks для событий (push, PR, CI status)
```

| Аспект | Детали |
|--------|--------|
| **API** | GitHub REST API v3 / GraphQL API v4 |
| **Авторизация** | GitHub App (предпочтительно) или OAuth |
| **Webhook-события** | push, pull_request, status, deployment |
| **Отображение** | Виджет в карточке проекта |

### Фаза реализации

| Этап | Описание |
|------|----------|
| **MVP** | Поле «GitHub URL» в карточке проекта (просто ссылка) |
| **Фаза 2** | Виджет с последними коммитами и открытыми PR |
| **Фаза 3** | Webhooks, автоматические обновления, привязка коммитов к задачам |

## Государственная система счетов (Налоговая)

### Назначение

Интеграция с государственной системой выставления счетов для автоматизации процесса создания официальных счетов-фактур.

### Текущий процесс

```
1. Invoice создаётся в NBOS (стадия "Create in Gov System")
2. Бухгалтер получает уведомление
3. Бухгалтер вручную открывает государственную программу
4. Создаёт счёт-фактуру с указанием реквизитов, суммы, НДС
5. Возвращается в NBOS и подтверждает: «Создано в госсистеме»
6. Invoice переходит на следующую стадию
```

### Планируемая интеграция (если API доступен)

| Функция | Описание |
|---------|----------|
| **Auto-create** | При переходе Invoice на стадию «Create in Gov System» → автоматическое создание через API |
| **Данные** | Реквизиты компании, сумма, НДС, описание — берутся из Invoice |
| **Подтверждение** | После успешного создания → автоматический переход на следующую стадию |
| **Номер** | Номер госсчёта сохраняется в Invoice для учёта |

### Техническая реализация

| Аспект | Детали |
|--------|--------|
| **Зависимость** | Наличие API у государственной системы (Армения) |
| **Формат** | REST API или SOAP (зависит от системы) |
| **Данные** | ИНН/НЗОУ компании, сумма, НДС, описание услуги |
| **Безопасность** | Сертификаты / токены для авторизации |

### Фаза реализации

| Этап | Описание |
|------|----------|
| **MVP** | Ручной процесс: уведомление бухгалтеру + подтверждение в NBOS |
| **Фаза 3+** | Автоматическое создание через API (если API доступен) |

## Google Calendar

### Назначение

Синхронизация встреч и событий из модуля Calendar в NBOS с Google Calendar сотрудников.

### Текущий процесс

- Встречи создаются отдельно в Google Calendar и в рабочих инструментах
- Нет единого календаря, объединяющего рабочие встречи и проектные события
- Пропущенные встречи из-за рассинхронизации

### Планируемая интеграция

| Функция | Описание |
|---------|----------|
| **Синхронизация встреч** | Встреча в NBOS Calendar → автоматически появляется в Google Calendar сотрудника |
| **Обратная синхронизация** | Событие из Google Calendar → отображается в NBOS Calendar (как busy time) |
| **Участники** | При добавлении участников встречи — Google Calendar invite отправляется |
| **Напоминания** | Стандартные Google Calendar напоминания работают для событий из NBOS |

### Техническая реализация

```
NBOS Calendar ←→ Google Calendar API (v3)
  - OAuth 2.0 для каждого сотрудника
  - Каждый сотрудник привязывает свой Google-аккаунт
  - События синхронизируются через API
```

| Аспект | Детали |
|--------|--------|
| **API** | Google Calendar API v3 |
| **Авторизация** | OAuth 2.0 (per-user) |
| **Синхронизация** | Webhook (push notifications) + периодическая |
| **Conflict resolution** | NBOS-событие имеет приоритет; Google-события как read-only |

### Фаза реализации

| Этап | Описание |
|------|----------|
| **MVP** | Встроенный календарь в NBOS (без Google-синхронизации) |
| **Фаза 2** | Одностороняя синхронизация: NBOS → Google Calendar |
| **Фаза 3** | Двусторонняя синхронизация + free/busy check |

## Instagram / Facebook (Meta Business)

### Назначение

Приём входящих сообщений и лидов из Instagram и Facebook. Клиенты часто обращаются в Neetrino через эти каналы, и входящие сообщения должны попадать в CRM.

### Текущий процесс

- Клиенты пишут в Instagram Direct / Facebook Messenger
- Продажники проверяют сообщения в приложениях Instagram/Facebook
- Лиды создаются в CRM вручную
- Переписка не сохраняется в системе

### Планируемая интеграция

| Функция | Описание |
|---------|----------|
| **Входящие сообщения** | Сообщения из Instagram Direct и Facebook Messenger → отображаются в NBOS (раздел «Коммуникации с клиентами») |
| **Автоматическое создание лида** | Новое сообщение от неизвестного контакта → создаётся Lead в CRM |
| **Привязка к контакту** | Если контакт известен → сообщение привязывается к существующему Contact |
| **Ответ из NBOS** | Сотрудник отвечает из NBOS → ответ отправляется обратно в Instagram/Facebook |
| **Источник лида** | Автоматическая пометка: Lead.source = «Instagram» / «Facebook» |

### Техническая реализация

```
Instagram / Facebook Messenger
        ↓
  Meta Business API (Webhook)
        ↓
  NBOS → Раздел «Коммуникации с клиентами»
        ↓
  Привязка к Contact / создание Lead
```

| Аспект | Детали |
|--------|--------|
| **API** | Meta Business API (Instagram Messaging API + Messenger API) |
| **Авторизация** | Facebook App + Page Access Token |
| **Webhook** | Входящие сообщения приходят через webhook |
| **Ограничения** | 24-часовое окно для ответа (правила Meta), шаблоны для инициации |
| **Медиа** | Поддержка изображений, голосовых сообщений, stories-replies |

### Поддерживаемые каналы

| Канал | Тип сообщений | Приоритет |
|-------|---------------|-----------|
| Instagram Direct | Текст, изображения, stories-replies, reels-shares | Высокий |
| Facebook Messenger | Текст, изображения, вложения | Средний |
| Facebook Comments | Комментарии к постам (опционально) | Низкий |
| Instagram Comments | Комментарии к постам (опционально) | Низкий |

### Фаза реализации

| Этап | Описание |
|------|----------|
| **MVP** | Ручное создание лидов из Instagram/Facebook |
| **Фаза 2** | Входящие сообщения отображаются в NBOS + автоматическое создание лидов |
| **Фаза 3** | Ответы из NBOS обратно в Instagram/Facebook + полная переписка |

## Приоритеты реализации

```
Приоритет 1 (MVP):
  → WhatsApp (клиентские уведомления)
  → Telegram (командные уведомления)

Приоритет 2 (Фаза 2):
  → Банк (импорт выписок)
  → Instagram/Facebook (входящие сообщения)

Приоритет 3 (Фаза 3+):
  → Google Drive (синхронизация)
  → Google Calendar (синхронизация)
  → GitHub (виджеты, webhooks)
  → Госсистема (API, если доступно)
```
