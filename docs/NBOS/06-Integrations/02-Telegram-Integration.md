# Интеграция с Telegram

## Общее описание

Telegram-интеграция в NBOS предназначена для **внутренней командной коммуникации и уведомлений**. В 99% случаев Telegram используется только внутри команды Neetrino — для общения с клиентами применяется WhatsApp.

Интеграция реализуется через **Telegram Bot API** и включает уведомления сотрудникам, а в перспективе — синхронизацию проектных чатов.

---

## Назначение и границы

| Аспект | Описание |
|--------|----------|
| **Основная цель** | Уведомления сотрудникам о задачах, дедлайнах, событиях |
| **Вторичная цель** | Проектные групповые чаты команды (синхронизация с платформой) |
| **НЕ используется для** | Клиентской коммуникации (для этого — WhatsApp) |
| **Тип подключения** | Telegram Bot API |
| **Направление** | Исходящие уведомления (основное) + двусторонняя синхронизация (будущее) |

---

## Подходы к интеграции

Рассматриваются три подхода к интеграции с Telegram. Рекомендуется **Подход C (гибридный)** — начать с простого и развивать по мере необходимости.

### Подход A: Telegram как канал уведомлений

**Описание:** Telegram-бот отправляет уведомления сотрудникам в личные чаты или каналы отделов. Односторонняя коммуникация — из NBOS в Telegram.

| Параметр | Значение |
|----------|----------|
| **Сложность разработки** | Низкая |
| **Стоимость** | Минимальная (Telegram Bot API бесплатен) |
| **Охват функциональности** | Только уведомления |
| **Время реализации** | 1–2 недели |

**Как работает:**
```
Событие в NBOS → Определение получателя → Формирование текста
  → Telegram Bot API → Сообщение в личный чат сотрудника
```

**Плюсы:**
- Быстрая реализация
- Низкий порог входа для команды (все уже используют Telegram)
- Не требует сложной инфраструктуры
- Высокая надёжность (API Telegram стабилен)

**Минусы:**
- Только односторонняя коммуникация
- Нет возможности ответить из Telegram и чтобы ответ попал в NBOS
- Не заменяет проектные чаты

### Подход B: Telegram как внутренний мессенджер

**Описание:** Telegram-группы привязаны к проектам в NBOS. Сообщения из Telegram-группы отображаются в проектном чате платформы и наоборот — двусторонняя синхронизация.

| Параметр | Значение |
|----------|----------|
| **Сложность разработки** | Высокая |
| **Стоимость** | Значительная (разработка + поддержка) |
| **Охват функциональности** | Полный мессенджер + уведомления |
| **Время реализации** | 4–8 недель |

**Как работает:**
```
Сообщение в Telegram-группе проекта
  → Webhook / Long Polling → NBOS
  → Отображается в проектном чате NBOS
  
Сообщение в проектном чате NBOS
  → Telegram Bot API → Пересылка в Telegram-группу проекта
```

**Структура Telegram-группы проекта:**
- Одна группа на проект с **Topics (темами)**:
  - General — общие обсуждения
  - Development — вопросы разработки
  - Design — вопросы дизайна
  - Bugs — баги и проблемы
- Участники: PM, разработчики, дизайнеры, назначенные на проект

**Плюсы:**
- Команда может работать из привычного Telegram
- Вся переписка дублируется в NBOS (история сохраняется)
- Удобно для мобильного использования

**Минусы:**
- Сложная разработка и поддержка синхронизации
- Возможны конфликты и задержки при синхронизации
- Telegram Bot API имеет ограничения для групповых чатов
- Требует управления правами доступа в двух системах

### Подход C: Гибридный (Рекомендуемый)

**Описание:** Поэтапная реализация — сначала простые уведомления, затем интеграция групповых чатов.

| Фаза | Что реализуется | Когда |
|------|-----------------|-------|
| **Фаза 1** | Telegram-бот для уведомлений (Подход A) | MVP / Фаза 1 платформы |
| **Фаза 2** | Привязка Telegram-групп к проектам (Подход B) | Фаза 3+ платформы |

**Преимущества:**
- Быстрый старт с минимальными затратами
- Команда сразу получает ценность (уведомления)
- Решение о Фазе 2 принимается по результатам использования Фазы 1
- Если Фаза 2 окажется не нужна — ресурсы не потрачены

## Фаза 1: Telegram-бот для уведомлений

### Привязка аккаунта

Каждый сотрудник привязывает свой Telegram-аккаунт в профиле NBOS:

```
1. Сотрудник открывает свой профиль в NBOS → раздел "Интеграции"
2. Нажимает "Привязать Telegram"
3. Открывается ссылка на бота: t.me/NeetrinoNBOS_bot
4. Сотрудник отправляет боту команду /start
5. Бот отправляет код верификации (или использует deep linking)
6. NBOS привязывает Telegram chat_id к профилю сотрудника
7. Готово — уведомления будут приходить в личный чат с ботом
```

| Поле в профиле | Описание |
|----------------|----------|
| **telegram_chat_id** | Идентификатор чата с ботом (заполняется автоматически) |
| **telegram_username** | Username в Telegram (опционально) |
| **telegram_notifications** | Вкл/Выкл уведомления (по умолчанию: Вкл) |
| **telegram_linked_at** | Дата привязки аккаунта |

### Полный список уведомлений

#### Задачи

| Событие | Текст уведомления | Получатель |
|---------|-------------------|------------|
| Задача назначена на вас | «📋 Вам назначена задача: {task_name}\nПроект: {project}\nДедлайн: {deadline}\nПриоритет: {priority}» | Исполнитель задачи |
| Дедлайн задачи завтра | «⏰ Завтра дедлайн задачи: {task_name}\nПроект: {project}\nОсталось: 1 день» | Исполнитель задачи |
| Задача просрочена | «🔴 Задача просрочена: {task_name}\nПроект: {project}\nПросрочка: {days} дн.» | Исполнитель + PM проекта |
| Статус задачи изменён | «🔄 Задача {task_name} → {new_status}\nИзменил: {user}» | Наблюдатели задачи |

#### Проекты

| Событие | Текст уведомления | Получатель |
|---------|-------------------|------------|
| Дедлайн проекта приближается | «⚠️ Дедлайн проекта {project_name} через {days} дн.\nТекущий статус: {status}» | PM проекта |
| Новый продукт добавлен | «📦 Новый продукт в проекте {project}: {product_name}\nТип: {product_type}» | PM проекта |
| Все задачи продукта выполнены | «✅ Все задачи продукта {product_name} выполнены\nПроект: {project}\nЗапустите чек-лист закрытия.» | PM проекта |

#### Поддержка

| Событие | Текст уведомления | Получатель |
|---------|-------------------|------------|
| Новый тикет поддержки | «🎫 Новый тикет: {ticket_title}\nКлиент: {client}\nПриоритет: {priority}» | PM + Support-команда |
| SLA 75% израсходовано | «⚠️ SLA тикета {ticket_title} — осталось 25% времени\nДедлайн SLA: {sla_deadline}» | Ответственный за тикет |
| SLA нарушен | «🚨 SLA НАРУШЕН: тикет {ticket_title}\nКлиент: {client}\nТребуется эскалация!» | PM + CEO |

#### Финансы

| Событие | Текст уведомления | Получатель |
|---------|-------------------|------------|
| Сделка выиграна | «🎉 Сделка выиграна! {deal_name}\nСумма: {amount}\nКлиент: {client}» | Seller + PM + CEO |
| Бонус начислен | «💰 Бонус начислен: {amount}\nЗа: {reason}\nПроект: {project}» | Сотрудник |
| Оплата от клиента получена | «✅ Оплата получена: {amount} от {client}\nСчёт: #{invoice_number}» | Финансовый директор |

#### Спринты

| Событие | Текст уведомления | Получатель |
|---------|-------------------|------------|
| Спринт завершается | «📊 Спринт {sprint_name} завершается {end_date}\nНезавершённых задач: {count}» | PM проекта |

### Каналы отделов (опционально)

Помимо личных уведомлений, бот может отправлять сообщения в Telegram-каналы отделов:

| Канал | Что публикуется |
|-------|-----------------|
| **#sales** | Новые лиды, выигранные сделки, квартальные итоги |
| **#support** | Новые тикеты, SLA-нарушения |
| **#general** | Общие объявления, новые проекты, достижения |
| **#finance** | Оплаты, просрочки (только для финансовой команды) |

## Фаза 2: Интеграция проектных Telegram-групп

### Привязка группы к проекту

```
1. PM создаёт Telegram-группу для проекта (с Topics)
2. Добавляет бота @NeetrinoNBOS_bot в группу как администратора
3. В NBOS → карточка проекта → «Привязать Telegram-группу»
4. Бот отправляет код подтверждения в группу
5. PM подтверждает привязку в NBOS
6. Синхронизация активирована
```

### Синхронизация сообщений

| Направление | Поведение |
|-------------|-----------|
| **Telegram → NBOS** | Сообщения из группы отображаются в проектном чате NBOS |
| **NBOS → Telegram** | Сообщения из проектного чата пересылаются в Telegram-группу |
| **Файлы** | Файлы из Telegram сохраняются в NBOS Drive проекта |
| **Упоминания** | @username в Telegram → уведомление в NBOS (и наоборот) |

### Управление участниками

| Действие | Поведение |
|----------|-----------|
| Сотрудник добавлен в проект в NBOS | Автоматически приглашается в Telegram-группу (если привязан аккаунт) |
| Сотрудник удалён из проекта | Удаляется из Telegram-группы |
| Новый участник в Telegram-группе | Отображается в NBOS как участник проекта (опционально) |

### Технические ограничения Фазы 2

| Ограничение | Описание |
|-------------|----------|
| Bot API vs User API | Bot API ограничен — не все функции групп доступны. Для полной интеграции может потребоваться Telegram User API (TDLib), что сложнее |
| Topics | Поддержка Topics в Bot API появилась недавно — могут быть ограничения |
| Медиа-контент | Фото/видео синхронизируются с задержкой и ограничениями по размеру |
| Редактирование сообщений | Отредактированные сообщения в Telegram должны обновляться в NBOS (и наоборот) |

## Настройки уведомлений для сотрудника

Каждый сотрудник может настроить свои уведомления в профиле:

| Настройка | Описание | По умолчанию |
|-----------|----------|--------------|
| **Уведомления о задачах** | Назначение, дедлайн, просрочка | Вкл |
| **Уведомления о проектах** | Дедлайны проекта, изменения статуса | Вкл |
| **Уведомления о поддержке** | Новые тикеты, SLA | Вкл (для PM/Support) |
| **Финансовые уведомления** | Бонусы, оплаты | Вкл |
| **Режим «Не беспокоить»** | Блокировка уведомлений в нерабочее время | Выкл |
| **Время «Не беспокоить»** | С какого по какое время не беспокоить | 22:00–08:00 |

### Приоритеты уведомлений

| Приоритет | Поведение | Примеры |
|-----------|-----------|---------|
| **Критический** | Отправляется всегда, игнорирует «Не беспокоить» | SLA нарушен, задача критического приоритета просрочена |
| **Высокий** | Отправляется в рабочее время, может игнорировать тихий режим | Дедлайн завтра, новый тикет высокого приоритета |
| **Обычный** | Отправляется в рабочее время, уважает тихий режим | Задача назначена, статус изменён |
| **Низкий** | Собирается в дайджест (1 раз в день) | Наблюдения за задачами, информационные обновления |

## Техническая реализация

### Telegram Bot

| Параметр | Описание |
|----------|----------|
| **Название** | NeetrinoNBOS Bot |
| **Username** | @NeetrinoNBOS_bot (пример) |
| **Создание** | Через @BotFather |
| **API** | Telegram Bot API (https://api.telegram.org/bot{token}/) |
| **Метод получения обновлений** | Webhook (предпочтительно) или Long Polling |

### API-вызовы

| Действие | Метод API |
|----------|-----------|
| Отправка уведомления | `sendMessage` с chat_id сотрудника |
| Отправка в группу | `sendMessage` с chat_id группы (+ message_thread_id для Topics) |
| Управление группой | `banChatMember`, `unbanChatMember`, `inviteChatMember` |
| Получение обновлений | Webhook → endpoint NBOS |

### Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Сотрудник заблокировал бота | Пометить telegram_linked = false, уведомить PM |
| Telegram API недоступен | Повторная попытка через 5 мин (до 3 попыток), затем fallback на email |
| Группа удалена | Убрать привязку к проекту, уведомить PM |
| Rate limit | Очередь сообщений с задержкой (Telegram: 30 сообщений/сек для бота) |
