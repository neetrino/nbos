# Страницы Tasks и Support

## 1. Общее описание

Модули Tasks и Support обеспечивают операционное управление работой команды: задачи — для внутренней разработки и доставки продуктов, поддержка — для обработки клиентских обращений и инцидентов. Оба модуля тесно интегрированы с Projects Hub и между собой.

**Доступ Tasks:**
- CEO, Head of Delivery — полный доступ ко всем задачам
- PM — задачи по своим проектам
- Developer, QA, Designer — назначенные задачи и задачи их проектов

**Доступ Support:**
- CEO, Head of Delivery — полный доступ
- PM — тикеты по своим проектам
- Support Manager — полный доступ к тикетам
- Developer, QA — назначенные тикеты

## 2. My Plan — Личный план задач

**Путь:** `/tasks/my`

### 2.1. Структура страницы

Страница разделена на временные секции, каждая визуально отделена и имеет свой цветовой акцент:

| Секция | Цвет | Описание |
|--------|------|----------|
| Overdue | Красный фон/полоса | Просроченные задачи (дедлайн прошёл) |
| Due Today | Оранжевый фон | Задачи с дедлайном сегодня |
| Due This Week | Синий фон | Задачи на текущую неделю |
| Upcoming | Серый фон | Задачи с дедлайном на следующую неделю и далее |
| No Date | Светло-серый | Задачи без установленного дедлайна |

Каждая секция показывает количество задач в бейдже заголовка.

### 2.2. Карточка задачи в My Plan

Каждая задача в списке отображает:

| Элемент | Описание |
|---------|----------|
| Чекбокс | Быстрая отметка как «Done» |
| Заголовок | Название задачи (кликабельно → открывает Task Detail) |
| Проект | Название проекта (ссылка) |
| Приоритет бейдж | Low (серый) / Medium (синий) / High (оранжевый) / Critical (красный) |
| Дедлайн | Дата с цветовым индикатором |
| Статус | Текущий статус задачи (To Do / In Progress / Review / Done) |

### 2.3. Действия на странице

- **Сортировка:** по дедлайну (по умолчанию), по приоритету, по проекту
- **Группировка:** по проекту, по приоритету, по статусу
- **Массовые действия:** выбор нескольких задач через чекбоксы:
  - Change Status — смена статуса
  - Reassign — переназначить
  - Set Deadline — установить/изменить дедлайн
  - Change Priority — сменить приоритет

## 3. Task Detail — Детальная панель задачи

### 3.1. Открытие

Клик по задаче из любого списка или доски открывает **слайд-панель справа** (slide-in panel). Панель занимает ~60% экрана. Фон слева затемняется, но остаётся кликабельным для закрытия.

### 3.2. Верхняя часть — контекст

- **Хлебные крошки:** Project Name > Product Name > Task Title
- **Кнопка закрытия** (×) в правом верхнем углу
- **Кнопка открытия на полный экран** (expand)

### 3.3. Левая часть — информация о задаче

**Заголовок задачи:**
- Редактируемое поле (inline edit)
- Под заголовком — ID задачи (например, TASK-1205)

**Основные поля:**

| Поле | Тип | Описание |
|------|-----|----------|
| Status | Selector | Выпадающий список: To Do → In Progress → Review → Done |
| Priority | Selector | Low / Medium / High / Critical |
| Due Date | Date Picker | Календарь для выбора даты |
| Assignee | User Picker | Основной исполнитель |
| Co-assignees | Multi User Picker | Дополнительные исполнители |
| Observers | Multi User Picker | Наблюдатели (получают уведомления) |
| Creator | Read-only | Кто создал задачу |
| Sprint | Selector | К какому спринту относится (если Scrum) |
| Estimated Hours | Number | Оценка трудозатрат |

**Описание:**
- Rich-text поле с поддержкой Markdown
- Вставка изображений, ссылок, таблиц
- @mentions сотрудников

**Чеклист (Subtasks):**
- Вложенный список пунктов с чекбоксами
- Drag & drop для перестановки
- Процент выполнения
- Кнопка `+ Add Item`

**Вложения (Attachments):**
- Drag & drop файлов
- Превью изображений и документов
- Список файлов с размером и датой

**Связанные задачи (Linked Tasks):**
- Блокирующие задачи (Blocks / Blocked by)
- Связанные задачи (Related to)
- Кнопка `+ Link Task`

### 3.4. Правая часть — чат-панель

Встроенный чат прямо в задаче:

- **Лента сообщений:** хронологическая лента комментариев
- **Системные события:** вплетены в ленту (изменение статуса, назначение, перенос дедлайна)
- **Файлы в чате:** возможность прикрепить файл к комментарию
- **@mentions:** упоминание сотрудника отправляет ему уведомление
- **Форматирование:** поддержка Markdown в сообщениях
- **Реакции:** emoji-реакции на комментарии

### 3.5. Быстрые действия (Quick Actions)

Панель действий в верхней или нижней части:

| Действие | Описание |
|----------|----------|
| Change Status | Быстрая смена статуса одной кнопкой (Next Status) |
| Add Subtask | Создать подзадачу |
| Add to Sprint | Добавить в спринт (при Scrum) |
| Duplicate | Дублировать задачу |
| Delete | Удалить задачу (с подтверждением) |
| Move to Project | Переместить в другой проект/продукт |

### 3.6. Участники задачи

Блок участников (отображается сверху или сбоку):

| Роль | Описание | Возможности |
|------|----------|------------|
| Creator | Кто создал задачу | Может редактировать, удалять |
| Assignee | Основной исполнитель | Может менять статус, комментировать |
| Co-assignees | Со-исполнители | Могут менять статус, комментировать |
| Observers | Наблюдатели | Получают уведомления, могут комментировать |

Все роли редактируемые (с учётом прав текущего пользователя).

## 4. Task Board — Доска задач внутри продукта

**Путь:** `/projects/:projectId/products/:productId/tasks`

### 4.1. Шкала стадий продукта (Stage Progress Bar)

В верхней части доски — горизонтальная шкала стадий продукта:

```
[New] → [Creating] → [Development] → [QA] → [Transfer] → [Done] → [Maintenance]
```

- Текущая стадия выделена цветом и увеличена
- Пройденные стадии — заполнены (с галочками)
- Будущие стадии — контурные
- Клик по стадии (для PM/Head of Delivery): переключает стадию с подтверждением

### 4.2. Переключатель Scrum / Kanban

Под шкалой стадий — переключатель:

```
[Scrum] [Kanban]
```

### 4.3. Kanban-вид

**Колонки** = статусы задач:

| Колонка | Описание |
|---------|----------|
| To Do | Запланированные задачи |
| In Progress | В работе |
| Review | На проверке (code review, QA) |
| Done | Завершённые |

**Карточка задачи на доске:**

| Элемент | Расположение | Описание |
|---------|-------------|----------|
| Заголовок | Верх карточки | Название задачи (кликабельно) |
| Аватар исполнителя | Правый нижний угол | Маленький аватар Assignee |
| Приоритет | Левый край (цветная полоса) | Цветовой индикатор приоритета |
| Дедлайн | Нижний левый угол | Дата (красная если просрочена) |
| Теги | Под заголовком | Метки (bug, feature, improvement и т.д.) |
| Прогресс чеклиста | Нижняя часть | Мини-бар: 3/5 выполнено |

**Drag & Drop:** перетаскивание карточек между колонками для смены статуса.

### 4.4. Scrum-вид

**Текущий спринт:**
- Название спринта (например, «Sprint 12»)
- Даты: дата начала — дата окончания
- Прогресс-бар: X из Y задач выполнено

**Задачи спринта:**
- Таблица или список задач текущего спринта с колонками: задача, статус, исполнитель, оценка, фактическое время

**Burndown Chart:**
- Линейный график: идеальная линия vs фактическая
- Ось X: дни спринта
- Ось Y: оставшийся объём работы (часы или story points)

**Sprint Planning:**
- Кнопка `Create Sprint` — создание нового спринта (название, даты начала и окончания)
- Пул задач (Backlog) слева — задачи, не включённые в спринт
- Перетаскивание задач из Backlog в Sprint
- Автоматический подсчёт ёмкости: сумма оценок vs запланировано

## 5. All Tasks — Все задачи

**Путь:** `/tasks/all`

Доступен PM и Head of Delivery. Показывает все задачи по всем проектам.

### 5.1. Виды

- **List:** таблица с полными фильтрами и сортировкой
- **Kanban:** доска по статусам (с фильтром по проекту)
- **Calendar:** задачи на календаре по дедлайнам

### 5.2. Фильтры

- По проекту, по продукту, по исполнителю, по статусу, по приоритету, по спринту, по дедлайну (диапазон)
- Сохранение фильтров как presets

## 6. Support — Тикеты поддержки

### 6.1. Страница тикетов (Tickets)

**Путь:** `/support/tickets`

### 6.2. Kanban-доска

Колонки = стадии тикета:

| Колонка | Описание |
|---------|----------|
| New | Новый тикет, ещё не обработан |
| Triaged | Классифицирован (категория + приоритет назначены) |
| Assigned | Назначен исполнителю |
| In Progress | В работе |
| Resolved | Решён (ожидает подтверждения) |
| Closed | Закрыт |

### 6.3. Карточка тикета на доске

| Элемент | Описание |
|---------|----------|
| Заголовок | Название тикета |
| Проект | Связанный проект (ссылка) |
| Категория бейдж | Bug / Feature Request / Question / Incident / Change Request |
| Приоритет бейдж | Low / Medium / High / Critical |
| SLA индикатор | Цветной кружок: зелёный (в пределах SLA), жёлтый (приближается), красный (нарушен) |
| Исполнитель | Аватар назначенного сотрудника |
| Время создания | Сколько времени назад создан |

### 6.4. SLA-таймер

На каждой карточке виден SLA-таймер:
- **Зелёный:** до нарушения SLA > 50% времени осталось
- **Жёлтый:** до нарушения SLA < 50% времени
- **Красный (мигающий):** SLA нарушен, отображается время превышения
- Формат: «2ч 15м осталось» или «Превышен на 45м»

### 6.5. Табличный вид

Переключатель: `Kanban | List`

Колонки: ID тикета, заголовок, проект, категория, приоритет, статус, SLA, исполнитель, дата создания, дата обновления.

### 6.6. Фильтры

- **По проекту:** выбор проекта
- **По категории:** Bug / Feature Request / Question / Incident / Change Request
- **По приоритету:** Low / Medium / High / Critical
- **По статусу:** мультивыбор стадий
- **По исполнителю:** выбор сотрудника
- **По SLA:** In SLA / SLA Warning / SLA Breached

### 6.7. Быстрые действия

- `+ New Ticket` — создание тикета: заголовок, описание, проект, категория, приоритет
- **Classify Ticket** — быстрая классификация: установка категории + приоритета одним действием (из контекстного меню карточки)

## 7. Ticket Detail — Детальная страница тикета

**Путь:** `/support/tickets/:ticketId`

### 7.1. Заголовок

- Название тикета (редактируемое)
- ID тикета (например, TKT-1205)
- Категория бейдж (цветной)
- Приоритет бейдж (цветной)
- SLA Countdown — обратный отсчёт до нарушения SLA (или время нарушения)

### 7.2. Левая часть — описание

| Блок | Описание |
|------|----------|
| Description | Полное описание проблемы (rich-text) |
| Steps to Reproduce | Шаги для воспроизведения (нумерованный список) |
| Expected Behavior | Ожидаемое поведение |
| Actual Behavior | Фактическое поведение |
| Environment | Окружение (браузер, ОС, версия продукта) |
| Attachments | Скриншоты, логи, файлы |

### 7.3. Правая часть — таймлайн

Хронологическая лента событий:
- Изменения статуса (с датой и автором)
- Комментарии сотрудников
- Внутренние заметки (видны только команде, не видны клиенту)
- Системные события (автоназначение, SLA предупреждения)
- Резолюция (когда тикет решён): описание решения

Форма добавления комментария внизу ленты:
- Переключатель: Public (видно клиенту) / Internal (только для команды)
- Вложение файлов
- @mentions

### 7.4. Нижняя часть — связанные сущности

| Сущность | Описание |
|----------|----------|
| Linked Tasks | Задачи, созданные для решения тикета. Кнопка `+ Create Task from Ticket` |
| Linked Extension Deal | Если тикет требует доработки — ссылка на Extension Deal. Кнопка `+ Create Extension Deal` |
| Related Tickets | Похожие или дублирующие тикеты |

### 7.5. Действия

| Действие | Описание |
|----------|----------|
| Assign | Назначить исполнителя |
| Change Status | Переход на следующую стадию |
| Mark as Billable | Отметить как оплачиваемый (если работа требует дополнительного финансирования) |
| Create Extension Deal | Создать сделку-продление в CRM (если тикет = Change Request) |
| Create Task | Создать задачу на основе тикета (привязка к продукту) |
| Close | Закрыть тикет (с обязательным описанием резолюции) |
| Reopen | Переоткрыть тикет (если решение не помогло) |

## 8. Support Dashboard

**Путь:** `/support/dashboard`

### 8.1. Сводные метрики

| Метрика | Описание |
|---------|----------|
| Open Tickets | Общее количество открытых тикетов |

### 8.3. Фильтры

- Период: This Week / This Month / This Quarter / Custom Range
- По проекту, по исполнителю

## 9. Интеграция между Tasks и Support

```
Support Ticket (Change Request)
  ↓ Create Extension Deal
CRM → Extension Deal
  ↓ Won
Project → Extension
  ↓ Tasks
Task Board
```

```
Support Ticket (Bug/Incident)
  ↓ Create Task
Task (linked to ticket)
  ↓ Done
Ticket → Resolved → Closed
```

- Когда задача, созданная из тикета, завершается — тикет автоматически предлагается перевести в «Resolved»
- Когда тикет типа Change Request создаёт Extension Deal — тикет помечается как «Billable» и привязывается к сделке
- Все связи двунаправленные и видны в обеих сущностях

## 10. UX-паттерны модулей Tasks и Support

| Паттерн | Реализация |
|---------|------------|
| Slide-in Panel | Детали задачи/тикета открываются в боковой панели без перехода на новую страницу |
| Real-time Chat | Комментарии в задаче/тикете обновляются в реальном времени |
| Drag & Drop | Перетаскивание на Kanban-досках для смены статуса |
| SLA Visibility | SLA-индикатор виден везде: на доске, в списке, в деталях |
| Quick Status | Один клик для перехода на следующий статус |
| @Mentions | Упоминание коллег в комментариях с автоматическим уведомлением |
| Keyboard Navigation | Стрелки для навигации по списку, Enter для открытия, Esc для закрытия |
| Filters Memory | Фильтры сохраняются между сессиями для каждого пользователя |
| Batch Actions | Массовые операции над задачами/тикетами через чекбоксы |
| Timer | Таймер работы над задачей (Start/Stop/Pause) для трекинга времени |
