# Страницы CRM модуля

## 1. Общее описание

CRM-модуль платформы NBOS охватывает полный цикл продаж: от первого контакта (лида) через квалификацию до закрытия сделки. Модуль состоит из трёх основных страниц:

1. **Lead List** — управление лидами
2. **Deal Pipeline** — воронка сделок
3. **Sales Analytics** — аналитика продаж

Доступ к CRM имеют: CEO, Head of Sales, Seller. Finance Director и Head of Delivery имеют ограниченный доступ к аналитике и сделкам (только просмотр).

---

## 2. Страница лидов (Lead List)

**Путь:** `/crm/leads`

### 2.1. Основной вид — Kanban-доска

По умолчанию страница отображается в виде Kanban-доски. Колонки соответствуют стадиям лида:

**Активные стадии (перетаскиваемые):**
| Колонка | Описание |
|---------|----------|
| New | Новый лид, ещё не обработан |
| Didn't Get Through | Не дозвонились / не связались |
| Contact Established | Контакт установлен, идёт первичное общение |
| Qualification MQL | Квалификация маркетингового лида |

**Терминальные стадии (закрытые):**
| Колонка | Описание |
|---------|----------|
| SPAM | Спам / нецелевой лид |
| Frozen | Заморожен (клиент не готов, вернуться позже) |
| SQL | Квалифицированный лид → конвертирован в сделку |

Терминальные колонки визуально отделены (серый фон, свёрнутые по умолчанию). При перемещении карточки в SQL автоматически предлагается создание сделки (Deal).

### 2.2. Карточка лида на доске

Каждая карточка отображает:

- **Имя контакта** — основной текст карточки
- **Телефон** — под именем, мелким шрифтом
- **Иконка источника** — визуальная иконка (Instagram, Web, Referral, Cold Call и т.д.)
- **Дата создания** — в правом нижнем углу
- **Аватар продавца** — маленький аватар назначенного Seller'а
- **Цветовая метка** — если лид «горячий» или требует внимания

### 2.3. Табличный вид (List View)

Переключатель в правом верхнем углу: `Kanban | List`

Табличный вид содержит сортируемые колонки:
| Колонка | Сортировка | Описание |
|---------|-----------|----------|
| Имя контакта | ✅ | ФИО или имя лида |
| Телефон | ❌ | Номер телефона |
| Email | ❌ | Email-адрес |
| Источник | ✅ | Канал привлечения |
| Стадия | ✅ | Текущая стадия лида |
| Продавец | ✅ | Назначенный Seller |
| Дата создания | ✅ | Когда лид был создан |
| Последняя активность | ✅ | Дата последнего действия |

### 2.4. Фильтры

Панель фильтров располагается над доской/таблицей:

- **По источнику:** выпадающий список (мультивыбор)
- **По продавцу:** выпадающий список сотрудников отдела продаж
- **По дате:** диапазон дат (date range picker)
- **По стадии:** мультивыбор стадий
- Кнопка «Сбросить фильтры»
- Сохранение пользовательских фильтров (presets)

### 2.5. Действия

**Быстрые действия (кнопки в заголовке):**

- `+ New Lead` — открывает модальное окно создания лида (имя, телефон, email, источник, комментарий)
- `Convert to SQL` — доступно при выделении лида; конвертирует в квалифицированный лид и создаёт Deal

**Массовые действия (bulk actions):**
При выборе нескольких карточек (через чекбоксы в List View) доступны:

- **Mark as SPAM** — массовое перемещение в спам
- **Assign to Seller** — массовое назначение продавца
- **Change Stage** — массовая смена стадии

## 3. Страница воронки сделок (Deal Pipeline)

**Путь:** `/crm/deals`

Правила обязательных полей, cumulative validation и popup переходов описаны в `../02-Modules/01-CRM/05-Deal-Stage-Gates-and-Won-Override.md`.

### 3.1. Основной вид — Kanban-доска

Колонки соответствуют **7 рабочим стадиям** воронки сделки плюс терминальные (см. `03-Deal-Pipeline.md`):

**Стадии сделки (рабочие):**

1. Start a Conversation
2. Discuss What Is Needed
3. Meeting
4. Can We Do It?
5. Send Offer
6. Get Answer
7. **Deposit & Contract** (последняя рабочая; далее не стадии CRM)

**Терминальные:**

- **Failed** — сделка проиграна (красный фон)
- **Deal Won** — условия Deposit & Contract выполнены; сделка закрыта, работа продолжается в Order / Projects / Finance (Creating, остатки оплат, Maintenance — **не** колонки CRM)

**Не отображаются в воронке Deal:** Creating, Get Final Pay, Maintenance Offer (ведутся в других модулях после Won).

### 3.2. Карточка сделки на доске

Каждая карточка отображает:

- **Имя контакта** — основной текст
- **Компания** — под именем
- **Сумма сделки** — крупным шрифтом, выделена цветом
- **Бейдж типа оплаты** — Upfront / 50/50 / Subscription / Milestone (цветные метки)
- **Дедлайн** — дата с индикатором (красный если просрочен)
- **Аватар продавца** — маленький аватар
- **Тег Extension** — если это сделка-продление, отображается специальный тег

### 3.3. Extension-сделки

Extension-сделки (продление существующих проектов) визуально отличаются:

- **Цвет карточки:** отличается от обычных сделок (например, синий фон вместо белого)
- **Иконка:** значок связи/продления рядом с названием
- **Тег:** метка «Extension» на карточке
- **Связь:** ссылка на исходный проект видна на карточке

### 3.4. Табличный вид (List View)

Переключатель: `Kanban | List`

Сортируемые колонки:
| Колонка | Описание |
|---------|----------|
| Название сделки | Автогенерируемое или ручное |
| Контакт | Имя контактного лица |
| Компания | Название компании |
| Сумма | Общая сумма сделки |
| Тип оплаты | Upfront / 50/50 / Subscription / Milestone |
| Тип сделки | New / Extension |
| Стадия | Текущая стадия |
| Продавец | Назначенный Seller |
| Дедлайн | Планируемая дата закрытия |
| Дата создания | Когда сделка была создана |

### 3.5. Фильтры

- **По продавцу:** выпадающий список
- **По типу сделки:** New / Extension
- **По типу оплаты:** Upfront / 50/50 / Subscription / Milestone
- **По сумме:** диапазон (от — до)
- **По дате:** диапазон дат создания или дедлайна
- **По стадии:** мультивыбор

### 3.6. Действия

- `+ New Deal` — создание новой сделки (модальное окно: контакт, компания, сумма, тип оплаты, описание)
- `+ Extension Deal` — создание сделки-продления (требует выбор существующего проекта)

## 4. Карточка сделки (Deal Card)

**Путь:** `/crm/deals/:dealId`

Открывается при клике на сделку. Полноэкранная страница или слайд-панель.

### 4.1. Заголовок

- Название сделки
- Бейдж текущей стадии (цветной)
- Сумма сделки (крупным шрифтом)
- Имя продавца (Seller)
- Бейдж типа сделки (New / Extension)

### 4.2. Левая часть — Детали сделки

| Поле            | Описание                                   |
| --------------- | ------------------------------------------ |
| Контакт         | Ссылка на контактное лицо                  |
| Компания        | Ссылка на компанию                         |
| Scope           | Описание объёма работ (текстовое поле)     |
| Тип оплаты      | Upfront / 50/50 / Subscription / Milestone |
| Ссылка на оффер | URL документа-предложения                  |
| Дедлайн         | Планируемая дата закрытия                  |
| Заметки         | Свободное текстовое поле                   |

### 4.3. Правая часть — Таймлайн

Хронологическая лента событий:

- История переходов между стадиями (с датой и автором)
- Комментарии сотрудников
- Действия (звонки, встречи, отправка оффера)
- Системные события (автоматические)

Возможность добавить комментарий или действие через форму внизу таймлайна.

### 4.4. Нижняя часть — Связанные сущности

- **Invoice** — связанный счёт (если создан), ссылка на карточку счёта
- **Project** — связанный проект (для Extension-сделок), ссылка на проект
- **Documents** — прикреплённые файлы (контракты, NDA, офферы)

### 4.5. Действия на карточке

| Действие           | Описание                                                               |
| ------------------ | ---------------------------------------------------------------------- |
| Move to Next Stage | Перевод на следующую стадию с возможным заполнением обязательных полей |
| Create Invoice     | Создание счёта на основе сделки                                        |
| Create Offer       | Генерация документа-предложения                                        |
| Schedule Meeting   | Планирование встречи (интеграция с Calendar)                           |
| Mark as Won        | Закрытие как выигранная                                                |
| Mark as Failed     | Закрытие как проигранная (с указанием причины)                         |

### 4.6. Stage Gates — визуальные индикаторы перехода

Для каждой стадии отображается индикатор готовности перехода:

- **Зелёная галочка** — все условия выполнены, можно перейти
- **Жёлтое предупреждение** — есть невыполненные условия, но переход возможен
- **Красный блок** — обязательные условия не выполнены, переход заблокирован

Условия перехода зависят от стадии (например, для «Advance Payment» нужен выставленный счёт).

## 5. Страница аналитики продаж (Sales Analytics)

**Путь:** `/crm/analytics`

### 5.1. Фильтры (верхняя панель)

- **Период:** предустановленные (This Week / This Month / This Quarter / This Year) + произвольный диапазон
- **Продавец:** все / конкретный Seller
- **Тип сделки:** все / New / Extension

### 5.2. Дашборд-виджеты

**Воронка конверсий (Pipeline Funnel):**

- Визуальная воронка: Leads → MQL → SQL → Deals → Won
- Показывает количество и процент конверсии на каждом этапе
- Кликабельные сегменты (ведут на отфильтрованные списки)

**Выручка по неделям/месяцам (Revenue Chart):**

- Столбчатая диаграмма с разбивкой по неделям или месяцам
- Переключатель периода: Weekly / Monthly
- Линия тренда

**Таблица конверсий (Conversion Rates):**

- Строки: стадии воронки
- Колонки: количество, конверсия в %, средняя длительность на стадии
- Цветовая индикация отклонений от целевых показателей

**Сделки по источникам (Deals by Source):**

- Круговая или столбчатая диаграмма
- Разбивка по каналам привлечения
- Показывает количество и суммы

**Тренд среднего чека (Average Deal Size):**

- Линейный график с трендом за выбранный период
- Раздельные линии для New и Extension сделок

**Длительность цикла продаж (Sales Cycle Duration):**

- Средняя длительность от создания лида до закрытия сделки
- Разбивка по стадиям (где «застревают» сделки)
- Сравнение с предыдущим периодом

**Лидерборд продавцов (Seller Leaderboard):**

- Рейтинг продавцов по метрикам:
  - Количество закрытых сделок
  - Общая сумма
  - Конверсия в %
  - Средний чек
- Временной фильтр (текущий месяц / квартал / год)
- Визуальные бары прогресса

## 6. Взаимодействие между страницами CRM

```
Lead (New)
  ↓ квалификация
Lead (SQL)
  ↓ конвертация
Deal (Meeting Scheduled)
  ↓ движение по стадиям
Deal (Won)
  ↓ создание проекта
Project + Product
```

- Конвертация лида в SQL автоматически создаёт Deal и перенаправляет на карточку сделки
- При закрытии сделки как Won предлагается создать проект (если это New Deal) или связать с существующим проектом (если Extension)
- Аналитика агрегирует данные из обеих сущностей (Lead + Deal) для построения полной воронки

| Drag & Drop        | Перетаскивание карточек между стадиями на Kanban-доске                   |

| Real-time Updates  | Изменения других пользователей отображаются в реальном времени           |
| Keyboard Shortcuts | Быстрые клавиши для смены стадии, создания сущностей                     |
| Undo               | Возможность отмены последнего перемещения карточки                       |
| Auto-save          | Автоматическое сохранение при редактировании полей                       |
