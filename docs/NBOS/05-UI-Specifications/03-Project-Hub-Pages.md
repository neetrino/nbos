# Страницы Projects Hub

## 0. Целевая модель навигации (утверждено 2026-03-31)

**Основной рабочий контекст — страница Product**, а не полный набор вкладок на уровне Project. Проект остаётся ключевой сущностью в данных, но **оболочка проекта** в UI в первую очередь ведёт к списку **Product** и далее в **продукто-центричные** экраны.

| Уровень                                    | Назначение                                                                                                                                  |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `/projects`                                | Список проектов (карточки / список) — как раньше по смыслу глобального каталога.                                                            |
| `/projects/:projectId`                     | **Оболочка проекта:** сводка + **список всех Product** в проекте; вход в работу — выбор продукта.                                           |
| `/projects/:projectId/products/:productId` | **Основной функционал** по линии поставки: вкладки вроде Overview, Work Space, Support, Credentials, Finance — **в разрезе этого Product**. |

На странице Product: переключатель между продуктами **этого же** проекта без возврата к списку; кнопка **«Посмотреть всё»** — агрегированный просмотр по всем продуктам проекта (**редкий** сценарий; MVP может обойтись без неё — см. `../../02-Modules/02-Projects-Hub/05-Product-Centric-Navigation.md`).

Разделы **ниже** описывают экраны и вкладки; при расхождении с § 0 приоритет у **продукто-центричной** модели; вкладки на уровне только проекта трактовать как **сводные / переходные**, пока UI не приведён полностью к целевой схеме.

## 1. Общее описание

Projects Hub — центральный модуль управления проектами в NBOS. Проект является ключевой сущностью платформы, объединяющей продукты, задачи, финансы, поддержку и коммуникации. Модуль обеспечивает полный обзор всех проектов компании и глубокое погружение в каждый конкретный проект и **в каждый продукт внутри проекта** (см. § 0).

**Доступ:**

- CEO, Head of Delivery — все проекты
- PM — назначенные проекты
- Seller — проекты, связанные с их сделками
- Developer, QA, Designer — проекты, в которых участвуют
- Finance Director — финансовые вкладки проектов

## 2. Страница списка проектов (Projects List)

**Путь:** `/projects`

### 2.1. Панель вкладок (Tab Bar)

Верхняя панель вкладок для быстрой фильтрации:

| Вкладка     | Описание                                                                               |
| ----------- | -------------------------------------------------------------------------------------- |
| All         | Все проекты (без фильтра)                                                              |
| Development | Проекты, имеющие хотя бы один продукт в стадиях разработки (New → Dev → QA → Transfer) |
| Maintenance | Проекты, имеющие хотя бы один продукт на поддержке (Maintenance)                       |
| Closed      | Завершённые и архивные проекты                                                         |

> **Важно:** проект может отображаться одновременно во вкладках Development и Maintenance, если у него есть продукты в разных состояниях. На карточке такого проекта видны индикаторы обоих состояний.

### 2.2. Переключатель вида

В правом верхнем углу: `Cards | List`

### 2.3. Карточный вид (Cards View)

Проекты отображаются в виде сетки карточек. Каждая карточка содержит:

| Элемент              | Расположение                | Описание                                                          |
| -------------------- | --------------------------- | ----------------------------------------------------------------- |
| Название проекта     | Верх карточки, жирный шрифт | Основное имя проекта                                              |
| Имя клиента          | Под названием               | Компания или контактное лицо                                      |
| Тип бейдж            | Правый верхний угол         | WL (White Label) / Mix / CC (Client Communication)                |
| Статус-индикаторы    | Центр карточки              | Цветные точки: Dev (синий), Maintenance (зелёный), Closed (серый) |
| Количество продуктов | Центр                       | «3 продукта» с иконками типов                                     |
| Аватар PM            | Нижний левый угол           | Маленький аватар менеджера проекта                                |
| Дедлайн              | Нижний правый угол          | Ближайший дедлайн среди активных продуктов                        |

При наведении на карточку появляется лёгкая тень и кнопка быстрого действия (Open).

### 2.4. Табличный вид (List View)

Сортируемые колонки:
| Колонка | Сортировка |
|---------|-----------|
| Название проекта | ✅ |
| Клиент | ✅ |
| Тип (WL/Mix/CC) | ✅ |
| PM | ✅ |
| Seller | ✅ |
| Активные продукты | ✅ |
| Статус | ✅ |
| Дедлайн | ✅ |
| Дата создания | ✅ |

### 2.5. Фильтры

Панель фильтров над списком:

- **По типу:** WL / Mix / CC (мультивыбор)
- **По PM:** выпадающий список менеджеров
- **По Seller:** выпадающий список продавцов
- **По клиенту:** поиск по имени компании/контакта
- **По дате:** диапазон дат создания
- Кнопка «Сбросить фильтры»
- Быстрый поиск (строка поиска по названию проекта)

## 3. Страница проекта (Project Detail)

**Путь:** `/projects/:projectId`

### 3.1. Заголовок проекта

Верхний блок страницы содержит:

- **Название проекта** — крупный заголовок
- **Клиент** — ссылка на контакт + ссылка на компанию
- **Тип бейдж** — WL / Mix / CC (цветной)
- **PM** — аватар + имя (кликабельно → профиль)
- **Seller** — аватар + имя
- **Дата создания** — когда проект был создан

### 3.2. Панель вкладок проекта (Tab Bar)

Под заголовком располагается горизонтальная панель вкладок:

```
Overview | Products | Extensions | Orders | Finance | Subscription | Tasks | Support | Credentials | Drive | Domains | Chat | Audit
```

Активная вкладка подсвечена. Вкладки с уведомлениями имеют бейдж (например, Tasks: 5, Support: 2).

## 4. Вкладка Overview

### 4.1. Сводные карточки (Summary Cards)

Четыре карточки в верхнем ряду:

| Карточка        | Значение                              | Иконка |
| --------------- | ------------------------------------- | ------ |
| Total Revenue   | Общая выручка по проекту              | 💰     |
| Active Products | Количество активных продуктов         | 📦     |
| Open Tasks      | Количество открытых задач             | ✅     |
| Open Tickets    | Количество открытых тикетов поддержки | 🎫     |

### 4.2. Статусы продуктов

Блок ниже показывает каждый продукт проекта с его текущей стадией:

- Название продукта
- Прогресс-бар стадии (визуальная шкала от Starting до Transfer / Closed)
- Текущая стадия — текстовый бейдж
- Ответственный PM/Developer

### 4.3. Лента активности (Recent Activity Feed)

Хронологическая лента последних событий по проекту:

- Новые задачи, изменения статусов
- Комментарии в задачах
- Финансовые события (счета, платежи)
- Тикеты поддержки
- Каждое событие: иконка типа, текст, автор, время

## 5. Вкладка Products

**Путь:** `/projects/:projectId/products`

### 5.1. Список продуктов

Каждый продукт отображается строкой или карточкой:

| Элемент      | Описание                                                                  |
| ------------ | ------------------------------------------------------------------------- |
| Название     | Имя продукта (например, «Website», «Mobile App»)                          |
| Тип          | Web / Mobile / Design / Integration / Other                               |
| Статус бейдж | Текущая стадия (Starting, Development, QA, Transfer) или terminal outcome |
| Прогресс-бар | Визуальная шкала прогресса по стадиям                                     |
| Дедлайн      | Планируемая дата завершения                                               |
| PM           | Ответственный менеджер                                                    |

Клик по продукту → открывает **Product Detail**.

### 5.2. Детальная страница продукта (Product Detail)

**Путь:** `/projects/:projectId/products/:productId`

**Верхняя часть — шкала стадий (Stage Progress Bar):**
Горизонтальная шкала с точками/сегментами:

```
Starting → Development → QA → Transfer
```

Текущая стадия выделена цветом, пройденные — заполнены, будущие — пустые. Клик по стадии (при наличии прав) позволяет переключить.

**Центральная часть — Work Space продукта:**
Переключатель: `Scrum | Kanban`

- **Kanban-вид:** daily execution board по workflow statuses задач.
- **Scrum-вид:** backlog, future sprints, active sprint, closed sprints и текущая sprint board.

**Левая боковая панель — информация о продукте:**

- Тип продукта
- Стадия
- Дедлайн
- Стек технологий
- Описание
- Ответственные
- Ссылки (staging, production, репозиторий)

**Нижняя часть — чеклисты:**
Секция чеклистов для продукта. Каждый чеклист:

- Название (например, «QA Checklist», «Transfer Checklist»)
- Список пунктов с чекбоксами
- Процент выполнения
- Ответственный за каждый пункт

## 6. Вкладка Extensions

**Путь:** `/projects/:projectId/extensions`

### 6.1. Список расширений

Каждое расширение отображает:

| Элемент      | Описание                                                    |
| ------------ | ----------------------------------------------------------- |
| Название     | Имя расширения                                              |
| Размер бейдж | S / M / L / XL (масштаб работ)                              |
| Статус       | Starting / Development / QA / Transfer или terminal outcome |
| Сумма        | Стоимость расширения                                        |
| Дедлайн      | Планируемая дата завершения                                 |
| Исполнитель  | Назначенный разработчик/PM                                  |

### 6.2. Детальная страница расширения (Extension Detail)

**Путь:** `/projects/:projectId/extensions/:extensionId`

- **Описание** — текстовое описание работ
- **Ссылка на заказ (Order)** — связанный Order в системе
- **Мини-доска задач** — Kanban с задачами конкретно этого расширения
- **Чеклист** — список пунктов для выполнения и приёмки
- **Файлы** — прикреплённые документы и макеты

---

## 7. Вкладка Orders

**Путь:** `/projects/:projectId/orders`

### 7.1. Список заказов

| Элемент          | Описание                             |
| ---------------- | ------------------------------------ |
| Название         | Имя заказа                           |
| Тип              | Product / Extension / Subscription   |
| Сумма            | Общая сумма заказа                   |
| Статус оплаты    | Partially Paid / Fully Paid / Unpaid |
| Ссылка на сделку | Связанный Deal в CRM                 |

### 7.2. Прогресс-бар оплаты

Для каждого заказа отображается горизонтальный прогресс-бар:

- Зелёная часть — оплаченная сумма
- Жёлтая часть — ожидаемые платежи (выставлены счета)
- Серая часть — неоплаченный остаток
- Текст: «$5,000 из $10,000 оплачено (50%)»

### 7.3. Детали заказа

Клик по заказу открывает панель с:

- Полная информация о заказе
- Список связанных счетов (Invoices) с их статусами
- Список платежей (Payments) с датами и суммами
- Ссылка на Deal

---

## 8. Вкладка Finance

**Путь:** `/projects/:projectId/finance`

### 8.1. P&L-сводка проекта

Верхний блок — ключевые финансовые показатели:
| Метрика | Описание |
|---------|----------|
| Revenue | Общая выручка (сумма всех оплаченных счетов) |
| Expenses | Общие расходы (прямые расходы по проекту) |
| Margin | Маржа = Revenue - Expenses |
| Margin % | Процент маржинальности |

### 8.2. Список счетов (Invoices)

Таблица всех счетов, связанных с проектом:

- Номер счёта, сумма, статус (бейдж), дата выставления, дата оплаты

### 8.3. Список платежей (Payments)

Таблица всех входящих платежей по проекту.

### 8.4. Список расходов (Expenses)

Таблица расходов, привязанных к проекту: зарплаты за работу по проекту, подписки, домены, субподряд.

---

## 9. Вкладка Subscription

**Путь:** `/projects/:projectId/subscription`

Отображается только для проектов с активной подпиской.

### 9.1. Информация о подписке

| Поле        | Описание                           |
| ----------- | ---------------------------------- |
| Сумма       | Ежемесячная сумма подписки         |
| Billing Day | Число месяца для выставления счёта |
| Тип         | Monthly / Quarterly / Annual       |
| Дата начала | Когда подписка была активирована   |

### 9.2. Мини-грид оплат

Матрица: строки = годы, колонки = Jan–Dec. Каждая ячейка показывает:

- Сумму за этот месяц
- Цвет: зелёный (оплачен), красный (не оплачен), жёлтый (ожидает), серый (не применимо)

### 9.3. История изменений

Таблица изменений суммы подписки с течением времени:

- Дата изменения, старая сумма, новая сумма, причина, кто изменил

## 10. Вкладка Tasks

**Путь:** `/projects/:projectId/tasks`

Объединяет все задачи по всем продуктам и расширениям проекта.

### 10.1. Переключатель видов

`Kanban | List | Timeline`

- **Kanban:** колонки по статусам задач
- **List:** таблица с сортировкой и фильтрами
- **Timeline:** диаграмма Ганта с зависимостями задач

### 10.2. Фильтры

- **По продукту:** выбор конкретного продукта
- **По расширению:** выбор конкретного расширения
- **По исполнителю:** фильтр по назначенному сотруднику
- **По статусу:** Open / In Progress / Review / Completed
- **По приоритету:** Low / Medium / High / Critical

---

## 11. Вкладка Support

**Путь:** `/projects/:projectId/support`

Тикеты поддержки, относящиеся к этому проекту. Kanban-доска со стадиями тикетов. Фильтры по категории и приоритету. Быстрое создание тикета привязанного к проекту.

---

## 12. Вкладка Credentials

**Путь:** `/projects/:projectId/credentials`

### 12.1. Список учётных данных

Организованы по категориям:

- **Hosting** — серверы, панели управления
- **Domains** — регистраторы, DNS
- **Services** — третьесторонние сервисы (API, CMS, CDN)
- **Repository** — Git-аккаунты
- **Client Accounts** — аккаунты клиента

### 12.2. Индикаторы доступа

Каждая запись показывает:

- Название сервиса
- URL
- Логин (виден)
- Пароль (скрыт, кнопка «Show» с аудит-логом)
- Уровень доступа (бейдж: кто может видеть)

### 12.3. Аудит-лог

Каждое нажатие «Показать пароль» записывается в лог: кто, когда, какой пароль просматривал.

Кнопка `+ Add Credential` для добавления новой записи (модальное окно с полями: название, URL, логин, пароль, категория, уровень доступа).

## 13. Остальные вкладки

### Drive (Файлы)

Файловое хранилище проекта. Папки по продуктам и категориям. Drag & drop загрузка. Превью файлов.

### Domains (Домены)

Список доменов, связанных с проектом. Информация: доменное имя, регистратор, дата истечения, статус (active/expired), привязка к продукту.

### Chat (Чат)

Проектный чат. Все участники проекта автоматически добавлены. История сообщений, файлы, @mentions. Ссылки на задачи и тикеты в чате.

### Audit (Аудит)

Полный лог изменений по проекту: кто, когда, что изменил. Фильтры по типу события, по автору, по дате. Экспорт лога.

## 14. UX-паттерны Projects Hub

| Паттерн            | Реализация                                             |
| ------------------ | ------------------------------------------------------ |
| Quick Switch       | Ctrl/Cmd + P для быстрого переключения между проектами |
| Tabbed Navigation  | Вкладки внутри проекта с бейджами-уведомлениями        |
| Contextual Actions | Кнопки действий зависят от текущей вкладки и роли      |
| Breadcrumbs        | Home > Projects > Project Name > Tab                   |
| Sticky Header      | Заголовок проекта и табы фиксированы при скролле       |
| Deep Linking       | Каждая вкладка и сущность имеют уникальный URL         |
