# Карта связей между сущностями

## Общее описание

Все сущности в NBOS Platform связаны между собой, образуя единую экосистему данных. Понимание этих связей критично для правильной архитектуры, автоматизации и обеспечения целостности данных. Ниже описаны все ключевые сущности, их атрибуты и связи.

---

### Дополнительные связи Project

```
PROJECT
  ├── PRODUCT (1:N) — продукты внутри проекта
  ├── EXTENSION (1:N) — доработки
  ├── SUBSCRIPTION (1:N) — подписки
  ├── CREDENTIAL (1:N) — логины, пароли, доступы
  ├── ASSET (1:N) — файлы, дизайны, документы
  ├── CHAT (1:N) — привязанные чаты (проектный, по задачам)
  ├── DOMAIN (1:N) — доменные имена
  └── ORDER (1:N) — заказы, привязанные к проекту
```

## Основные цепочки связей

### Цепочка 1: Lead-to-Cash

```
Lead → Deal → Order → Invoice → Payment → Bonus
```

| Переход           | Триггер                     | Описание                             |
| ----------------- | --------------------------- | ------------------------------------ |
| Lead → Deal       | Lead.status = SQL           | Квалифицированный лид создаёт сделку |
| Deal → Order      | Deal.status = Won + Payment | Выигранная сделка с оплатой → заказ  |
| Order → Invoice   | Ручное / автоматическое     | Счёт на оплату по заказу             |
| Invoice → Payment | Финдиректор подтверждает    | Оплата получена на банковский счёт   |
| Payment → Bonus   | Автоматический              | Бонус начисляется по правилам        |

### Цепочка 2: Contact-to-Task

```
Contact → Project → Product → Sprint → Task
```

| Переход           | Описание                                                  |
| ----------------- | --------------------------------------------------------- |
| Contact → Project | Контакт — владелец проекта (один контакт, много проектов) |
| Project → Product | Проект содержит продукты (Website, App и т.д.)            |
| Product → Sprint  | Продукт содержит спринты (Scrum)                          |
| Sprint → Task     | Спринт содержит задачи                                    |

### Цепочка 3: Subscription Billing

```
Project → Subscription → Invoice → Payment
```

| Переход                | Описание                                              |
| ---------------------- | ----------------------------------------------------- |
| Project → Subscription | Проект имеет подписку (Maintenance и/или Development) |
| Subscription → Invoice | Каждый месяц автоматически создаётся Invoice          |
| Invoice → Payment      | Клиент оплачивает → Финдиректор подтверждает          |

### Цепочка 4: Project Infrastructure

```
Project → Credential / Asset / Chat / Domain
```

| Связь                | Описание                                              |
| -------------------- | ----------------------------------------------------- |
| Project → Credential | Логины и пароли для сервисов проекта                  |
| Project → Asset      | Файлы: дизайны, ТЗ, документация                      |
| Project → Chat       | Чаты: проектный, по сделке, по разработке, по задачам |
| Project → Domain     | Доменные имена: дата покупки, срок продления, сумма   |

### Цепочка 5: Partner Flow

```
Inbound:
Partner → Lead / Deal → Order / Subscription → Payment → Partner Accrual → Partner Balance → Payout Batch → Expense

Outbound:
Client request → Partner Service Terms → Subscription / Invoice to Partner → Payment → Partner Service Revenue
```

| Направление                  | Описание                                                         |
| ---------------------------- | ---------------------------------------------------------------- |
| **Inbound** (партнёр → нам)  | Партнёр передаёт клиента, мы выполняем заказ, партнёр получает % |
| **Outbound** (мы → партнёру) | Мы передаём клиента партнёру, партнёр платит Neetrino за клиента |

### Цепочка 6: Expenses

```
Expense → Project / Order (опциональная привязка)
Planned Expense → Expense (автоматическое создание по расписанию)
```

| Тип затраты         | Привязка                                   | Описание                                       |
| ------------------- | ------------------------------------------ | ---------------------------------------------- |
| Запланированная     | К проекту / подписке                       | Ежемесячная/ежегодная, создаётся автоматически |
| Незапланированная   | К проекту (опционально)                    | Разовая, создаётся вручную                     |
| Партнёрская выплата | К Partner Accrual / Payout Batch + Partner | Expense создаётся только после payout batch    |
| Зарплата            | К сотруднику                               | Ежемесячная, из Salary Board                   |

## Описание каждой сущности

### Contact (Контакт)

| Поле         | Описание                         |
| ------------ | -------------------------------- |
| Имя, Фамилия | ФИО контактного лица             |
| Телефон      | Основной номер                   |
| Email        | Электронная почта                |
| WhatsApp     | Номер WhatsApp (для уведомлений) |
| Компании     | Привязанные компании (1:N)       |
| Проекты      | Привязанные проекты (1:N)        |
| Лиды         | Привязанные лиды (1:N)           |
| Сделки       | Привязанные сделки (1:N)         |

**Ключевое правило:** Один контакт = один человек. Контакт может иметь множество проектов для разных брендов/бизнесов.

### Company (Компания)

| Поле       | Описание                                 |
| ---------- | ---------------------------------------- |
| Название   | Название юридического лица / бренда      |
| Тип        | Юрлицо / Физлицо / ИП                    |
| Реквизиты  | ИНН, ОГРН, адрес и т.д.                  |
| Tax Status | Tax / Tax-Free (по умолчанию для счетов) |
| Контакт    | Привязанный контакт                      |
| Проекты    | Привязанные проекты                      |

### Lead (Лид)

| Поле                 | Описание                                  |
| -------------------- | ----------------------------------------- |
| Источник             | Cold Call / Marketing / Partner / Inbound |
| Статус               | New → ... → SQL / SPAM / Frozen           |
| Контакт              | Привязанный контакт                       |
| Описание потребности | Текст                                     |
| Рекламная кампания   | Откуда именно пришёл (для маркетинга)     |

### Deal (Сделка)

| Поле        | Описание                                       |
| ----------- | ---------------------------------------------- |
| Стадия      | Start a Conversation → ... → Deal Won / Failed |
| Сумма       | Ожидаемая сумма сделки                         |
| Тип оплаты  | Classic / Subscription                         |
| Тип проекта | WL / Mix / Custom Code                         |
| Тег         | New / Extension                                |
| Контакт     | Привязанный контакт                            |
| Компания    | Привязанная компания                           |
| Проект      | Привязанный проект (для Extension)             |
| Продажник   | Ответственный Seller                           |
| Дедлайн     | Срок сдачи проекта клиенту                     |

### Order (Заказ)

| Поле          | Описание                                    |
| ------------- | ------------------------------------------- |
| Сумма         | Общая сумма заказа                          |
| Тип оплаты    | Classic (50/50 или 30/30/40) / Subscription |
| Статус оплаты | Частично оплачен / Полностью оплачен        |
| Проект        | Привязанный проект                          |
| Сделка        | Из какой сделки создан                      |
| Invoices      | Привязанные счета (1:N)                     |
| Бонусы        | Привязанные бонусы (1:N)                    |

### Project (Проект)

| Поле          | Описание                                   |
| ------------- | ------------------------------------------ |
| Название      | Название проекта (= бренд)                 |
| Статус        | Development / Maintenance / Done / On Hold |
| Контакт       | Владелец проекта                           |
| Компания      | Юрлицо                                     |
| Products      | Продукты внутри проекта (1:N)              |
| Extensions    | Доработки (1:N)                            |
| Subscriptions | Подписки (1:N)                             |
| Orders        | Заказы (1:N)                               |
| Credentials   | Доступы (1:N)                              |
| Domains       | Домены (1:N)                               |
| Assets        | Файлы (1:N)                                |
| Chats         | Чаты (1:N)                                 |
| PM            | Ответственный проектный менеджер           |
| Team          | Команда проекта                            |

### Product (Продукт)

| Поле        | Описание                                            |
| ----------- | --------------------------------------------------- |
| Тип         | Website / Mobile App / CRM / Other                  |
| Тип проекта | WL / Mix / Custom Code                              |
| Стадия      | New → Creating → Development → QA → Transfer → Done |
| Проект      | Родительский проект                                 |
| Спринты     | Привязанные спринты (1:N)                           |
| Задачи      | Все задачи продукта                                 |

### Extension (Доработка)

| Поле     | Описание                                 |
| -------- | ---------------------------------------- |
| Название | Описание доработки                       |
| Стадия   | New → Development → QA → Transfer → Done |
| Проект   | Родительский проект                      |
| Order    | Связанный заказ                          |
| Задачи   | Задачи по доработке                      |

### Subscription (Подписка)

| Поле        | Описание                                      |
| ----------- | --------------------------------------------- |
| Тип         | Maintenance / Development / Dev+Mnt / Partner |
| Сумма       | Ежемесячная сумма                             |
| Billing Day | День выставления счёта                        |
| Tax Status  | Tax / Tax-Free                                |
| Проект      | Привязанный проект                            |
| Invoices    | Созданные счета (1:N)                         |

### Invoice (Счёт)

| Поле       | Описание                                                  |
| ---------- | --------------------------------------------------------- |
| Сумма      | Сумма счёта                                               |
| Стадия     | New → Gov System → Send Message → Overdue → Paid / Unpaid |
| Tax Status | Tax / Tax-Free (наследуется)                              |
| Источник   | Order ИЛИ Subscription                                    |
| Payment    | Привязанная оплата (1:1)                                  |
| Дедлайн    | До какого числа оплатить                                  |

### Payment (Оплата)

| Поле          | Описание                                             |
| ------------- | ---------------------------------------------------- |
| Сумма         | Сумма оплаты                                         |
| Дата          | Дата поступления                                     |
| Invoice       | Привязанный счёт                                     |
| Подтверждение | Финдиректор подтвердил                               |
| Триггеры      | Какие события запускает (Bonus, статус Order и т.д.) |

### Bonus (Бонус)

| Поле      | Описание                                  |
| --------- | ----------------------------------------- |
| Тип       | Sales / Delivery / Marketing              |
| Сумма     | Расчётная сумма бонуса                    |
| Состояние | Incoming → Earned → ... → Paid / Clawback |
| Сотрудник | Кому начислен                             |
| Order     | К какому заказу привязан                  |
| Payment   | Какой платёж является триггером           |

### Domain (Домен)

| Поле           | Описание               |
| -------------- | ---------------------- |
| Доменное имя   | URL домена             |
| Провайдер      | Beget, Reg.ru и т.д.   |
| Дата покупки   | Когда куплен           |
| Дата продления | Когда нужно продлить   |
| Стоимость      | Сумма продления        |
| Аккаунт        | Привязанный Credential |
| Проект         | Привязанный проект     |

### Credential (Доступ)

| Поле             | Описание                               |
| ---------------- | -------------------------------------- |
| Сервис/Провайдер | Название сервиса (из справочника)      |
| Логин            | Логин                                  |
| Пароль           | Пароль (зашифрованный)                 |
| URL              | Ссылка на сервис                       |
| Заметки          | Дополнительная информация              |
| Проект           | Привязанный проект                     |
| Доступ           | Кто из сотрудников видит данный пароль |

### Expense (Затрата)

| Поле          | Описание                                                         |
| ------------- | ---------------------------------------------------------------- |
| Тип           | Запланированная / Незапланированная                              |
| Сумма         | Сумма затраты                                                    |
| Периодичность | Ежемесячная / Квартальная / Ежегодная / Разовая                  |
| Стадия        | This Month → Pay Now → Delayed → On Hold → Paid / Unpaid         |
| Проект        | Привязка к проекту (опционально)                                 |
| Order         | Привязка к заказу (опционально)                                  |
| Partner       | Привязка к партнёру: inbound accruals / outbound partner service |

### Partner (Партнёр)

| Поле         | Описание                                                                   |
| ------------ | -------------------------------------------------------------------------- |
| Название/Имя | Название партнёра                                                          |
| Тип          | Standard / Premium (с договором)                                           |
| % комиссии   | Commission Policy по Deal Type; Payment Type влияет только на ритм выплаты |
| Проекты      | Привязанные проекты (через которых пришли клиенты)                         |
| Payouts      | История выплат (через Expenses)                                            |

## Ключевые правила целостности данных

### Обязательные связи

| Правило                                           | Описание                                    |
| ------------------------------------------------- | ------------------------------------------- |
| Каждый Invoice принадлежит Order ИЛИ Subscription | Invoice не может существовать без источника |
| Каждый Bonus принадлежит Order                    | Бонус всегда привязан к конкретному заказу  |
| Каждый Payment привязан к Invoice                 | Оплата всегда относится к конкретному счёту |
| Каждый Product принадлежит Project                | Продукт не существует без проекта           |
| Каждый Extension принадлежит Project              | Доработка привязана к проекту               |
| Каждая Subscription привязана к Project           | Подписка не существует без проекта          |
| Каждый Domain привязан к Project                  | Домен принадлежит проекту                   |
| Каждый Credential привязан к Project              | Доступ хранится в контексте проекта         |

### Триггерные связи (Payment как источник событий)

| Событие Payment                                   | Что запускает                                                         |
| ------------------------------------------------- | --------------------------------------------------------------------- |
| Payment подтверждён (из Order)                    | Проверка: все ли Invoice по Order оплачены → Order.status обновляется |
| Payment подтверждён (из Subscription)             | Ячейка в Subscription Grid → зелёная                                  |
| Payment подтверждён (первый по Deal)              | Deal → Order → Project создаётся                                      |
| Payment подтверждён (последний по Order, Classic) | Бонус Dev переходит в состояние Vested                                |
| Payment подтверждён (месяц 4, Subscription)       | Бонус Dev переходит в состояние Vested                                |
| Payment подтверждён (Sales)                       | Бонус Sales → Earned                                                  |
| Refund / Chargeback                               | Bonus → Clawback                                                      |

### Каскадные операции

| Действие              | Каскад                                                        |
| --------------------- | ------------------------------------------------------------- |
| Удаление Project      | Запрещено, если есть активные Orders, Subscriptions, Products |
| Закрытие Order        | Требует: все Invoices Paid, все Products Done                 |
| Удаление Contact      | Запрещено, если есть активные Projects                        |
| Закрытие Subscription | Прекращение создания новых Invoice с следующего месяца        |
