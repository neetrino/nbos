# Бонусы и зарплата (Bonus & Payroll)

## Общая концепция

Этот документ описывает UI/board-часть бонусной и зарплатной системы — то, как финансовый директор и сотрудники видят и взаимодействуют с бонусами и зарплатами в интерфейсе.

> Бизнес-логика расчёта бонусов (формулы, проценты, условия начисления) описана в отдельном документе: `03-Business-Logic/03-Bonus-Payroll-Logic.md`

```
Событие (оплата заказа, доставка) → Bonus создан → Bonus Board → Payroll → Expense → Paid
```

---

## Bonus Board (Доска бонусов)

### Назначение

Центральное место для просмотра и управления всеми бонусами компании. Каждый сотрудник видит свои бонусы, финансовый директор — бонусы всех.

### Колонки (Kanban-стиль)

```
┌──────────────┬──────────────┬──────────────┐
│   Incoming   │    Active    │     Paid     │
│  (прогноз)   │  (к выплате) │ (выплачено)  │
├──────────────┼──────────────┼──────────────┤
│              │              │              │
│ ┌──────────┐ │ ┌──────────┐ │ ┌──────────┐ │
│ │Project A │ │ │Project B │ │ │Project C │ │
│ │Order #12 │ │ │Order #8  │ │ │Order #5  │ │
│ │30,000 ₽  │ │ │45,000 ₽  │ │ │25,000 ₽  │ │
│ │Sales 5%  │ │ │Delivery  │ │ │Sales 5%  │ │
│ │⏳ awaiting│ │ │✅ earned  │ │ │✅ paid    │ │
│ └──────────┘ │ └──────────┘ │ └──────────┘ │
│              │              │              │
│ ┌──────────┐ │ ┌──────────┐ │ ┌──────────┐ │
│ │Project D │ │ │Project A │ │ │Project B │ │
│ │Order #15 │ │ │Order #11 │ │ │Order #7  │ │
│ │50,000 ₽  │ │ │20,000 ₽  │ │ │15,000 ₽  │ │
│ │Delivery  │ │ │Marketing │ │ │Marketing │ │
│ │⏳ awaiting│ │ │✅ earned  │ │ │✅ paid    │ │
│ └──────────┘ │ └──────────┘ │ └──────────┘ │
└──────────────┴──────────────┴──────────────┘
```

### Описание колонок

| Колонка | Статус | Описание |
|---------|--------|----------|
| **Incoming** | Прогноз | Бонусы, которые будут начислены после выполнения условий (доставка проекта, полная оплата). Показывает, сколько сотрудник заработает в будущем |
| **Active** | К выплате | Бонусы, условие начисления которых выполнено. Заработаны в текущем месяце. Будут выплачены в следующем месяце с зарплатой |
| **Paid** | Выплачено | Бонусы, которые уже выплачены сотруднику. Исторические данные |

### Карточка бонуса

| Поле | Описание |
|------|----------|
| `bonus_id` | Уникальный идентификатор |
| `employee` | Сотрудник-получатель |
| `project` | Проект |
| `order` | Заказ (если привязан) |
| `amount` | Сумма бонуса |
| `percentage` | Процент (от суммы заказа/подписки) |
| `type` | Тип: Sales / Delivery / Marketing |
| `earn_event` | Событие начисления (полная оплата, доставка, подписка) |
| `earn_date` | Дата начисления (когда условие выполнено) |
| `status` | Incoming / Active / Paid |
| `paid_date` | Дата выплаты |
| `payroll_month` | Месяц зарплатной ведомости |

### Переходы статусов

```
                    Условие выполнено
    Incoming ──────────────────────────→ Active
    (прогноз)     (оплата получена,       (к выплате)
                   проект сдан)
                                            │
                                            │ Payroll run
                                            ▼
                                          Paid
                                       (выплачено)
```

Примеры условий перехода Incoming → Active:

| Тип бонуса | Условие |
|------------|---------|
| Sales bonus | Заказ полностью оплачен (все Invoice = Paid) |
| Delivery bonus | Проект сдан клиенту + полная оплата получена |
| Marketing bonus | По условиям маркетинговой программы |
| Subscription referral | Очередной месяц подписки оплачен |

### Фильтры Bonus Board

| Фильтр | Описание |
|--------|----------|
| **По сотруднику** | Бонусы конкретного сотрудника |
| **По проекту** | Все бонусы по проекту |
| **По месяцу** | Бонусы за определённый месяц |
| **По типу** | Sales / Delivery / Marketing |
| **По статусу** | Incoming / Active / Paid |

### Итоговые показатели (для сотрудника)

Каждый сотрудник видит:

| Показатель | Описание |
|------------|----------|
| **Incoming Total** | Сумма всех прогнозных бонусов |
| **Active Total** | Сумма бонусов к выплате в следующем месяце |
| **Paid Total (месяц)** | Выплаченные бонусы за текущий месяц |
| **Paid Total (год)** | Выплаченные бонусы за год |

## Salary Board (Зарплатная ведомость)

### Назначение

Ежемесячный обзор зарплат всех сотрудников. Используется финансовым директором для расчёта и обработки payroll.

### Структура

```
┌─────────────────┬────────────┬──────────────┬──────────────┬──────────┐
│ Сотрудник       │ Базовая ЗП │ Бонусы (мес) │ Итого к выпл │ Статус   │
├─────────────────┼────────────┼──────────────┼──────────────┼──────────┤
│ Иванов И.И.     │   300,000  │     45,000   │    345,000   │ ✅ Paid  │
│ Петров П.П.     │   250,000  │     20,000   │    270,000   │ ⏳ Pending│
│ Сидоров С.С.    │   400,000  │          0   │    400,000   │ ⏳ Pending│
│ Козлова К.К.    │   200,000  │     15,000   │    215,000   │ ✅ Paid  │
├─────────────────┼────────────┼──────────────┼──────────────┼──────────┤
│ ИТОГО:          │ 1,150,000  │     80,000   │  1,230,000   │ 2/4 paid│
└─────────────────┴────────────┴──────────────┴──────────────┴──────────┘
```

### Поля записи Salary Board

| Поле | Описание |
|------|----------|
| `employee` | Сотрудник |
| `month` | Месяц начисления |
| `base_salary` | Базовая зарплата |
| `bonuses_total` | Сумма всех Active бонусов этого сотрудника |
| `bonus_details` | Детализация бонусов (список карточек) |
| `total_payout` | Итого к выплате (ЗП + бонусы) |
| `status` | Pending / Paid |
| `paid_date` | Дата выплаты |

### Дополнительные колонки (расширение)

- Удержания (если применимо)
- Налоги
- Авансы
- Бонусы с разбивкой по типам (Sales / Delivery / Marketing)

## Payroll Cycle (Цикл выплаты зарплат)

### Временная шкала

```
Месяц N                              Месяц N+1
────────────────────────────────────  ──────────────────────
                                      1-е     ...    10-е
Бонусы зарабатываются                 │              │
(earn events происходят)              ▼              ▼
                                    Payroll        Payroll
                                    Review         Done
```

### Пошаговый процесс

#### 1. Подготовка (1-е число месяца N+1)

```
Система автоматически формирует:
  ├─ Список всех сотрудников с базовыми зарплатами
  ├─ Все Active бонусы за месяц N (заработанные)
  └─ Итоговые суммы к выплате для каждого
```

#### 2. Ревью финансового директора (1–5-е число)

```
Финансовый директор:
  ├─ Просматривает Salary Board
  ├─ Проверяет корректность бонусов
  ├─ Сверяет с бюджетом
  └─ Подтверждает готовность к выплате
```

#### 3. Обработка выплат (5–10-е число)

```
Для каждого сотрудника:
  ├─ Создаётся Expense (категория: Salary)
  │   ├─ Сумма: base_salary + bonuses_total
  │   └─ Описание: "Зарплата [сотрудник] за [месяц]"
  ├─ Финансовый директор производит платёж
  ├─ Отмечает в Salary Board: Paid
  └─ Все Active бонусы сотрудника → Paid
```

#### 4. Завершение

```
После выплаты всем сотрудникам:
  ├─ Salary Board за месяц N → закрыт (все Paid)
  ├─ Все Active бонусы → Paid
  └─ Expense записи созданы для P&L
```

## Связь бонусов с расходами

Бонусы — это не отдельная статья расхода сами по себе. Они включаются в зарплатную ведомость:

```
Active Bonus ──→ включается в Salary Board ──→ Expense (Salary) ──→ Company P&L
```

При этом для целей Project P&L и Order P&L:
- Бонус продажника (Sales) → расход заказа и проекта
- Бонус доставки (Delivery) → расход заказа и проекта
- Бонус маркетинга (Marketing) → расход компании (если не привязан к проекту)

## Доступы и видимость

| Роль | Что видит |
|------|----------|
| **CEO** | Все бонусы, все зарплаты, полная ведомость |
| **Финансовый директор** | Все бонусы, все зарплаты, управление Salary Board |
| **Сотрудник** | Только свои бонусы (Incoming, Active, Paid) и свою зарплатную строку |
| **Руководитель отдела** | Бонусы своего отдела (опционально) |

## Уведомления

| Событие | Получатель | Описание |
|---------|-----------|----------|
| Бонус перешёл в Active | Сотрудник | «Ваш бонус 30,000 по проекту X заработан» |
| Бонус выплачен | Сотрудник | «Бонус 30,000 выплачен с зарплатой» |
| Payroll готов к ревью | Финансовый директор | «Зарплатная ведомость за [месяц] готова» |
| Payroll завершён | CEO | «Все зарплаты за [месяц] выплачены. Итого: X» |
| Новый Incoming бонус | Сотрудник | «Прогнозный бонус: 50,000 по проекту Y» |

## Метрики

| Метрика | Описание |
|---------|----------|
| **Total Payroll** | Общая сумма зарплат + бонусов за месяц |
| **Bonus Ratio** | Доля бонусов в общем payroll |
| **Avg Bonus per Employee** | Средний бонус на сотрудника |
| **Incoming Pipeline** | Сумма прогнозных бонусов (Incoming) |
| **Payroll as % of Revenue** | Доля payroll в выручке (для контроля рентабельности) |

## Связи с другими модулями

```
CRM (Order paid) ──→ Bonus created (Sales, Delivery)
Projects Hub (project delivered) ──→ Bonus Active (Delivery)
Subscriptions (monthly payment) ──→ Bonus created (referral)
Partners (partner payout) ──→ Expense (Partner Payout) — отдельно от Bonus Board
My Company (employees) ──→ Base salary data
Expenses ──→ Salary Expense entries
P&L Reports ──→ Payroll costs
```
