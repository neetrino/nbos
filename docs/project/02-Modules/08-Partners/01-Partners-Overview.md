# Модуль Partners — Обзор

> NBOS Platform — управление партнёрскими отношениями: входящие и исходящие направления

## Назначение

Партнёрский модуль управляет **двусторонней партнёрской моделью** Neetrino. Компания одновременно получает заказы от партнёров (Inbound) и передаёт клиентов партнёрам (Outbound). Модуль отслеживает все партнёрские отношения, реферальные сделки, начисления, выплаты и аналитику.

**Ключевой принцип:** Партнёрство — это бизнес-отношение с прозрачным финансовым учётом. Каждый рубль, заработанный или выплаченный через партнёра, фиксируется и связывается с конкретным проектом, заказом и сделкой.

---

## Двусторонняя партнёрская модель

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   DIRECTION 1: INBOUND                DIRECTION 2: OUTBOUND        │
│   (Партнёры приводят заказы)          (Neetrino передаёт клиентов) │
│                                                                     │
│   Партнёр ──── Клиент ──→ Neetrino   Клиент ──→ Neetrino ──→ Партнёр│
│                                                                     │
│   Neetrino выполняет работу           Партнёр выполняет работу     │
│   Партнёр получает % от заказа        Партнёр платит Neetrino      │
│                                                                     │
│   Neetrino ПЛАТИТ партнёру            Neetrino ПОЛУЧАЕТ от партнёра│
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Direction 1: INBOUND (Партнёры приводят заказы)

### Как это работает

1. Партнёр рекомендует Neetrino клиенту, которому нужна разработка (сайт, приложение, система)
2. Клиент обращается в Neetrino → создаётся лид с пометкой «Источник: Партнёр»
3. Neetrino проводит клиента через весь цикл: продажа → разработка → сдача → обслуживание
4. Партнёр получает **30% от суммы** — стандартная ставка

### Уровни партнёров

| Уровень | Описание | Договор | Обязательства |
|---------|----------|---------|---------------|
| **Regular** | Неформальное партнёрство. Партнёр иногда передаёт заказы | Нет | Нет гарантий, нет эксклюзива |
| **Premium** | Подписанное партнёрское соглашение. Стабильный поток заказов | Да | Все заказы в нише → Neetrino, выделенный менеджер |

### Правила выплат партнёрам (Inbound)

#### Classic-заказ (одноразовая оплата)

```
Клиент оплачивает проект → Проект разрабатывается → Проект сдан
  → 1 месяц тестового периода (стабильность)
  → Все Invoice оплачены (100% суммы получено)
  → Создаётся Expense: 30% от суммы заказа → партнёру
  → Срок выплаты: 30 дней
```

Пример:
- Сумма заказа: 1,000,000 ₽
- Оплата: 50/50 (500,000 предоплата + 500,000 после сдачи)
- Обе части получены → заказ полностью оплачен
- 1 месяц тестового периода прошёл без проблем
- Партнёру: 1,000,000 × 30% = **300,000 ₽** (одноразово)

#### Subscription-заказ (ежемесячная подписка)

```
Клиент на подписке → Период разработки (партнёр не получает ничего)
  → Проект сдан → Начинаются ежемесячные подписочные платежи
  → Каждый месяц после получения подписки от клиента:
    → Создаётся Expense: 30% от месячной подписки → партнёру
```

Пример:
- Подписка клиента: 100,000 ₽/мес (разработка, 3 месяца)
- Разработка завершена → клиент переходит на Maintenance
- Подписка: 80,000 ₽/мес (обслуживание)
- Каждый месяц при получении 80,000 ₽ от клиента:
- Партнёру: 80,000 × 30% = **24,000 ₽/мес**

#### Extension (Доработка)

Те же правила, что и для основного заказа:
- Classic Extension → одноразовая выплата 30% после полной оплаты + тест-период
- Subscription Extension → ежемесячная выплата 30%

#### Все типы платежей

**Всё, за что клиент платит Neetrino** — партнёр получает свою долю:

| Тип платежа | Выплата партнёру |
|-------------|------------------|
| Разработка (Classic) | 30% от суммы заказа (одноразово) |
| Разработка (Subscription) | 30% ежемесячно после сдачи |
| Доработка (Extension) | 30% по правилам типа оплаты |
| Техобслуживание (Maintenance) | 30% ежемесячно |
| Любые дополнительные услуги | 30% от суммы |

## Direction 2: OUTBOUND (Neetrino передаёт клиентов)

### Как это работает

1. Клиент обращается в Neetrino за услугой, которую компания не предоставляет (SMM, SEO, маркетинговое продвижение, контекстная реклама)
2. Neetrino рекомендует клиенту проверенного партнёра
3. Партнёр заключает договор с клиентом напрямую и оказывает услугу
4. Партнёр платит Neetrino фиксированную сумму (ежемесячно или одноразово)

### Финансовая модель Outbound

```
Клиент ←───── Услуга ←───── Партнёр
                              │
                              │ Платит Neetrino
                              ▼
                           Neetrino
                     (доход как подписка)
```

- Neetrino выступает **генератором лидов** для партнёра
- Партнёр платит Neetrino за каждого переданного клиента
- Оплата может быть: ежемесячная фиксированная или одноразовая
- В системе учитывается как **входящий доход** (Subscription / Income)

### Учёт в системе

Outbound-партнёрство учитывается как Subscription (подписка) со стороны партнёра:

| Поле | Значение |
|------|----------|
| **Тип** | Partner Service (Партнёрская услуга) |
| **Плательщик** | Партнёрская компания |
| **Получатель** | Neetrino |
| **Сумма** | Фиксированная ежемесячная или одноразовая |
| **Привязка** | К клиенту и к партнёру |

## Партнёрский справочник (Partner Directory)

### Карточка партнёра

| Поле | Тип | Описание |
|------|-----|----------|
| `partner_id` | Auto | Уникальный идентификатор |
| `company_name` | Text | Название компании партнёра |
| `contact_person` | Text | Основное контактное лицо |
| `phone` | Phone | Телефон |
| `email` | Email | Email |
| `telegram` | Text | Telegram |
| `type` | Select | Inbound / Outbound / Both |
| `level` | Select | Regular / Premium |
| `agreement_status` | Select | No Agreement / Draft / Active / Expired |
| `default_percentage` | Number | Стандартный процент (по умолчанию: 30%) |
| `start_date` | Date | Дата начала партнёрства |
| `notes` | Text | Заметки |
| `status` | Select | Active / Paused / Terminated |

### Представления (Views)

```
┌────────────────┬──────────┬──────────┬────────────┬──────────────┬──────────┐
│ Партнёр        │ Тип      │ Уровень  │ Договор    │ Сделок (all) │ Revenue  │
├────────────────┼──────────┼──────────┼────────────┼──────────────┼──────────┤
│ Digital Agency │ Inbound  │ Premium  │ ✅ Active   │     12       │ 4,500,000│
│ SEO Masters   │ Outbound │ Regular  │ ❌ No       │      3       │   180,000│

└────────────────┴──────────┴──────────┴────────────┴──────────────┴──────────┘
```

Фильтры:
- По типу (Inbound / Outbound / Both)
- По уровню (Regular / Premium)
- По статусу договора
- По активности (Active / Paused / Terminated)
- Поиск по названию / контакту

## Реферальные сделки (Referral Deals)

### Привязка партнёра к сделке

При создании лида или сделки в CRM можно указать **партнёра-реферала**:

```
Lead/Deal Card:
  ├─ Источник: Партнёр
  ├─ Партнёр: [Digital Agency] (выбор из Partner Directory)
  ├─ Реферальный %: 30% (берётся из default_percentage партнёра)
  └─ Может быть переопределён: 25% (для конкретной сделки)
```

### Связь реферала с финансами

```
Deal (партнёр: Digital Agency) → Deal Won → Order Created
  └─ Order полностью оплачен
     └─ Тест-период пройден (Classic) / Месячный платёж получен (Subscription)
        └─ Partner Payout создан автоматически
           └─ Появляется на Partner Payout Board
```

## Partner Payout Board (Доска партнёрских выплат)

Аналогична Expense Board — центральное место управления партнёрскими выплатами.

### Состояния партнёрской выплаты (Payout States)

```
                     Условие выполнено
    Earned ──────────────────────────→ Eligible
    (заработано)   (Classic: тест       (готово к
                    пройден;             утверждению)
                    Subscr: платёж
                    получен)
                                          │
                                          │ Finance Dir утверждает
                                          ▼
                                       Payable
                                    (утверждено к
                                      оплате)
                                          │
                                          │ Деньги переведены
                                          ▼
                                        Paid
                                    (выплачено)
```

| Статус | Описание | Триггер перехода |
|--------|----------|------------------|
| **Earned** | Партнёр заработал выплату. Для Classic — заказ полностью оплачен. Для Subscription — месячный платёж получен | Авто: при полной оплате заказа или получении подписки |
| **Eligible** | Условия выплаты выполнены. Для Classic — прошёл 1 месяц тестового периода. Для Subscription — платёж подтверждён | Авто: по таймеру (Classic) или подтверждению (Subscription) |
| **Payable** | Утверждено Finance Director к оплате | Вручную: Finance Director подтверждает |
| **Paid** | Деньги переведены на счёт партнёра | Вручную: Finance Director отмечает выплату |

### Визуализация Payout Board

```
┌──────────────────┬──────────────────┬──────────────────┬──────────────────┐
│     Earned       │    Eligible      │    Payable       │      Paid        │
├──────────────────┼──────────────────┼──────────────────┼──────────────────┤

│ │Project F     │ │ │Project D     │ │ │Project C     │ │ │Project A     │ │
```

### Автоматическое создание Payout

Партнёрские выплаты создаются **автоматически** при наступлении условий:

| Событие | Действие |
|---------|----------|
| Classic Order → полная оплата | Payout создан со статусом Earned |
| Classic Order → тест-период 1 мес прошёл | Payout → Eligible |
| Subscription → месячный платёж получен | Payout создан со статусом Earned |
| Subscription → платёж подтверждён | Payout → Eligible |
| Extension → полная оплата + тест | По правилам типа оплаты |

При создании Payout также автоматически создаётся **Expense** (категория: Partner Payout) на доске расходов Finance Director.

## Partner Performance (Аналитика партнёров)

### Метрики Inbound-партнёра

| Метрика | Описание |
|---------|----------|
| **Referred Revenue** | Общая выручка от клиентов партнёра |
| **Deals Referred** | Количество переданных сделок |
| **Conversion Rate** | % сделок партнёра, завершившихся успешно |
| **Avg Deal Size** | Средний чек сделок от партнёра |

| **Total Payouts** | Сумма выплат партнёру за период |
| **ROI** | Revenue / Payouts (сколько компания зарабатывает на каждый рубль выплат) |

## Документы и соглашения (Agreements)

### Хранение документов

В карточке каждого Premium-партнёра хранятся:

| Документ | Описание |
|----------|----------|
| **Партнёрское соглашение** | Подписанный договор (PDF/скан) |
| **Доп. соглашения** | Изменения условий, специальные ставки |
| **История переписки** | Ключевые письма/решения |
| **NDA** | Соглашение о неразглашении (если есть) |

### Статусы соглашения

| Статус | Описание |
|--------|----------|
| **No Agreement** | Нет подписанного договора (Regular партнёры) |
| **Draft** | Договор на согласовании |
| **Active** | Договор подписан и действует |
| **Expired** | Срок договора истёк, требуется продление |
