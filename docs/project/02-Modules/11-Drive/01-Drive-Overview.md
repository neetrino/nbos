# Модуль Drive — Обзор

> NBOS Platform — централизованное хранилище файлов и документов

## Назначение

Модуль Drive объединяет **все файлы компании** в одном структурированном пространстве. Вместо хаотичного хранения в Bitrix Drive, Google Drive и OneDrive — единое хранилище с автоматической структурой по проектам, клиентам и категориям.

**Ключевой принцип:** каждый файл принадлежит контексту — проекту, клиенту или компании. Загрузка файла в любой сущности (сделка, задача, счёт) автоматически сохраняет его в Drive с правильной привязкой.

---

## Проблема, которую решает Drive

### Текущее состояние (до NBOS)

| Проблема | Описание |
|----------|---------|
| **Разрозненное хранение** | Файлы разбросаны между Bitrix Drive, Google Drive, OneDrive, личными папками |
| **Отсутствие структуры** | Нет единого стандарта именования и организации папок |
| **Сложность поиска** | Чтобы найти макет для проекта — нужно обойти 3–4 хранилища |
| **Проблемы доступа** | Разработчик не имеет доступа к папке PM, и наоборот |
| **Потеря файлов** | При смене сотрудника часть файлов теряется (были на личном диске) |
| **Дублирование** | Один файл копируется в несколько мест, версии расходятся |

### Целевое состояние (NBOS Drive)

- Единое хранилище для всей компании
- Автоматическая структура папок при создании проекта
- Доступ на основе ролей и участия в проектах
- Быстрый поиск по всем файлам
- Версионирование документов

## Структура хранилища

### 1. Папки проектов (Project Folders)

При создании нового проекта автоматически генерируется структура папок:

```
📁 [Название проекта]/
 ├── 📁 Design/          — макеты, PSD, ссылки на Figma
 ├── 📁 Documents/       — брифы, ТЗ, спецификации, договоры
 ├── 📁 Development/     — техническая документация, env-файлы
 ├── 📁 Assets/          — логотипы, брендбук, контент, изображения
 └── 📁 Delivery/        — release notes, документы передачи
```

**Правила:**
- Структура создаётся автоматически — не требует ручного создания
- PM может добавлять дополнительные подпапки по необходимости
- Файлы, загруженные в карточку проекта, автоматически попадают в корневую папку проекта
- Файлы из задач проекта привязываются к папке проекта

### 2. Библиотека клиентов (Client Library)

Файлы, организованные по клиентам (Contact / Company):

```
📁 Clients/
 ├── 📁 ClientBrand LLC/
 │    ├── 📄 Договор_2024.pdf
 │    ├── 📄 NDA.pdf
 │    └── 📄 Реквизиты.docx
 ├── 📁 AnotherClient Inc/
 │    ├── 📄 Контракт.pdf
 │    └── 📄 Доверенность.pdf
 └── ...
```

**Содержимое:**
- Договоры и юридические документы
- Реквизиты компаний
- История переписки (экспорт)
- Доступно из модуля Clients → Client Portfolio

### 3. Библиотека компании (Company Library)

Общие файлы компании Neetrino:

```
📁 Company/
 ├── 📁 Templates/       — шаблоны КП, договоров, актов, SOP
 ├── 📁 Brand/           — логотип Neetrino, брендбук, визитки
 ├── 📁 Training/        — обучающие материалы, инструкции
 └── 📁 Shared/          — общие ресурсы, полезные документы
```

**Назначение:**
- Шаблоны документов (Seller берёт шаблон КП, PM — шаблон брифа)
- Брендовые материалы Neetrino
- Обучающие материалы для новых сотрудников
- SOP-документы (стандартные операционные процедуры)

### 4. Личный диск (Personal Drive)

- У каждого сотрудника есть личное файловое пространство
- Файлы не видны другим, если не расшарены явно
- Для черновиков, личных заметок, рабочих файлов

## Функция Quick Attach

### Концепция

Из любой сущности в NBOS можно прикрепить файл. Файл сохраняется в Drive и **привязывается** к сущности:

| Откуда прикрепляется | Куда сохраняется в Drive | Связь |
|---------------------|------------------------|-------|
| Карточка сделки (Deal) | Папка проекта (если есть) или Company | Deal → File |
| Карточка задачи (Task) | Папка проекта задачи | Task → File |
| Счёт (Invoice) | Папка проекта | Invoice → File |
| Тикет (Support Ticket) | Папка проекта | Ticket → File |
| Контакт (Contact) | Client Library | Contact → File |
| Credential | Не сохраняется в Drive | Зашифрованное хранение |

### Поведение

1. Пользователь нажимает «Прикрепить файл» в карточке сущности
2. Выбирает файл (загрузка или выбор из Drive)
3. Файл сохраняется в Drive в соответствующей папке
4. В карточке сущности отображается ссылка на файл
5. Один файл может быть привязан к нескольким сущностям (не дублируется)

## Основные возможности

### Предпросмотр (Preview)

Просмотр файлов без скачивания прямо в интерфейсе NBOS:

| Тип файла | Поддержка предпросмотра |
|-----------|----------------------|
| Изображения (PNG, JPG, SVG, WebP) | ✅ Встроенный просмотр |
| PDF | ✅ Встроенный просмотр |
| Документы (DOCX, XLSX) | ✅ Через рендерер |
| Видео (MP4, WebM) | ✅ Встроенный плеер |
| Код (JSON, JS, HTML, CSS) | ✅ Подсветка синтаксиса |
| Архивы (ZIP, RAR) | ❌ Только скачивание |

### Версионирование

- При загрузке файла с таким же именем → создаётся новая версия
- История версий: кто загрузил, когда, комментарий к изменению
- Возможность откатиться к предыдущей версии
- Сравнение версий для текстовых файлов

### Поиск

- Полнотекстовый поиск по имени файла
- Фильтры: по проекту, типу файла, дате загрузки, автору
- Поиск по тегам (если файл помечен)
- Быстрый переход к файлу из любого модуля

### Массовые операции

- Множественная загрузка (drag & drop нескольких файлов)
- Массовое скачивание (выбрать файлы → скачать как ZIP)
- Массовое перемещение между папками
- Массовое удаление (с подтверждением)

## Управление доступом

### Уровни доступа

| Область | Кто имеет доступ | Правило |
|---------|-----------------|---------|
| **Папки проектов** | Участники проекта | На основе роли в проекте |
| **Client Library** | Сотрудники, работающие с клиентом | Seller, PM привязанных проектов |
| **Company Library** | Все сотрудники | Общий доступ на чтение |
| **Personal Drive** | Только владелец | Приватный по умолчанию |
| **Конфиденциальные документы** | Явно указанные сотрудники | Ограниченный доступ (Secret) |

### Конфиденциальные документы

Для файлов с чувствительной информацией (договоры с финансовыми деталями, NDA, внутренние отчёты):

- Метка **«Конфиденциально»** при загрузке
- Доступ только для явно указанных сотрудников
- **Аудит-лог:** фиксация кто просмотрел / скачал файл и когда
- Запрет на расшаривание без разрешения владельца

### Таблица прав по ролям

| Роль | Project Files | Client Library | Company Library | Personal Drive |
|------|--------------|----------------|-----------------|---------------|
| **CEO** | Full (все проекты) | Full | Full (+ управление) | Свой |
| **PM** | Full (свои проекты) | Read (свои клиенты) | Read | Свой |
| **Seller** | Read (свои проекты) | Full (свои клиенты) | Read | Свой |
| **Developer** | Read/Write (свои проекты) | ❌ | Read | Свой |
| **Finance Director** | Read (финансовые файлы) | Read (договоры) | Read | Свой |

## Интеграция с Google Drive (будущее)

### Планируемая функциональность

- Синхронизация конкретных папок NBOS Drive ↔ Google Drive
- Ссылки на файлы в Google Drive из карточек сущностей
- Импорт существующих файлов из Google Drive при миграции

## Связь с другими модулями

### Drive → Projects Hub
- Автоматическое создание структуры папок при создании проекта
- Файлы проекта доступны из карточки проекта → вкладка «Files»

### Drive → CRM
- Файлы сделки (КП, договор) сохраняются в Drive
- При конвертации Deal → Order файлы переносятся в папку проекта

### Drive → Tasks
- Файлы, прикреплённые к задачам, хранятся в Drive
- Доступ к файлам задачи из карточки задачи

### Drive → Finance
- Сканы договоров, актов, счетов-фактур
- Финансовые отчёты (экспорт P&L)

### Drive → Support
- Скриншоты и логи из тикетов Support
- Документация по решению инцидентов

### Drive → Clients
- Client Library интегрирована с модулем Contacts / Companies
- Юридические документы доступны из карточки клиента

## Ограничения и лимиты

| Параметр | Значение |
|----------|---------|
| Максимальный размер файла | 500 MB (настраиваемо) |
| Хранилище на компанию | Определяется тарифом (начальный: 100 GB) |
| Версий одного файла | До 50 |
| Глубина вложенности папок | До 10 уровней |
| Поддерживаемые символы в именах | UTF-8, без спецсимволов (\ / : * ? " < > |) |
