# Модуль Messenger — Обзор

> NBOS Platform — три направления коммуникации: клиенты, проектные чаты, внутренняя команда

## Назначение

Модуль Messenger объединяет **все коммуникации** компании в одном месте. Вместо разрозненных чатов в разных системах (Instagram DM, WhatsApp, Telegram, чаты в CRM) — единая точка входа с контекстом: каждое сообщение привязано к контакту, проекту или задаче.

**Ключевой принцип:** Контекст важнее канала. Неважно, откуда пришло сообщение — важно, к какому проекту/клиенту/задаче оно относится. Messenger автоматически привязывает коммуникацию к контексту.

## Направление 1: Клиентская коммуникация (CRM Omnichannel)

### Концепция

Все входящие сообщения от клиентов из разных каналов собираются в **единый inbox** внутри NBOS. Сотрудник (Seller, PM) отвечает клиенту прямо из платформы, не переключаясь между мессенджерами.

### Поддерживаемые каналы

| Канал | Тип интеграции | Приоритет |
|-------|----------------|-----------|
| **WhatsApp** | Через агрегатор (API) | Высокий — основной канал общения с клиентами |
| **Instagram** | Через агрегатор (Messenger API) | Высокий — источник лидов |
| **Facebook Messenger** | Через агрегатор (Messenger API) | Средний |
| **Telegram** | Через Bot API / агрегатор | Средний (рассматривается) |
| **Email** | IMAP/SMTP интеграция | Низкий (для формальных коммуникаций) |

### Как работает Omnichannel Inbox

```
Клиент пишет в Instagram → Агрегатор → NBOS Inbox
  ├─ Автоматическое определение контакта (по номеру/имени)
  ├─ Привязка к проекту (если контакт привязан к проекту)
  ├─ Назначение ответственному (Seller или PM проекта)
  └─ Уведомление ответственному
```

### WhatsApp: особое место

WhatsApp — главный канал коммуникации с клиентами в регионе. Используется для:

| Сценарий | Описание |
|----------|----------|
| **Автоматические уведомления** | Счёт выставлен, напоминание об оплате, просрочка |
| **Группа по проекту** | Для каждого проекта создаётся WhatsApp-группа с клиентом |
| **Поддержка** | Клиент пишет о проблеме — сообщение попадает в Support тикет |
| **Продажи** | Общение с лидом/сделкой на стадиях CRM |

### Авто-уведомления в WhatsApp

| Триггер | Сообщение клиенту |
|---------|-------------------|
| Invoice создан | «Ваш счёт на [сумма] выставлен. Оплата до [дата]» |
| 5 дней до дедлайна оплаты | «Напоминаем: срок оплаты счёта [дата]» |
| Оплата просрочена | «Счёт просрочен. Пожалуйста, оплатите до [дата]» |
| Просрочка 2-е напоминание | «Повторное напоминание. При неоплате возможна приостановка» |
| Проект готов к сдаче | «Ваш проект готов! Пожалуйста, проверьте и подтвердите» |
| Подписка: очередной месяц | «Ваш ежемесячный счёт на [сумма] выставлен» |

## Направление 2: Проектные внутренние чаты

### Концепция

Каждый проект имеет **один чат** с несколькими **топиками** (аналогично Telegram Topics). Это решает главную проблему: вместо разрозненных чатов (чат сделки, чат разработки, чат задачи) — всё в одном месте с разделением по контексту.

### Структура чата проекта

```
Project Chat: "E-Commerce Brand X"
├── 📌 General (Общий)
│   └── Доступен: всем участникам проекта
│
├── 💰 Sales / Deals (Сделки / Финансы)
│   └── Доступен: Seller, Finance Director, CEO
│
├── 💻 Development (Разработка)
│   └── Доступен: Developer, Designer, QA, PM
│
├── 🔧 Task: "Исправить баг на главной" (#task-142)
│   └── Доступен: Creator, Assignee, Co-Assignee, Observer
│
├── 🔧 Task: "Интеграция платёжной системы" (#task-156)
│   └── Доступен: Creator, Assignee, Co-Assignee, Observer
│
└── 📦 [Архив задач]
    ├── Task: "Дизайн лендинга" (#task-98) — ✅ Closed
    └── Task: "Настройка домена" (#task-102) — ✅ Closed
```

### Топики чата

| Топик | Доступ | Описание |
|-------|--------|----------|
| **General** | Все участники проекта | Общие обсуждения, объявления, организационные вопросы |
| **Sales / Deals** | Seller, Head of Sales, FD, CEO | Обсуждение финансовых вопросов, условий сделки, оплат |
| **Development** | Developer, Designer, QA, PM | Техническое обсуждение, code review, вопросы по реализации |
| **Task Chats** | Участники задачи | Создаются автоматически при первом сообщении в задаче |

### Role-Based Visibility (видимость по ролям)

| Роль сотрудника | General | Sales | Development | Task Chats |
|-----------------|---------|-------|-------------|------------|
| **CEO** | ✅ | ✅ | ✅ | Все |
| **PM** | ✅ | ✅ | ✅ | Все проекта |
| **Seller** | ✅ | ✅ | — | Где участник |
| **Developer** | ✅ | — | ✅ | Где участник |
| **Designer** | ✅ | — | ✅ | Где участник |
| **QA** | ✅ | — | ✅ | Где участник |
| **Finance Director** | — | ✅ | — | — |

### Task Chats: автоматические чаты задач

- Чат задачи **не создаётся по умолчанию**
- Чат появляется, когда кто-то написал первое сообщение в контексте задачи
- Task Chat автоматически привязан к проекту и топику Development
- Участники: Creator, Assignee, Co-Assignee, Observer задачи
- Когда задача закрыта (Done) — Task Chat перемещается в **Архив задач**
- Если задача переоткрыта — чат возвращается в активные

### Расположение чатов

#### В карточке проекта
Секция **«Chat»** в Project Card показывает:
- Все активные топики (General, Sales, Development)
- Все активные Task Chats с непрочитанными сообщениями
- Архив закрытых Task Chats (сворачиваемая секция)

#### В основном Messenger
Все проектные чаты доступны в главном интерфейсе Messenger:
- Группировка по проектам
- Индикаторы непрочитанных сообщений
- Фильтрация: по проекту, по типу топика

## Направление 3: Внутренняя коммуникация команды

### Текущее состояние

Внутренняя коммуникация команды Neetrino ведётся в **Telegram**. Это работает хорошо для:
- Быстрых обсуждений
- Групповых чатов по проектам
- Голосовых и видеозвонков
- Обмена файлами

### Стратегия развития

| Вариант | Описание | Решение |
|---------|----------|---------|
| **A. Полный внутренний мессенджер** | Построить свой аналог Slack/Telegram | Дорого, сложно. Не для v1 |
| **B. Интеграция с Telegram** | Синхронизация Telegram-чатов с NBOS | Предпочтительный вариант для v1 |
| **C. Гибрид** | Простые чаты в NBOS + Telegram для rich-коммуникации | Компромиссный вариант |

### Решение для v1

- **Проектные чаты с топиками** — внутри NBOS (направление 2)
- **Командные обсуждения** — в Telegram (как сейчас)
- **Связь:** Telegram-группы привязываются к проектам в NBOS (ссылка в карточке проекта)

### Развитие (v2+)

Если интеграция с Telegram работает хорошо:
1. Синхронизация сообщений Telegram ↔ NBOS (двусторонняя)
2. Уведомления из NBOS в Telegram-группу проекта
3. Постепенный перенос командных обсуждений в NBOS

## Ключевые функции Messenger

### 1. Создание задачи из сообщения

Любое сообщение в чате можно превратить в задачу:

```
Сообщение в чате: "Нужно исправить баг на странице оплаты"
  → Кнопка "Создать задачу"
    ├─ Title: берётся из текста сообщения
    ├─ Project: текущий проект чата
    ├─ Product: текущий Product (если из Development-топика)
    ├─ Creator: автор сообщения
    ├─ Assignee: выбирается при создании
    └─ Task Chat: автоматически создаётся и привязывается к сообщению
```

Это критически важная функция: не нужно покидать мессенджер, чтобы зафиксировать задачу.

### 2. Группировка и закрепление чатов

| Функция | Описание |
|---------|----------|
| **Pin (Закрепить)** | Важные чаты закрепляются вверху списка |
| **Groups (Группы)** | Пользовательские группы чатов (например: «Мои приоритетные», «Финансовые вопросы») |
| **Archive** | Неактивные чаты перемещаются в архив (не удаляются) |
| **Mute** | Отключение уведомлений для конкретного чата |

### 3. Поиск

- **Глобальный поиск** по всем доступным чатам
- **Поиск внутри чата** по сообщениям
- **Фильтры:** по дате, по отправителю, по типу (текст, файлы, задачи)

### 4. Файлы и вложения

- Обмен файлами в чатах (изображения, документы, архивы)
- Файлы из чатов автоматически попадают в **Drive** проекта (если чат проектный)
- Превью изображений и документов прямо в чате

### 5. Уведомления

| Тип уведомления | Описание |
|-----------------|----------|
| **Badge** | Счётчик непрочитанных на иконке чата |
| **Push** | Push-уведомление (если включено) |
| **In-app** | Уведомление внутри NBOS |
| **Sound** | Звуковой сигнал (настраиваемый) |

Настройки уведомлений — **per-chat**: для каждого чата можно настроить индивидуально.

### 6. Mentions (@упоминания)

- `@имя` — упоминание конкретного участника (уведомление)
- `@all` / `@everyone` — упоминание всех участников чата
- `@задача` / `#task-142` — ссылка на задачу

## Фазы реализации

### Phase 1: Проектные чаты с топиками (MVP)

**Scope:**
- Чат-сущность для каждого проекта
- Три стандартных топика: General, Sales, Development
- Role-based visibility (кто видит какой топик)
- Базовые функции: текстовые сообщения, файлы, @mentions
- Интерфейс: встроен в Project Card + доступен из основного Messenger

**Приоритет:** Высокий — решает основную боль (разрозненные чаты)

### Phase 2: Task Chats + создание задач из чата

**Scope:**
- Автоматические Task Chats при первом сообщении
- Кнопка «Создать задачу» из любого сообщения
- Архив закрытых Task Chats
- Группировка и закрепление чатов

**Приоритет:** Высокий — тесная связь с Task System

### Phase 3: WhatsApp / Instagram / Facebook Omnichannel

**Scope:**
- Подключение агрегатора (Wazzup, Chat2Desk, или аналог)
- Единый Inbox для входящих сообщений клиентов
- Авто-привязка к контактам и проектам
- Автоматические уведомления клиентам (Invoice, напоминания)
- Шаблоны сообщений для WhatsApp Business

**Приоритет:** Средний — требует интеграций с внешними сервисами

### Phase 4: Telegram Sync для команды

**Scope:**
- Привязка Telegram-групп к проектам NBOS
- Двусторонняя синхронизация сообщений
- Бот для создания задач прямо из Telegram
- Уведомления из NBOS в Telegram

**Приоритет:** Низкий — зависит от успешности Phases 1–3

## Технические решения

### Агрегатор для Omnichannel

Рекомендуется использовать сервис-агрегатор для подключения каналов (Instagram, Facebook, WhatsApp). Варианты:
- **Wazzup** (фокус на WhatsApp + Instagram)
- **Chat2Desk** (мультиканальный)
- **MessageBird / Vonage** (международные решения)

Критерии выбора: поддержка WhatsApp Business API, Instagram Messaging API, API для интеграции с NBOS, стоимость, надёжность.

### Real-time

Чаты требуют **real-time** доставки сообщений:
- WebSocket для web-клиента
- Push Notifications для мобильных
- Серверная очередь для гарантированной доставки
