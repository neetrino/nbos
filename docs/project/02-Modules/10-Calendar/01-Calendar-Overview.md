# Модуль Calendar — Обзор

> NBOS Platform — календарные представления событий, дедлайнов и расписаний

## Назначение

Модуль Calendar предоставляет **календарные представления** для всех событий, происходящих в компании. Встречи с клиентами, внутренние совещания, дедлайны проектов, расписания сотрудников и даты финансовых обязательств — всё собрано в одном интерфейсе с возможностью фильтрации по категориям.

**Ключевой принцип:** Calendar — это не отдельная система планирования, а **агрегирующий модуль**, который собирает даты и события из всех остальных модулей платформы и отображает их в удобном календарном формате.

---

## Структура модуля: четыре вкладки

| Вкладка | Назначение | Источник данных |
|---------|-----------|----------------|
| **Meetings** | Встречи с клиентами и внутренние совещания | Ручное создание, CRM, Projects Hub |
| **Deadlines** | Дедлайны проектов, продуктов, спринтов, задач | Projects Hub, Tasks |
| **Team Schedule** | Рабочие графики сотрудников | My Company → Team |
| **Billing Calendar** | Даты финансовых обязательств | Finance, Subscriptions |

## Вкладка 1: Meetings (Встречи)

### Типы встреч

| Тип | Участники | Примеры |
|-----|----------|---------|
| **Client Meeting** | Сотрудники + клиент(ы) | Презентация КП, обсуждение требований, демо проекта |
| **Internal Meeting** | Только сотрудники | L10 Weekly, планирование спринта, ретроспектива, 1-on-1 |

### Карточка встречи

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| **Название** | Текст | ✅ | Тема встречи |
| **Дата и время** | DateTime | ✅ | Когда проходит встреча |
| **Длительность** | Число (мин) | ✅ | Плановая продолжительность |
| **Тип** | Enum | ✅ | Client Meeting / Internal Meeting |
| **Участники (внутренние)** | Пользователи | ✅ | Сотрудники компании |
| **Участники (внешние)** | Контакты | ❌ | Клиенты (из модуля Contacts) |
| **Проект** | Связь | ❌ | К какому проекту относится встреча |
| **Сделка** | Связь | ❌ | К какой сделке относится (для sales meetings) |
| **Локация / Ссылка** | Текст / URL | ❌ | Офис, адрес клиента или ссылка на Zoom/Google Meet |
| **Заметки** | Rich Text | ❌ | Повестка, подготовительные материалы |
| **Итоги** | Rich Text | ❌ | Заполняется после встречи: результаты, договорённости |
| **Напоминание** | Enum | ❌ | За 15 мин / 30 мин / 1 час / 1 день |

### Создание встречи

Встречу можно создать из нескольких мест:

1. **Модуль Calendar** — напрямую через кнопку «Новая встреча»
2. **Карточка сделки (Deal)** — кнопка «Назначить встречу» на стадии «Meeting»
3. **Карточка проекта (Project)** — кнопка «Назначить встречу» в проектном контексте
4. **Карточка контакта (Contact)** — кнопка «Назначить встречу» для клиентской встречи

### Обнаружение конфликтов

Система автоматически проверяет:

- Если у участника уже есть встреча на это время → **предупреждение о конфликте**
- Пользователь может проигнорировать предупреждение или выбрать другое время
- Конфликт проверяется для всех внутренних участников

### Google Calendar (будущая интеграция)

- Двусторонняя синхронизация с Google Calendar
- Встречи из NBOS → Google Calendar (для уведомлений на телефоне)
- Внешние события из Google Calendar → отображаются в NBOS (для обнаружения конфликтов)

## Вкладка 2: Deadlines (Дедлайны)

### Источники дедлайнов

Дедлайны автоматически агрегируются из различных модулей:

| Источник | Сущность | Пример |
|----------|---------|--------|
| **Projects Hub** | Order Deadline | Сдача проекта «ClientBrand Website» — 15 марта |
| **Projects Hub** | Product Deadline | Готовность MVP мобильного приложения — 1 февраля |
| **Tasks** | Sprint End Date | Окончание Sprint 3 — 28 января |
| **Tasks** | Task Due Date | Задача «Дизайн главной» — 20 января |
| **CRM** | Deal Follow-up | Перезвонить клиенту — 12 января |

### Цветовая кодировка

| Цвет | Статус | Условие |
|------|--------|---------|
| 🟢 Зелёный | **On Track** | До дедлайна > 7 дней |
| 🟡 Жёлтый | **Approaching** | До дедлайна ≤ 7 дней |
| 🔴 Красный | **Overdue** | Дедлайн просрочен |

### Фильтры

- По проекту: показать дедлайны конкретного проекта
- По продукту: дедлайны в рамках одного продукта
- По исполнителю: дедлайны задач конкретного сотрудника
- По типу: только Order / только Task / только Sprint
- По статусу: только просроченные, только приближающиеся

### Навигация

Клик по дедлайну → переход к исходной сущности:
- Дедлайн заказа → карточка проекта
- Дедлайн задачи → карточка задачи
- Дедлайн спринта → доска спринта

## Вкладка 3: Team Schedule (Расписание команды)

### Источник данных

Каждый сотрудник заполняет своё рабочее расписание в профиле (My Company → Team → карточка сотрудника):

- Рабочие дни (например: пн–пт)
- Рабочие часы (например: 10:00–19:00)
- Выходные дни
- Отпуска (с датами начала и окончания)
- Больничные / отгулы

### Отображение

| Элемент | Цвет / Стиль | Описание |
|---------|-------------|----------|
| Рабочий день | Обычный фон | Сотрудник доступен |
| Выходной | Серый фон | Сотрудник не работает |
| Отпуск | Синяя полоса | Плановый отпуск |
| Больничный | Красная полоса | Отсутствие по болезни |

### Использование

- **Планирование встреч:** при создании встречи видно, кто доступен в выбранный день/час
- **Планирование нагрузки:** PM видит доступность разработчиков при назначении задач
- **Учёт отпусков:** CEO/HR видит кто в отпуске, планирование замен

## Вкладка 4: Billing Calendar (Календарь платежей)

### Назначение

Финансовый директор использует эту вкладку для **планирования денежного потока** (cash flow). Здесь отображаются все предстоящие финансовые даты.

### Типы событий

| Тип | Источник | Пример |
|-----|---------|--------|
| **Subscription Invoice** | Finance → Subscriptions | Ежемесячный счёт клиенту «ClientBrand» — 1-е число |
| **Domain Renewal** | Finance → Domains | Продление домена clientbrand.com — 15 марта |
| **Hosting Payment** | Finance → Hosting | Оплата VPS Beget — 1 февраля |
| **Planned Expense** | Finance → Expenses | Оплата Figma — 10 числа каждого месяца |
| **License Renewal** | Finance → Licenses | Продление Apple Developer — 5 июня |

### Цветовая кодировка

| Цвет | Значение |
|------|---------|
| 🟢 Зелёный | Доход (входящие платежи от клиентов) |
| 🔴 Красный | Расход (исходящие платежи) |
| 🟡 Жёлтый | Pass-through (клиент платит за инфраструктуру) |

### Использование

- Просмотр предстоящих входящих и исходящих платежей за месяц
- Оценка остатка на конец месяца
- Раннее обнаружение кассовых разрывов
- Планирование оплаты крупных расходов

## Общий вид календаря

### Режимы отображения

| Режим | Описание |
|-------|---------|
| **Месяц** | Обзор всех событий за месяц в сетке |
| **Неделя** | Детальный вид по часам за неделю |
| **День** | Почасовое расписание на один день |

### Комбинированный вид

По умолчанию календарь показывает **все события из всех вкладок** с возможностью фильтрации:

- Чекбоксы для включения/выключения категорий (Meetings / Deadlines / Schedule / Billing)
- Фильтр по проекту
- Фильтр по сотруднику

### Персональный календарь

Каждый сотрудник видит **свой персональный календарь**, который включает:

- Встречи, где он участник
- Дедлайны его задач
- Его рабочий график и отпуска
- Billing-события (только для Finance Director и CEO)

## Связь с другими модулями

### Calendar → CRM
- Встречи из Deal Pipeline (стадия «Meeting») отображаются в Calendar
- Follow-up дедлайны сделок видны во вкладке Deadlines

### Calendar → Projects Hub
- Дедлайны заказов и продуктов агрегируются автоматически
- Встречи по проектам отображаются в Calendar и карточке проекта

### Calendar → Tasks
- Дедлайны задач и спринтов отображаются во вкладке Deadlines
- При назначении задачи можно проверить загрузку сотрудника через Team Schedule

### Calendar → Finance
- Billing Calendar получает данные из модуля Finance
- Даты подписок, продлений доменов, плановых расходов

### Calendar → My Company
- Team Schedule основан на данных сотрудников из My Company → Team
- Отпуска и больничные отображаются в расписании

### Calendar → Messenger
- Уведомление о предстоящей встрече за N минут
